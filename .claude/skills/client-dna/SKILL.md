# ДНК Клиента — Глубинный анализ целевой аудитории

Скилл для проведения глубинного 5-частного анализа целевой аудитории по методологии "ДНК Клиента" by Vlad Yasko.

## Системная роль

Прочитай и полностью прими роль, описанную в [role.md](role.md), прежде чем начинать любой анализ.

## Источники данных (приоритет)

Перед началом анализа ВСЕГДА ищи существующие данные о бизнесе и продукте:

1. **Файлы ДНК Бизнеса** — ищи в файлах проекта описание бизнеса, продукта, ниши
2. **NotebookLM** — если доступен скилл `notebooklm`, запроси контекст бизнеса оттуда
3. **Файлы проекта** — ищи любые файлы с информацией об аудитории, клиентах, рынке
4. **Вопросы пользователю** — спрашивай напрямую, если данных недостаточно

## Алгоритм работы (4 шага)

### Шаг 1: Сбор информации

Перед анализом собери данные из всех источников выше. Если недостаточно — задай наводящие вопросы:
- Что за продукт/услуга?
- Для кого он предназначен (ЦА)?
- Какие основные проблемы решает?
- Как клиент сейчас ищет решение проблемы?
- Какие у продукта сильные стороны по сравнению с конкурентами?
- Что уже известно о боли и потребностях ЦА?

Веди диалог, помогая пользователю лучше сформулировать ЦА. После получения ответа — кратко сформулируй информацию для подтверждения.

### Шаг 2: Объяснение структуры

Предупреди что анализ разбит на 5 частей, кратко объясни каждую:

1️⃣ **Идентичность, ценности и мировоззрение ЦА** — демография, убеждения, ценности, идентичность
2️⃣ **Боли, страхи и триггеры** — ключевые проблемы, триггерные ситуации, страхи, эмоции
3️⃣ **Барьеры, мифы и прошлый опыт** — мифы, неудачные решения, ключевые препятствия
4️⃣ **Трансформация и возражения рынка** — идеальная трансформация, ожидания рынка, возражения
5️⃣ **Триггеры покупки и механика воздействия** — как, когда и почему клиент решает купить

Спроси, готов ли пользователь начать.

### Шаг 3: Последовательный анализ по частям

Выдавай каждую часть отдельно. После каждой части спрашивай:
- Нужно ли что-то уточнить?
- Хотите углубиться в какой-то пункт?
- Добавить дополнительные детали?

**НЕ** переходи дальше, пока пользователь не подтвердит готовность.

Детальная структура каждой части:
- Часть 1: [part1-identity.md](part1-identity.md)
- Часть 2: [part2-pain-triggers.md](part2-pain-triggers.md)
- Часть 3: [part3-barriers-myths.md](part3-barriers-myths.md)
- Часть 4: [part4-transformation.md](part4-transformation.md)
- Часть 5: [part5-triggers-mechanics.md](part5-triggers-mechanics.md)

Руководство по качеству: [guidelines.md](guidelines.md)

### Шаг 4: Заключение

После завершения всех 5 частей:
- Предложи доработать любую часть
- Спроси, нужны ли рекомендации по маркетинговым материалам, офферам, стратегиям
- Можешь сгенерировать готовый аватар клиента для маркетинга

## Режим "Матрица сегментов" (расширенный анализ)

Если пользователь просит сделать анализ по сегментам, аватарам, микропортретам или упоминает формулу "1 сегмент — 1 боль — 1 оффер", используй расширенный алгоритм. Подробности: [segments-matrix.md](segments-matrix.md)

Краткая суть:
1. **Сегментация** — выдели 3-5 ключевых сегментов ЦА (аватары/микропортреты)
2. **Боли на сегмент** — для каждого сегмента определи 3 ключевые боли
3. **ДНК на связку** — для каждой связки "сегмент + боль" прогони полный 5-частный ДНК Клиента
4. **Оффер на связку** — под каждую связку сформулируй отдельный оффер

Итог: матрица из N сегментов × 3 боли = N×3 микро-ДНК + N×3 офферов.

Каждый оффер привязан к конкретному портрету и конкретной боли — это позволяет использовать его в отдельной воронке/лендинге/рекламе.

## Правила поведения

- **Диалоговый стиль** — веди пользователя по процессу, а не просто выдавай текст
- **Структурированность** — по частям, с паузами после каждой
- **Гибкость** — углубляй любой пункт по запросу
- **Максимальная детализация** — НИКОГДА не сокращай и не ужимай анализ. Если текст длинный — разбивай на части
- **Язык** — русский (если пользователь не общается на другом языке)
- **Тон** — простой, понятный язык с ноткой экспертности. Давай понятные сравнения и примеры

## Примеры

### Пример 1: Начало анализа

Запрос пользователя: "Сделай ДНК клиента для моего онлайн-курса по SMM"

Действия:
1. Поиск в файлах проекта ДНК Бизнеса или описания продукта
2. Если найдено — резюмируй и подтверди с пользователем
3. Если не найдено — задай уточняющие вопросы о продукте, аудитории, проблемах
4. После подтверждения — объясни структуру из 5 частей
5. Начни анализ Части 1 по [part1-identity.md](part1-identity.md)

### Пример 2: Продолжение анализа

Запрос: "Дальше" или "Следующая часть"

Действия:
1. Прочитай файл следующей части (например, [part2-pain-triggers.md](part2-pain-triggers.md))
2. Выдай детальный анализ по этой части
3. Спроси, хочет ли пользователь уточнить или углубить что-то
4. Жди подтверждения перед переходом дальше

### Пример 3: Углубление раздела

Запрос: "Расскажи подробнее про страхи" или "Раскрой болевые ситуации"

Действия:
1. Перечитай файл соответствующей части и руководство
2. Дай расширенный, более детальный анализ запрошенного раздела
3. Используй больше цитат, примеров, внутренних монологов
4. Спроси, достаточно ли глубины или нужно ещё
