Рейтинг під вимоги BINFIELD

  1. Odoo (Enterprise/Custom) — єдина з прямим доступом до БД (PostgreSQL) для Power BI + необмежена кастомізація + нативний склад для роздатки. Якщо «прямий SQL» — жорсткий, це №1.
  2. Zoho CRM (Enterprise) — найшвидше і найергономічніше для 30 польових менеджерів (Blueprint, Territory, мобільний, телефонія). Power BI — через Analytics/ODBC.
  3. Planfix (Business/Corporate) — максимум гнучкості процесів і документів за менші гроші. Підходить, якщо BI годуємо через API/експорт (не прямий SQL).
  4. Perfectum CRM+ERP (коробка) — UA all-in-one, box = прямий доступ до MariaDB, є виробництво/склад. Менший вендор, слабший мобільний.
  5. Uspacy (поточна) — не рекомендуємо лишати під ці вимоги.
Головна розвилка: вимога Power BI «прямий доступ до БД / SQL» (ст. 4.2 ТЗ) — жорстка?
ТАК → Odoo або Perfectum (єдині з прямим доступом до БД).
BI через конектор/експорт прийнятно → у фінал виходять Zoho та Planfix.

Що і як порівнюємо

5 систем зі стеку CRM Solutions, відібраних під B2B-агро та це ТЗ. Поза порівнянням свідомо: Bitrix24 (рос. походження — ризик), Salesforce/Creatio (enterprise, але дорого й поза нашим основним впровадженням). Критерії згруповано за розділами ТЗ + інтеграції + комерція. Конфігурації: Odoo Custom (потрібні External API + Studio + multi-company), Zoho Enterprise, Planfix Business/Corporate, Perfectum коробка/ERP, Uspacy — поточний тариф.

Важливо по інтеграції BAS/1С: нативної інтеграції немає в жодної системи — скрізь кастом через API. Перевага Odoo/Perfectum — фреймворки/досвід UA-партнерів і прямий доступ до БД, що спрощує і обмін, і BI.

Сильні та слабкі сторони кожної системи
1. Odoo (Enterprise / Custom)self-host · ~$14–17.5k/рік (25 юз) + впровадження

Плюси

  • Прямий PostgreSQL → Power BI без API-лімітів
  • Необмежена кастомізація (код + Studio)
  • Нативний склад → віртуальні склади роздатки з rollup
  • Власне виробництво (MRP), якщо знадобиться
  • Повне володіння даними, без vendor lock-in
  • UA-партнери з досвідом BAS-інтеграцій

Мінуси

  • Найдовше і найдорожче впровадження (6–12 міс)
  • Потрібен партнер + адміністратор/DevOps
  • UA-локалізація офіційно відсутня (бухгалтерія лишається в BAS)
  • Ергономіка воронки слабша за «чисту» CRM
2. Zoho CRM (Enterprise)хмара · ~$12–15k/рік (25 юз) + впровадження

Плюси

  • Blueprint — ідеально під статусну логіку ТЗ
  • Territory Management + Roles/Profiles під регіони
  • Найсильніший мобільний + GPS check-in + RouteIQ
  • PhoneBridge (100+ телефоній), дедуп по полю
  • Найшвидший запуск (8–12 тижнів)

Мінуси

  • Power BI — не прямий SQL (через Analytics/ODBC)
  • Склад роздатки — кастом-модуль
  • Один контакт ↔ кілька компаній — обхідно
  • Дані в закритій хмарі (без прямого доступу до БД)
3. Planfix (Business / Corporate)хмара · найгнучкіша · нижча ціна

Плюси

  • Objects + сценарії — глибока кастомізація процесів
  • Історія статусів у звітах — дослівно вимога ТЗ
  • Аналітики (позиції угод) + генерація документів
  • Гранульовані ролі; телефонія; UA; ціна

Мінуси

  • Power BI прямого доступу до БД немає (API/експорт)
  • API ліміт 1 req/сек → ризик для real-time BAS + повного BI на обсязі
  • Склад роздатки з rollup — кастом на аналітиках
  • Мобільний/GPS і воронка на планшеті слабші за Zoho
4. Perfectum CRM+ERP (коробка)self-host · UA all-in-one

Плюси

  • Коробка = прямий доступ до MariaDB для Power BI
  • Є виробництво + склад + CRM в одному
  • UA-вендор, ближче до BAS
  • Одноразова ліцензія коробки

Мінуси

  • Менший вендор; менше перевірено на такій складності
  • Слабкий iOS-застосунок (критично для поля)
  • Кастомізація/екосистема менші за Odoo/Zoho
5. Uspacy (поточна)хмара · вже впроваджена, але не влаштовує

Плюси

  • Уже в експлуатації, команда знайома
  • UA, дешева, є Process Builder + API/вебхуки
  • Базові ролі/права

Мінуси

  • Немає прямого доступу до БД для Power BI
  • Глибина кастомних об'єктів нижча за потребу ТЗ
  • Поточне налаштування визнане невдалим (Ліди замість Угод)
  • Слабші: склад/rollup, поле/GPS
Рекомендація та наступний крок

Якщо «прямий SQL до БД» для Power BI — жорстка вимога (а в ТЗ вона прописана прямо) → рекомендуємо Odoo як основний варіант: він єдиний нативно дає прямий доступ до даних, закриває склад роздатки і має найбільший запас кастомізації; Perfectum — бюджетна UA-альтернатива з тією ж перевагою (коробка/БД), але з ризиками меншого вендора й мобільного.

Якщо Power BI можна годувати через конектор/експорт (Analytics/ODBC чи нічні вивантаження) → у фінал виходять Zoho (найкраще для 30 польових менеджерів і найшвидший запуск) та Planfix (максимум гнучкості процесів за менші гроші).

Uspacy під повний обсяг ТЗ лишати не радимо — навіть після перебудови залишаються стелі по BI та глибині об'єктів.

Щоб зафіналити — потрібно від клієнта 3 відповіді: (1) Power BI «прямий SQL до БД» — обов'язково чи можна через конектор? (2) Бюджет на ліцензії + впровадження і бажаний строк запуску? (3) Самохостинг прийнятний (для прямого доступу до БД потрібен self-host / Odoo.sh / коробка)?
