## ⚡ Специфіка будівельної ніші: чому звичайна CRM не вирішує задачу

Будівельно-ремонтний бізнес має унікальний операційний цикл, якого немає в торгівлі чи сервісі:

```
Лід → Замір (виїзд) → Кошторис з версіями → КП/Договір → Авансовий платіж
   → Об'єкт (адреса, площа, тип, фото) → Поетапне виконання → Поетапна оплата
   → Закупівлі під об'єкт → Бригади/прораби → КБ2/КБ3/Акти → Гарантія
```

**Ключова відмінність від e-commerce або B2B послуг:**

| Критерій | E-commerce CRM | Будівельна CRM/ERP |
|----------|---------------|-------------------|
| Цикл угоди | Години–дні | Тижні–місяці |
| Кошторис | Ціна товару × кількість | Матеріали + Роботи + Норми + Версії |
| Оплата | Передоплата або постоплата | Графік платежів по етапах |
| Об'єкт | Адреса доставки | Окрема сутність: площа, тип, стадія, фото, документи |
| Виконавці | Кур'єр | Прораб + бригади + субпідрядники |
| Документи | Накладна, рахунок | Договір, КБ2, КБ3, Акт виконаних робіт, Гарантійний лист |
| Склад | Залишки товарів | Матеріали «під об'єкт», списання, постачальники |

**Ключовий висновок:** Для більшості будівельних та ремонтних сценаріїв чиста товарна CRM (KeyCRM, KeepinCRM без ERP-доналаштування) закриває лише частину задачі — **продажний шар**. Для повного циклу потрібна або **спеціалізована будівельна система**, або **CRM+ERP із кастомним впровадженням**, або **зв'язка CRM + PM-система**.

```

CRM + PM (зв'язка)       |  Sales + управління об'єктами/задачами             |  Ремонтні та монтажні компанії
```

| 🥈 2 | **Uspacy** | Найкращий баланс CRM+комунікації+задачі; AI-транскрибація дзвінків; підходить для кількох паралельних ремонтів |

## ⚠️ Підводні камені специфічні для будівництва

### 1. Кошториси — головний біль будівельників

- **Жодна загальна CRM** (KeyCRM, KeepinCRM, Kommo, Uspacy, Pipedrive, HubSpot) **не має нативного будівельного кошторису** з матеріалами+роботами+версіями+нормами.
- Збирати кошторис через «товари в угоді» — **не те саме**, що повноцінний будівельний кошторис.
- **Рішення:** Міністерство Ремонту (нативний), Perfectum/OneBox (через кастомне впровадження), або зовнішній Google/Excel + синхронізація з CRM.
- ⚠️ **Прихована вартість:** При впровадженні KeepinCRM/KeyCRM під будівництво неминучі витрати на інтегратора ($900–$1 200+) та паралельний Excel для кошторисів.

### 2. Поетапна оплата — специфіка «не торгівлі»

- Більшість CRM заточені під «один платіж по угоді».
- Для будівництва потрібен **графік платежів прив'язаний до етапів об'єкта**: аванс → фундамент → коробка → здача → гарантійне утримання.
- Без ERP-логіки (Perfectum, OneBox) цей графік доведеться вести в Excel паралельно з CRM.

### 3. Картка об'єкта — не картка клієнта

- У більшості CRM центральна сутність — «Клієнт» або «Угода».
- У будівництві центральна сутність — **«Об'єкт»**: адреса, площа, тип (квартира/котедж/офіс), стадія, відповідальний прораб, фото по етапах, документи, матеріали.
- Вирішення: Спеціалізовані системи (Міністерство Ремонту, G-Plus, Perfectum) мають нативний об'єкт; загальні CRM потребують налаштування кастомних полів і сутностей.

### 4. Мобільний прораб на об'єкті

- Замірники та прораби не сидять в офісі — їм потрібен зручний мобільний застосунок, не лише браузерна адаптація.
- ✅ Нативний мобільний «польовий» застосунок: Міністерство Ремонту (Remka), Worksection, KeyCRM, Perfectum, Kommo.
- ❌ Без повноцінного мобільного застосунку: KeepinCRM (веб-адаптація), G-Plus (не підтверджено).
- **Унікальна фіча Міністерства Ремонту:** «Буферна зона» — прораб фотографує чек → матеріал в облік.

### 5. КБ2 / КБ3 — специфіка B2B будівництва

- Форми КБ2 (Акт виконаних будівельних робіт) і КБ3 (Довідка про вартість виконаних будівельних робіт та витрати) — обов'язкові для бюджетного/державного будівництва та великого B2B.
- **Жодна з розглянутих CRM не має нативних КБ2/КБ3 шаблонів** — потрібне кастомне налаштування або зовнішня спеціалізована програма (кошторисне ПЗ: AVersBudget, BuilderSoft та ін.).
- Perfectum/OneBox закривають це через кастомні шаблони документів.

### 6. Тендери та субпідряд

- Для генпідрядників важлива тендерна документація, комунікація з субпідрядниками, контроль якості на будмайданчику.
- Жодна українська CRM не має готового «тендерного модуля» — потрібна кастомна логіка або міжнародний Procore.

### 7. Bitrix24 — ризик зупинки бізнесу

- Компанії, що залишаються на Bitrix24, **позбавляють себе оновлень, технічної підтримки та інтеграцій з українськими сервісами**.
- Дані клієнтів, кошториси та документи по об'єктах — потенційно вразливі.
- Участь у державних тендерах/держзамовленні при використанні РФ-ПЗ — **юридичний ризик**.
- **Рекомендація:** Планувати міграцію, не відкладаючи.
