# E-commerce / Інтернет-магазин — бізнес-процеси та промпт

## Типи бізнесу в ніші

| Тип | Опис | Особливості CRM |
|-----|------|-----------------|
| Monostore | Один товар / вузька категорія | Простіша воронка, акцент на повторні продажі |
| Мультикатегорійний | Широкий асортимент | Потрібен складський облік |
| Маркетплейс-продавець | Тільки Prom/Rozetka/OLX | Критична інтеграція з майданчиками |
| Соцмережевий продаж | Instagram/Telegram/Facebook | Мультиканальний inbox — must have |
| Дропшипінг | Без власного складу | Синхронізація з постачальником |
| B2B оптовий | Продаж партіями бізнесам | Довгий цикл, рахунки, договори |

## Ключові бізнес-процеси

### 1. Отримання замовлень (мультиканал)
- Сайт (WooCommerce, Хорошоп, Shopify, Prom, Rozetka)
- Instagram Direct, Facebook Messenger, Telegram-бот
- Телефон / Viber / WhatsApp
- OLX, маркетплейси
- **Проблема:** замовлення з 5-10 каналів треба зводити в одне місце

### 2. Обробка замовлення
- Підтвердження клієнту (дзвінок або повідомлення)
- Уточнення адреси, варіанту товару, кольору/розміру
- Перевірка наявності на складі
- **Проблема:** менеджер вручну перевіряє наявність

### 3. Оплата
- LiqPay, WayForPay, Monobank (Plata), накладений платіж
- Передоплата vs накладений — різні воронки
- Контроль оплачено/не оплачено/відмова
- **Проблема:** хто не доплатив, хто відмовився від посилки

### 4. Склад і залишки
- Резервування товару під замовлення
- Поповнення стоку від постачальника
- Синхронізація залишків між майданчиками
- **Проблема:** продали те чого немає; пересорт

### 5. Доставка
- Оформлення ТТН Nova Poshta / Укрпошта / Justin / Meest
- Автозаповнення ТТН з картки замовлення
- Трекінг посилок, статуси доставки
- **Проблема:** ручне введення ТТН = помилки і час

### 6. Повернення / Відмови
- Клієнт не забрав посилку (НП повертає)
- Обмін / повернення коштів
- Рекламації по якості
- **Проблема:** облік повернень і збитків

### 7. Комунікація з клієнтом
- Автоповідомлення про статус замовлення
- Відповіді на питання в месенджерах
- Нагадування про незавершене замовлення
- **Проблема:** ручна комунікація при 50+ замовленнях/день

### 8. Повторні продажі / Лояльність
- RFM-аналіз (хто купував що і коли)
- Розсилки по базі (email, Viber, SMS)
- Програма лояльності, знижки постійним клієнтам

### 9. Аналітика
- Продажі по каналах (звідки прийшов клієнт)
- Конверсія по менеджерах
- Середній чек, маржа, повернення

## Промпт для Deep Research

```
Зроби детальне порівняння CRM-систем для інтернет-магазину
(e-commerce) в Україні, 2025-2026 рік.

КОНТЕКСТ БІЗНЕСУ:
- Інтернет-магазин: 1-20 менеджерів, 50-2000 замовлень/місяць
- Канали продажів: власний сайт + Instagram + Prom.ua/Rozetka +
  Telegram
- Доставка: Nova Poshta, Укрпошта, Justin
- Оплата: LiqPay, WayForPay, накладений платіж
- Склад: власний або дропшипінг

КЛЮЧОВІ БІЗНЕС-ПРОЦЕСИ ДЛЯ ОЦІНКИ:
1. Збір замовлень з усіх каналів в одному вікні
2. Автоматичне оформлення ТТН Nova Poshta
3. Синхронізація залишків між майданчиками
4. Комунікація з клієнтом (месенджери + автоповідомлення)
5. Контроль оплат і повернень
6. RFM-аналітика і повторні продажі

КРИТЕРІЇ ПОРІВНЯННЯ:
1. Інтеграції з українськими маркетплейсами
   (Prom, Rozetka, OLX, Etsy, Amazon)
2. Інтеграція Nova Poshta (автоТТН, трекінг)
3. Мультиканальний inbox (Instagram, Telegram, Viber, WhatsApp)
4. Складський облік залишків
5. AI-функції (автовідповіді, скоринг, прогнози)
6. Ціна (щомісяця, в USD або грн)
7. Наявність україномовної підтримки
8. Складність впровадження (хто налаштовує?)
9. Кейси українських інтернет-магазинів

НЕ ОБМЕЖУЙСЯ відомими CRM. Включи:
- Українські: KeyCRM, KeepinCRM, SalesDrive, LP-CRM
- Міжнародні: Kommo, Pipedrive, HubSpot, Zoho, Odoo
- Нішеві: будь-які спеціалізовані для e-commerce
- Платформенні: вбудований CRM Хорошоп, Prom, Rozetka

Якщо існує спеціалізована CRM для e-commerce з Nova Poshta /
українськими маркетплейсами — вона пріоритетна.

ФОРМАТ ВІДПОВІДІ:
1. Таблиця порівняння по всіх критеріях
2. Топ-3 рекомендації для різних сценаріїв:
   - малий магазин (до 100 замовлень/міс)
   - середній (100-500 замовлень/міс)
   - великий (500+ замовлень/міс)
3. AI-функції окремим блоком — що реально є зараз
4. Підводні камені кожної CRM для e-commerce
```
