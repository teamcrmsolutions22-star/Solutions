## ⚡ Ключове розмежування: CRM ≠ LOS/LMS ≠ страхова core-система

```
Клас               |  Що вміє                                                |  Кому підходить
-------------------|----------------------------------------------------------|-----------------
CRM (front-office) |  Ліди, воронка, комунікації, активності, pipeline        |  Будь-яка ФК як доповнення
LOS                |  Loan Origination: анкета → скоринг → рішення → видача   |  МФО, ліцензовані кредитори
LMS                |  Loan Management: нарахування %, графік платежів, NPL     |  МФО, банки, лізинг
Collection CRM     |  NPL-колекшн: стадії, автодзвінки, суд, реструктуризація |  МФО + банки
Insurance Core     |  Тарифи, полісами, збитки, перестрахування, регул. звітність | Страховики
Agent Portal       |  Порівняння тарифів, продаж полісів агентами/брокерами   |  Агентські мережі, СК
Обмін валют        |  Каса, курси, Form 377, мультивалюта                     |  Обменники, банки
Enterprise CRM/BPM |  Кредитний конвеєр, 360° клієнт, BPM, AI-агенти          |  Великі банки / ФК
```

**Ключовий висновок:** Для МФО/мікрокредиторів **CRM без LOS/LMS не закриває задачу**. Скоринг, кредитний конвеєр, нарахування відсотків і колекшн живуть у спеціалізованих системах (MicroF1, Lendex, Mahaon). CRM — лише sales-шар поверх core. Для банківського enterprise CRM — потрібні Creatio, XRM Banking (MS Dynamics), Salesforce FSC.

## 🔑 Підводні камені та застереження

### 1. Kommo/amoCRM — буква закону, не просто "ризик"
Обидва major research-джерела (Perplexity, ChatGPT) описали Kommo як "краще уникати через РФ-засновників". Насправді — **Верховна Рада прямо заборонила Kommo/amoCRM** законом від 19.08.2024. Для ФУ під наглядом НБУ це означає пряме порушення, а не просто репутаційний ризик.

### 2. BAS — не "українська альтернатива 1С"
Ряд фінансових установ досі використовував BAS як "безпечну заміну 1С". З 09.01.2026 BAS ERP, BAS Бухгалтерія КОРП, BAS ЗУП — у переліку ДССЗЗІ. Технічно BAS = ліцензована копія 1С (Business Automation Framework від польської NetHelp Jarocki Piotr = ex-1С-розробник).

### 3. INRISK ≠ страхова платформа для продажу
insk.com.ua "INRISK" — власний продукт INSK для управління своїм портфелем. Якщо страхова компанія або агент шукає "страхову CRM" — це не їхній варіант.

### 4. Creatio on-premise ≠ cheap
При мінімальному контракті ~$10K/рік Creatio підходить для банків 100+ осіб. Для МФО на 5–10 осіб — надлишок; краще Perfectum або OneBox.

### 5. Diia.Підпис стає платним
Мінцифри 2025–2026 переходить до платної моделі для юросіб. Враховуйте в TCO при проєктуванні кредитного конвеєру на Diia.Підпис.

### 6. УБКІ = SOAP/XML, не REST
Інтеграція з УБКІ через SOAP/XML API — не RESTful. Додайте 5–15 днів на розробку адаптера якщо система не має нативного коннектора.

### 7. USU для обмінників — due diligence
Незважаючи на широке використання USU в UA обмінних мережах, платформа має ознаки РФ-спадщини у базовому стеку. Для обмінного пункту під ліцензією НБУ — уточнювати актуальний стан юрисдикції та ДССЗЗІ-статус.
