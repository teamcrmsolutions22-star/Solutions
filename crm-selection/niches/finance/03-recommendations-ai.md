## 🔌 ЧАСТИНА 6: Інтеграційні сервіси (НЕ CRM, але обов'язкові для ФК)

### 15. УБКІ — Українське Бюро Кредитних Історій

**Тип:** Кредитне бюро; API для перевірки кредитної історії  
**Документація:** wiki-ubki.atlassian.net  
**API протокол:** SOAP/XML  
**Звіти:**
- **Report 10** — кредитна історія фізичної особи
- **Report 25** — кредитна історія юридичної особи
- **IndScore** — індивідуальний скоринг 0–700 (вищий = краще)

**Де вже інтегровано нативно:**
- MicroF1 ✅
- Lendex ✅
- OneBox ✅ (нативний конектор)
- Perfectum ⚠️ (уточнити)
- Mahaon ✅ (скоринговий модуль)

**Доступ:** Потрібен договір з УБКІ (не публічний API); лише для ліцензованих ФУ або їхніх IT-підрядників

### 16. BankID НБУ ✅ — цифрова ідентифікація

**Масштаб (2025):**
- 39 банків підключено
- 112 провайдерів послуг
- 109,4 млн ідентифікацій проведено

**Як використовується:**
- e-KYC: верифікація особистості без фізичної присутності
- Онлайн-підписання договорів (з електронним підписом через банк)
- Перевірка паспортних даних та РНОКПП

**Інтеграція:** OAuth 2.0; документація на api.bankid.org.ua  
**Підходить для:** МФО, страхові агентства, BNPL-сервіси, будь-яка ФУ що потребує e-KYC

**Тип:** Кваліфікований електронний підпис через застосунок Diia  
**Статус:** Юридично рівнозначний власноручному підпису

**Нативно в CRM/LOS:**
- MicroF1 ✅ (Diia.Підпис для підписання кредитних договорів)

**⚠️ Важлива зміна 2025–2026:** Diia.Підпис **стає платним для юридичних осіб**. Уточнювати актуальну модель монетизації у Мінцифри.

**Для чого використовувати в ФС:**
- Підписання кредитних договорів онлайн
- Верифікація клієнта (e-KYC)
- Підписання страхових полісів

## 🔑 Підводні камені та застереження

### 1. Kommo/amoCRM — буква закону, не просто "ризик"
Обидва major research-джерела (Perplexity, ChatGPT) описали Kommo як "краще уникати через РФ-засновників". Насправді — **Верховна Рада прямо заборонила Kommo/amoCRM** законом від 19.08.2024. Для ФУ під наглядом НБУ це означає пряме порушення, а не просто репутаційний ризик.

### 2. BAS — не "українська альтернатива 1С"
Ряд фінансових установ досі використовував BAS як "безпечну заміну 1С". З 09.01.2026 BAS ERP, BAS Бухгалтерія КОРП, BAS ЗУП — у переліку ДССЗЗІ. Технічно BAS = ліцензована копія 1С (Business Automation Framework від польської NetHelp Jarocki Piotr = ex-1С-розробник).

### 3. INRISK ≠ страхова платформа для продажу
insk.com.ua "INRISK" — власний продукт INSK для управління своїм портфелем. Якщо страхова компанія або агент шукає "страхову CRM" — це не їхній варіант.

### 4. Creatio on-premise ≠ cheap
При мінімальному контракті ~$10K/рік Creatio підходить для банків 100+ осіб. Для МФО на 5–10 осіб — надлишок; краще Perfectum або OneBox.

### 5. Diia.Підпис стає платним
Мінцифри 2025–2026 переходить до платної моделі для юросіб. Враховуйте в TCO при проєктуванні кредитного конвеєру на Diia.Підпис.

### 6. УБКІ = SOAP/XML, не REST
Інтеграція з УБКІ через SOAP/XML API — не RESTful. Додайте 5–15 днів на розробку адаптера якщо система не має нативного коннектора.

### 7. USU для обмінників — due diligence
Незважаючи на широке використання USU в UA обмінних мережах, платформа має ознаки РФ-спадщини у базовому стеку. Для обмінного пункту під ліцензією НБУ — уточнювати актуальний стан юрисдикції та ДССЗЗІ-статус.

### 8. BAS ≠ простий в інтеграції з CRM
Більшість українських компаній використовує BAS для бухобліку. Інтеграція CRM ↔ BAS супроводжується типовими труднощами:
- **Різна структура даних:** конфлікти синхронізації карток контрагентів та платежів
- **Навантаження на сервери:** передача в реальному часі може перевантажувати серверну частину BAS
- Вирішення: досвідчені інтегратори + проміжні шини даних (API gateway)
- ⚠️ Нагадування: BAS ERP/Бухгалтерія КОРП/ЗУП — у переліку ДССЗЗІ (09.01.2026)

### 9. Приховані витрати хмарних CRM у фінтеху
- **KeyCRM:** Немає плати за користувачів, але стягується +$9/міс за кожні наступні 200 замовлень, 2000 карток у воронках або 20 000 повідомлень. При високій інтенсивності фінтех-транзакцій чек зростає швидко.
- **KeepinCRM:** Безкоштовний тариф (1 user) — після оновлення серпня 2024 заблоковано ведення фінансів в угодах, створення завдань та кастомних полів. Непридатний для реального тестування фінансовими компаніями.
- **Uspacy:** У зв'язку зі зростанням витрат на EU-хмарну інфраструктуру (курс EUR +25% з моменту запуску), компанія оголосила підвищення тарифів з 1 січня 2026 року. Враховуйте при довгостроковому бюджетуванні.
- **Enterprise (Salesforce, MS Dynamics):** Від $10 000 на старті впровадження + штат сертифікованих розробників для підтримки кастомного коду.

### 10. Маркетингові твердження "є KYC/AML" — перевіряти
Часто це означає лише форму збору документів без реальної інтеграції з Diia, BankID, УБКІ, санкційними списками або PEP-базами. Необхідно вимагати: конкретний перелік інтеграцій + технічну документацію + SLA. Вартість кастомної інтеграції з УБКІ (SOAP/XML → REST-адаптер): 5–15 робочих днів розробки.

## 📋 Детальні вимоги Постанови НБУ №99 (ключові для вибору CRM)

4 ключові вимоги, що безпосередньо впливають на вибір CRM та Core-систем:

1. **Географічні обмеження:** Заборонено обробляти дані на тимчасово окупованих територіях України, у державах-агресорах та суб'єктів під санкціями. Під час ВС (та 2 роки після) дозволено: ЄС / Велика Британія / США / Канада.
2. **Сертифікація провайдера:** Хмарний провайдер щорічно надає сертифікати відповідності ISO/IEC 27001 та ін.
3. **План безперервності бізнесу (BCP):** ФУ повинна мати план дій при зупинці провайдера, систему резервування даних та швидкий перехід до іншого хмари.
4. **Контроль та моніторинг:** У договорі з провайдером — право ФУ моніторити доступ співробітників провайдера до конфіденційних даних; 18 обов'язкових пунктів договору (Розділ IV Положення НБУ).

**Виявлення заборонених систем під час перевірки НБУ** гарантовано призводить до санкцій аж до відкликання ліцензії.
