# Медицина / Клініки / Стоматологія — бізнес-процеси та промпт

## Типи бізнесу в ніші

| Тип | Опис | Особливості CRM |
|-----|------|-----------------|
| Стоматологія | 1-5 лікарів, 20-200 прийомів/міс | Розклад крісел, нагадування, план лікування |
| Приватна клініка | Мультиспеціалізація, 5-30 лікарів | Медична карта, напрямки між лікарями |
| Косметологія / Естетична медицина | 1-10 майстрів | Близько до beauty-ніші, але з медичними протоколами |
| Лабораторія / Діагностика | Аналізи, УЗД, МРТ | Велика кількість звернень, короткий цикл |
| Онлайн-медицина / Телемедицина | Консультації онлайн | Відеодзвінки, оплата онлайн, електронні рецепти |
| Реабілітаційний центр | Курси лікування | Довгий цикл, повторні візити, програми |

## Ключові бізнес-процеси

### 1. Первинний запис пацієнта
- Дзвінок, сайт, Instagram, Viber, Telegram, Google Maps (відгуки → дзвінок)
- Онлайн-запис (виджет на сайті або у месенджері)
- Перевірка доступності лікаря у потрібний час
- **Проблема:** адміністратор вручну веде кілька журналів + телефон + месенджери

### 2. Розклад і управління записами
- Сітка часових слотів по кожному лікарю / кабінету
- Повторний запис після першого візиту
- Нагадування пацієнту за 24 год та 2 год до прийому (знижує no-show)
- Переносення / скасування запису
- **Проблема:** no-show rate 20-30% без автонагадувань

### 3. Медична карта пацієнта
- Анамнез, діагнози, призначення, протоколи лікування
- Фото (до/після для косметології, стоматології)
- Результати аналізів та обстежень
- Алергії та протипоказання
- **Проблема:** паперові карти або Excel → втрата даних, немає доступу з мобільного

### 4. План лікування та рахунки
- Стоматологія: план лікування = список процедур → сума
- Генерація рахунку / договору на послуги
- Контроль оплат: передоплата / оплата після / розстрочка
- **Проблема:** ручне виставлення рахунків і контроль боргів

### 5. Комунікація з пацієнтами
- Нагадування про запис (SMS, Viber, Telegram)
- Нагадування про повторний огляд через N місяців
- Збір зворотного зв'язку після візиту (NPS, Google Reviews)
- Розсилки по базі: акції, нові послуги, сезонні пропозиції
- **Проблема:** без автоматизації адміністратор витрачає 2-3 год/день на обзвон

### 6. Аналітика клініки
- Завантаженість лікарів / кабінетів
- Конверсія: первинний дзвінок → запис → прийом → повторний
- Виручка по лікарях і послугах
- Повернення пацієнтів (retention rate)
- Джерела лідів: звідки приходять нові пацієнти

## Промпт для Deep Research

```
Зроби детальне порівняння CRM та медичних інформаційних систем (МІС)
для приватних медичних закладів України, 2025-2026 рік.

КОНТЕКСТ БІЗНЕСУ (три сегменти):
A. Стоматологія: 1-5 лікарів, 50-300 прийомів/місяць
B. Приватна клініка: мультиспеціалізація, 5-30 лікарів
C. Косметологія / естетична медицина: 1-10 майстрів, акцент на
   повторні візити та Instagram-продажі

КЛЮЧОВІ БІЗНЕС-ПРОЦЕСИ ДЛЯ ОЦІНКИ:
1. Онлайн-запис пацієнта (виджет на сайті, Viber, Telegram, Instagram)
2. Розклад лікарів / кабінетів з управлінням слотами
3. Автоматичні нагадування про запис (SMS, Viber, Telegram) для
   зниження no-show
4. Медична карта пацієнта (анамнез, діагнози, фото до/після, аналізи)
5. План лікування → рахунок → контроль оплат
6. Нагадування про повторний огляд через N місяців
7. Збір відгуків (Google Reviews, NPS) після візиту
8. Аналітика: конверсія первинний → повторний, виручка по лікарях

КРИТЕРІЇ ПОРІВНЯННЯ:
1. Онлайн-запис — є нативно чи через виджет (Booksy, Calendly, власний)
2. Сітка розкладу (calendar view по лікарях і кабінетах)
3. Медична карта з протоколами лікування (є / базова / відсутня)
4. Автонагадування — канали (SMS, Viber, TG) і гнучкість налаштування
5. Інтеграція з Instagram / Facebook Lead Ads (збір заявок з соцмереж)
6. ПРРО / фіскалізація (Checkbox або власна касова програма)
7. AI-функції: транскрибація дзвінків, автовідповіді, скоринг
8. Відповідність GDPR та українським вимогам захисту медичних даних
9. Ціна (USD або грн)
10. Мова підтримки та документації
11. Кейси українських клінік або стоматологій
12. Можливість роботи без IT-спеціаліста (self-service налаштування)

НЕ ОБМЕЖУЙСЯ відомими CRM. Включи:
- МІС (медичні інформаційні системи) для України: Medics, MedControl,
  CleverClinic, ClinicIQ, HealthControl, GoodDoc, MedDesk, Medesk
- Нішеві для краси/медицини: YCLIENTS, Sycret, CleverBOX:CRM,
  Beauty Pro (якщо має медичний модуль)
- Українські загальні CRM: KeyCRM, KeepinCRM, Uspacy, Kommo (UA),
  Planfix (якщо адаптований)
- Міжнародні: HubSpot, Zoho CRM, Salesforce Health Cloud, Pipedrive
- Спеціалізовані для стоматології: Dental4Windows, StomPRO, DentalCRM
- Телемедицина: якщо є рішення з відеоконсультаціями та електронними
  рецептами — включи

СПЕЦИФІКА НІШІ (врахуй у порівнянні):
- Медична карта — не "угода", а окрема сутність з анамнезом і GDPR
- Розклад лікарів — не "воронка", а календар слотів
- No-show — критична проблема: 20-30% записів не приходять без нагадувань
- Розмежування: МІС (облік медичних даних) vs CRM (маркетинг і
  комунікації) — іноді потрібні обидві системи одночасно
- Стоматологія специфічна: план лікування = кілька сесій = довгий цикл
- Косметологія: акцент на фото до/після, повторний продаж курсів

ФОРМАТ ВІДПОВІДІ:
1. Таблиця порівняння по всіх критеріях
2. Топ-3 рекомендації окремо для:
   - Стоматології (1-5 лікарів)
   - Приватної клініки (5-30 лікарів, мультиспеціалізація)
   - Косметології / естетичної медицини
3. Блок: МІС vs CRM — коли брати нішеву МІС, коли вистачить CRM
4. AI-функції — що реально є у 2025-2026 для медицини
5. GDPR / захист медичних даних — як різні системи вирішують питання
6. Підводні камені кожного рішення для медичного бізнесу
```
