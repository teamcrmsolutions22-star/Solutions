## Контекст ринку

- Лише **5–8%** українських агентств нерухомості використовують CRM
- Постачальники **Bitrix24, amoCRM, Kommo** мають **російське походження** → ризики санкцій, витоку даних, репутаційні втрати. Більшість успішних компаній уже мігрують на вітчизняні платформи
- Цикл угоди: **1–6 місяців** (продаж), 1–2 тижні (оренда)
- Об'єкт нерухомості — **не товар**: кожен унікальний, потребує окремої сутності з атрибутами (площа, поверх, секція, юридична історія, фото, статус)
- **~82% продажів** нерухомості відбуваються через рекомендації та повторних клієнтів → CRM має підтримувати довгостроковий nurture-сценарій (нагадування через 3–6 місяців)
- Ієрархія об'єктів у девелопера: **проєкт → будинок → секція → поверх → квартира** — краще реалізується в галузевих CRM
- **LUN.ua** інтеграція: вимагає власного домену 2-го рівня + мінімум 200 об'єктів в активній базі (технічний бар'єр для малих агентств)
- **DOM.RIA / OLX** — основні портали з API-інтеграціями для більшості нішевих CRM
- Договір купівлі-продажу нерухомості (ДКП) в Україні **обов'язково нотаріально посвідчується** — жоден e-підпис не замінює нотаріуса для ДКП
- Воронка нерухомості проходить жорстко регламентовані етапи: перший контакт → презентація пропозицій → облік і організація показів → переговори → завдаток → укладання договору → **державна реєстрація прав власності** → фізична передача ключів
- **Двосторонній пошук**: CRM має підтримувати автопідбір об'єктів під параметри покупця І пошук потенційних покупців під новий об'єкт, що з'явився в базі ✅
- **Комісійна модель** у агентств: нарахування комісії розподіляється між залученим агентом, агентом-виконавцем та компанією за окремою сіткою — в нішевих CRM реалізовано "з коробки", в універсальних — треба доробляти
- **Мультиканальність лідів**: порталі (LUN, DOM.RIA, OLX), соцмережі (FB/IG Lead Ads), месенджери, сайт, рекомендації — оптимальна CRM збирає все в один інбокс з наскрізною аналітикою по джерелах

## Топ-3 рекомендації по сегментах

### A. Ріелторське агентство (5–15 агентів)

**1. Plektan CRM** — найкращий баланс функцій/ціни для команди

- Офіційний партнер OLX — стабільна інтеграція чатів і вивантаження в 1 клік
- Viber, Telegram, SMS в одному вікні
- Впровадження за 7–10 днів, підтримка Пн–Нд 10:00–18:00
- Ціна: **299–459 грн/user/міс** (Solo — від 399 грн/міс; команди — **349 грн/user/міс**)
- Підводний камінь: сайт + Telegram-бот — одноразово 7 000–14 000 грн + хостинг і API-підключення окремо

**2. LigaPro CRM** — лідер ринку за кількістю ріелторів (3000+)

- Спеціалізований модуль об'єктів + парсер власників
- 35+ порталів, включно LUN.ua (при наявності 200+ об'єктів і власного домену)

- Підводний камінь: SMS-орієнтований інбокс, не повноцінний омніканал

**3. RealtSoft** — якщо потрібна і агентська, і девелоперська логіка

- 40+ порталів, автогенерація описів через OpenAI
- Окремий developer-модуль зі шахматкою (якщо агентство має свій ЖК)

## Портальні інтеграції: деталі

### LUN.ua
- **Технічна вимога:** власний домен 2-го рівня + мінімум 200 об'єктів в активній базі
- Без виконання умов — XML-фід відхиляється модератором
- Нативна інтеграція: **LigaPro** (підтверджена), **RealtSoft** (через XML/API), **G-PLUS** (частково)
- Міжнародні CRM: ❌ немає нативних інтеграцій

### DOM.RIA (DIM.RIA)
- Офіційний API-канал
- Нативна: **G-PLUS CRM**, **LigaPro**, **RealtSoft**, **Plektan** (DIM.RIA)
- Міжнародні CRM: через Zapier/Make/ApiX-Drive

### OLX
- REST API; найширша підтримка
- Нативна: **KeyCRM** (чати + замовлення), **KeepinCRM** (чати), **LigaPro**, **Plektan**, **RealtSoft**, **SalesDrive**
- Важлива деталь: обов'язкові поля (наприклад, площа кухні) — помилка у заповненні → весь фід відхиляється OLX

### Flatfy, Rieltor.ua
- Переважно XML-фіди; нативних CRM-інтеграцій майже немає
- Реалізується через XML-експорт з нішевих CRM або ApiX-Drive

## Підводні камені

### ⚠️ Санкційний ризик (критично для UA ринку 2025–2026)

- **Bitrix24, amoCRM, Kommo** — російське походження. Ризик: блокування акаунту, заборона передачі персональних даних UA-громадян на RU-сервери, репутаційні втрати
- **Profitbase (profitbase.ru)** — російська компанія. Шахматка якісна, але використання в Україні несе ризики. Альтернатива: G-PLUS CRM або Flatris
- **INTRUM (intrumnet.com)** — російська компанія. Не рекомендується для UA ринку
- Рекомендація: перевіряти країну реєстрації вендора перед підписанням договору

### ⚠️ LUN.ua XML-фід

- Вимагає власного домену 2-го рівня + мінімум 200 активних об'єктів
- Без цього — весь фід відхиляється автоматично
- Малі агентства (до 30 об'єктів) фактично не можуть інтегруватися з LUN.ua

### ⚠️ KeyCRM приховані витрати

- Базовий тариф $19/міс містить: лише 2 000 карток у воронках та 20 000 повідомлень
- Кожен наступний блок — $9/місяць
- Підключення Viber/Telegram/WhatsApp-номерів — окрема плата за кожен
- Для великої команди реальна вартість може бути вдвічі вищою за заявлену

### ⚠️ Відсутність бази об'єктів у міжнародних CRM

- Kommo, Pipedrive, HubSpot, Zoho — **немає окремої сутності "Об'єкт нерухомості"** з коробки (виняток: HubSpot Listings Object з 2024)
- Pipedrive: архітектурна проблема — немає linked custom objects взагалі (підтверджено Community)
- Агентство змушене вести базу об'єктів в Google Sheets / Airtable зовні CRM

### ⚠️ Vendor lock-in у девелоперських CRM

- Шахматка G-PLUS / CRMSTROY — специфічний формат даних, складно мігрувати до іншої системи
- Вимагайте документоване API та можливість повного CSV-експорту даних при підписанні договору

### ⚠️ OLX API нестабільність

- Інтеграції з OLX потребують регулярної переавторизації при зміні API OLX
- **KeepinCRM OLX**: потребує реавторизації кожні 30 днів — без цього чати перестають надходити
- Навіть нативні інтеграції (KeyCRM, Plektan) можуть тимчасово ламатися при оновленнях з боку OLX
- Завжди перевіряйте наявність активної підтримки від вендора

### ⚠️ Дата знаку на документах

- CRM може згенерувати PDF договору, але ДКП нерухомості потребує нотаріуса
- E-підпис (Дія.Підпис) підходить для: попереднього договору, завдатку, актів приймання-передачі, внутрішніх документів
- Для ДКП — тільки нотаріальне посвідчення (незалежно від системи)

## Підводні камені при виборі типу CRM

### Профільні нішеві (LigaPro / Plektan / RealtSoft)
- ✅ Ідеальні для класичної рієлторської моделі (вторинка, оренда, первинка через агентство)
- ⚠️ Якщо агентство виходить на закордонні ринки або потребує розширеної AI/enterprise-аналітики — може бракувати функціоналу
- ⚠️ Інтеграції з OLX потребують реавторизації при оновленнях API

### Галузеві для забудовників (G-PLUS / CRMSTROY / RealtSoft Developer)
- ✅ Шахматка + розстрочка + синхронізація з сайтом ЖК — вирішується "з коробки"
- ⚠️ Висока складність впровадження: потребують участі вендора, інтеграції з ERP/1С, бухгалтерією
- ⚠️ Специфічний формат даних — складно мігрувати; вимагайте документоване API і CSV-експорт при підписанні договору

### Універсальні UA (KeyCRM / KeepinCRM / SalesDrive / Uspacy / NetHunt)
- ✅ Кращий омніканальний інбокс, швидкий старт
