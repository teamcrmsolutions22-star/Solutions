## ⚡ Ключове розмежування: 3 різних класи систем

| Клас | Функція | Для кого |
|------|---------|----------|
| **CRM (sales)** | Ліди, воронка продажів, документи, оплати, месенджери | Турагентства, туроператори, екскурсії, MICE |
| **PMS** (Property Management System) | Бронювання номерів, шахматка, тарифи, check-in/out, housekeeping | Готелі, хостели, апартаменти |
| **Channel Manager** | Синхронізація цін і наявності між PMS та OTA (Booking/Airbnb/Expedia) | Готелі, апартаменти |

**CRM без PMS** не керує завантаженням готелю. **PMS без CRM** не має повноцінної воронки продажів. Оптимально: **PMS + Channel Manager + CRM-шар** для готелів; **спеціалізована CRM** для турагентств.

| **B. Туроператор** | Формування турів, агентська мережа (B2B2C), блоки місць | ERP + B2B-кабінет агентів |

| **D. Апартаменти / хостел / подобова оренда** | Channel manager, кілька об'єктів, прибирання | Channel Manager + легкий PMS |
| **E. Екскурсії / MICE / DMC** | Групові тури, події, заявки, розклад | CRM + месенджери + документи |

### 📡 IT-Tour (ittour.com.ua) — Екосистема пошуку турів в Україні

> **КРИТИЧНО ДЛЯ КОНТЕКСТУ:** IT-Tour — це не CRM, а **агрегатор турів**, вбудований на ~90% сайтів українських туристичних агентств. Більшість спеціалізованих CRM для турагентств інтегруються саме з ним.

**Що таке IT-Tour:**
- **67+ українських туроператорів** в одному пошуку ✅
- **Безкоштовно** для агентств (монетизується через комісії від ТО) ✅
- ~4 млн пошукових операцій на місяць ✅
- Вбудований на сайт через iframe/API ✅
- Покриває всі популярні напрямки: Єгипет, ОАЕ, Туреччина, Греція, Болгарія та ін. ✅

**Зв'язок з CRM:** All-Inclusive CRM, CRM Travels, МоїТуристи та TurScanner CRM ТАТ мають інтеграцію або плагіни, що дозволяють імпортувати тур з IT-Tour/пошукових систем операторів безпосередньо в CRM-картку одним кліком.

## 🏨 ЧАСТИНА 2: PMS + Channel Manager для готелів

### Правило вибору: PMS vs Channel Manager vs All-in-One

```
Маленький готель (до 20 номерів):
→ All-in-One: Little Hotelier або KWHotel Standard
→ Якщо UA ПРРО обов'язково: EasyMS (українська) або Servio HMS

Середній готель (20-100 номерів, UA ринок):
→ Servio HMS + YieldPlanet (золота комбінація для UA, ПРРО)

→ Beds24 або Smoobu (бюджетно, CloudBeds (потужно)
→ або Servio HMS + YieldPlanet (якщо UA ПРРО критична)

Санаторій/велика база (ПРРО критична):
→ Servio HMS + YieldPlanet + Vchasno.Kasa
```

## ⚠️ Підводні камені

### 1. Bnovo тепер є частиною 1C-групи (квітень 2024)
Smartway (51% акцій якого належить 1C) купив Bnovo за ~1 млрд рублів. 1C заборонено в Україні з 09.01.2026. **Готелям, що ще використовують Bnovo, рекомендується негайна міграція.**

### 2. OtelMS — не повноцінне UA-рішення
Незважаючи на українську мову інтерфейсу, ChatGPT Deep Research (2026-06) вказує: сервери і розробка залишаються в Ставрополі (РФ). Відсутні: UA ПРРО, UA-платежі (тільки Stripe/PayPal), UA SMS-шлюзи. Для критичних UA-потреб → Servio HMS.

### 3. МоїТуристи (moituristy.com) — перевірте перед рекомендацією
ChatGPT Deep Research вказує на потенційний зв'язок з РФ-бенефіціаром (Ельдар Костоєв / TourDom.ru). До підтвердження або спростування цього факту — розглядайте All-Inclusive CRM або TurScanner CRM ТАТ як безпечніші альтернативи.

### 4. TravelLine — 70% ринку РФ
Найбільший hotel channel manager в РФ (~12 000 клієнтів). Широко використовувався UA готелями до 2022. **Зараз — пряма загроза.**

### 5. CRM ≠ PMS для готелів
Використання лише CRM (KeyCRM, Kommo) без PMS не вирішує проблему управління завантаженням, тарифами і OTA-синхронізацією. Готелям потрібні обидва класи систем.

### 6. ПРРО — критично для UA готелів
Більшість міжнародних PMS (Cloudbeds, Little Hotelier, KWHotel, Beds24, Smoobu) не мають нативної підтримки ПРРО. Для UA ринку єдине готове рішення — **Servio HMS + Vchasno.Kasa**. Або — зовнішній фіскальний реєстратор.

### 7. Пошук турів: лише спеціалізовані системи мають нативний пошук
Загальні CRM не мають нативного пошуку турів. Лише **All-Inclusive CRM**, **МоїТуристи**, **TurScanner CRM ТАТ** та **CRM Travels** мають цю функцію. Агентства на загальних CRM шукають тур через IT-Tour/сайти операторів вручну і вносять в CRM.

### 8. IT-Tour як обов'язковий контекст
IT-Tour (ittour.com.ua) — це безкоштовний агрегатор 67+ UA операторів, вбудований на ~90% UA туристичних сайтів. Спеціалізовані CRM інтегруються з IT-Tour або подібними пошуковими системами. Без розуміння IT-Tour-екосистеми рекомендації для ТА будуть неповними.

### 9. Servio HMS / KWHotel — «не підтверджено» у міжнародних звітах

Perplexity Deep Research (червень 2026) позначає Servio HMS і KWHotel як «не підтверджено» по всіх функціях — через брак публічних англомовних специфікацій. **Це артефакт джерела, а не відсутність функцій.** Servio HMS верифіковано через:
- expertsolution.com.ua — офіційний дистриб'ютор: ПРРО (Vchasno.Kasa), 130+ звітів, конференц-модуль
- ribashotelsgroup.ua — Ribas Hotels Group (28+ UA готелів) публічно описує перехід на Servio
- Контракт + демо верифікують конкретні фічі під UA-специфіку

KWHotel верифіковано через kwhotel.com: 21 мова (UA включно), від €8/міс Cloud, модуль Channel Manager.

**Правило:** для UA-систем — надавайте пріоритет UA-джерелам (expertsolution.com.ua, ribashotelsgroup.ua) над міжнародними агрегаторами, які не покривають локальний ринок.

### 10. Frontdesk24 — РОСІЙСЬКА
Деякі вважали Frontdesk24 нейтральним продуктом — це неправда. Frontdesk24 об'єдналася зі Shelter PMS (Moscow). Обидві — РФ-компанії.

### Блок: Прямі бронювання vs OTA — економіка комісій

> **Ключовий фінансовий аргумент** для впровадження booking engine і метапошуку.

#### Реальні втрати на комісіях OTA

| OTA | Комісія | Середнє за 2025 |
|-----|---------|----------------|
| Booking.com | 15–25% від вартості бронювання | ~18% |
| Airbnb (host fee) | 3–5% + guest fee 14% | ~18% total |
| Expedia | 15–30% | ~22% |

**Підраховано (2025 дані):** Пряме бронювання в середньому $516 vs OTA $312 — тобто пряме бронювання на $204 доходніше per booking. (Джерело: SiteMinder 2025 research)

#### Як зменшити залежність від OTA

```
Booking Engine на сайті → 0% комісії OTA
Google Hotel Ads (метапошук) → платиш лише за кліки або % від бронювання (commission model)
Прямий email-маркетинг → повторні гості без OTA
Програма лояльності → преференції при прямому бронюванні
```

### Підводні камені (доповнення)

#### 10. Mews — «ціна питання» прихована

Mews не публікує ціни. Реальний стек для 20-номерного готелю: base PMS (€300+) + channel manager через marketplace (€50-100) + Atomize RMS (€100+) + premium support = **€500-700+/міс** з 2-річним зобов'язанням. Малим готелям краще підходить Sirvoy, KWHotel або Little Hotelier.

#### 11. Google Hotel Ads — нова обов'язкова точка дотику (2026)

З 2026 Google AI Mode почав показувати готелі безпосередньо в пошуку. Готелі без booking engine і Google Hotel Ads **втрачають прямі бронювання** на рівні пошукової видачі. SiteMinder Demand Plus — найпростіше рішення (commission model, без абонплати).

#### 12. Всі міжнародні PMS без ПРРО — але вихід є

ПРРО обов'язковий для всіх з 01.01.2024. Для готелів на Cloudbeds, Mews, Little Hotelier та ін. — потрібна окрема каса (наприклад, Checkbox або Vchasno.Kasa). Servio HMS + Vchasno.Kasa = єдина нативна комбінація.

#### 13. OtelMS — увага на юрисдикцію

ChatGPT Deep Research (2026-06) підтверджує: попри заявлений вихід з РФ-ринку, сервери та офіс розробки OtelMS залишаються в Ставрополі (РФ). Офіційна юрисдикція — Кіпр. Підтримує тільки Stripe/PayPal (без LiqPay/WayForPay). Для UA ПРРО — не підходить.
