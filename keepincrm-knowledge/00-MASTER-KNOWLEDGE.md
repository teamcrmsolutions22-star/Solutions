## 1. ЗАГАЛЬНИЙ ОГЛЯД СИСТЕМИ

**KeepinCRM** — українська CRM-система для малого та середнього бізнесу.  
Сайт: https://keepincrm.com/

### Тарифи
- **Безкоштовний:** 1 користувач, базові функції, без автоматизації
- **Платний:** ~350 грн/місяць за 1 користувача (є знижки на квартал/рік)
- **Тестовий період:** 10 днів
- На платному тарифі кількість користувачів необмежена (фіксована ціна)
- Автоматизація (тригери) — **тільки на платному тарифі**

### Що входить до системи (єдиний простір)
- Ліди та клієнти (контрагенти)
- Угоди / замовлення з воронками продажів
- Завдання та планування
- Фінансовий облік (доходи, витрати, PNL, cashflow)
- Документообіг (рахунки, акти, договори за шаблонами)
- Складський облік та каталог товарів
- ТТН — Нова Пошта, Укрпошта, Justin, Meest
- Чати — Instagram, Facebook, Viber, Telegram, WhatsApp
- Email інтеграція
- Аналітика та звіти
- Автоматизація (тригери)
- API та Webhook для інтеграцій
- Програма лояльності (через myBot)

### Дашборд
Показує: нові клієнти/ліди/угоди, джерела трафіку, топ-товари, статистика менеджерів, конверсія воронки, грошовий потік, середній чек, час активності клієнтів.

### Гарячі клавіші
`Alt+Shift+[клавіша]` — швидке створення угоди, ліда, завдання.

## 2. ЛІДИ ТА КЛІЄНТИ

### Різниця лід/клієнт
- **Лід** — потенційний замовник, ще не купив
- **Клієнт** — вже здійснив покупку або укладено угоду
- Контрагент не може бути одночасно лідом і клієнтом
- Лід конвертується в клієнта автоматично або вручну

### Коли використовувати ліда
- При роботі з чатами (Instagram, Telegram, Viber) — з нового чату можна створити тільки ліда
- При роботі з маркетплейсами (Rozetka, Prom) — нові замовлення потрапляють в ліди
- При лідогенерації (лендінги, холодні дзвінки)

### Перевірка дублів
Перед створенням контрагента — обов'язково перевірити за **телефоном або email** (унікальні ідентифікатори). CRM підсвічує дублікатів.

### Відображення
- **Канбан** — по статусах
- **Список** — таблиця

### Структура картки контрагента
- Назва компанії / ім'я
- Контактна особа (та додаткові контакти)
- Телефон, Email
- Статус (лід / клієнт / VIP / тощо)
- Джерело
- Відповідальний менеджер
- Теги
- Блок «Угоди» — всі угоди
- Блок «Завдання» — з усіх угод
- Блок «Фінанси» — зведення по всіх угодах
- Вкладка «Чати» — вся комунікація
- Журнал Активності — всі дії

### Масові дії
- Зміна відповідального
- Зміна тегів та статусу
- Архівування

### Імпорт/Експорт
- CSV/Excel (двостороннє)

## 3. УГОДИ ТА ВОРОНКИ ПРОДАЖІВ

### Воронка продажів
Логічний шлях угоди від першого контакту до результату. Налаштування: **Налаштування → Типи даних → Воронка продажів**.

- Необмежена кількість воронок (на платному тарифі)
- Кожна воронка — власні кольорові етапи
- Для кожного етапу — бажаний максимальний термін перебування
- Фінальні статуси: **Виграно** / **Програно**

### Відображення угод
- **Канбан** — drag & drop між етапами
- **Список** — таблиця з налаштовуваними колонками
- **Архів** — завершені угоди

### Візуальні індикатори
- **Жовте підсвічення** — угода ще не переглядалась
- **Червона смуга + годинник** — угода довго без руху (перевищено термін)
- **«Важлива»** — ручне позначення
- **«Непереглянута»** — ручна позначка через 3 крапки

### Структура картки угоди

**Верхній блок:**
- Клієнт (перехід в картку)
- Назва угоди
- Кнопки «Виграна» / «Програна»
- Панель етапів поточної воронки
- Зміна воронки (через 3 крапки)

**Ліва частина — поля:**
- Контактні дані клієнта
- Кастомні поля (відповідно до бізнес-процесу)

**Блоки:**
- **Товари** — каталог, кількість, ціна, списання
- **Доставка** — ТТН (НП, Укрпошта, Justin, Meest)
- **Завдання** — автоматичні або вручну
- **Фінанси** — оплати, заборгованість
- **Документи** — генерація з шаблонів
- **Файли** — завантаження з комп'ютера
- **Додаткові продажі** — окремий блок

**Права частина — Журнал Активності:**
- Хронологія всіх дій
- Нотатки з файлами
- Email відправка
- Переписка в чаті
- Фільтрація журналу

### Контроль якості через фільтри
- Угоди без завдань
- Протерміновані завдання
- Угоди що зависли на етапі
- Непереглянуті угоди

## 5. ФІНАНСИ

### В картці угоди (вкладка Фінанси)

### В картці клієнта (вкладка Фінанси)
- Зведення по всіх угодах клієнта

### Глобальний розділ «Фінанси»
- Доходи та витрати по гаманцях
- Банківські транзакції (після підключення monobank/ПриватБанк)

## 6. ДОКУМЕНТИ

### Блок «Документи» в угоді
- Генерація з шаблонів (рахунки, акти, договори, накладні)
- Автозаповнення даних компанії, клієнта, товарів
- Перегляд, редагування, скачування, надсилання Email
- Статуси документів

### Шаблони документів
Налаштування: **Налаштування → Шаблони документів**

**Змінні:**
```
{{client.name}}, {{client.phone}}, {{client.email}}
{{deal.title}}, {{deal.total}}, {{deal.date}}
{{company.name}}, {{company.address}}, {{company.edrpou}}
{{products}} — таблиця товарів
```

### ТТН

**Підтримувані сервіси:** Нова Пошта, Укрпошта, Justin, Meest

**Нова Пошта — налаштування:**
- Метод доставки: відділення / адресна / поштомат
- Платник за доставку: відправник / одержувач
- Опис вантажу, тип, форма оплати
- Підв'язка статусів НП до етапів воронки
- Кілька акаунтів НП одночасно

**Процес створення ТТН:**
1. Угода → Блок «Доставка» → «+»
2. Дані відправника — з налаштувань інтеграції
3. Дані отримувача — з картки клієнта
4. Заповнити: відділення, вага, оголошена вартість
5. Зберегти → ТТН створюється в кабінеті НП
6. Відправити SMS клієнту з номером ТТН

## 8. АВТОМАТИЗАЦІЯ (ТРИГЕРИ)

> ⚠️ Тільки на платному тарифі

Налаштування: **Налаштування → Автоматизація → Тригери**

### Розділи для тригерів
Ліди, Клієнти, Угоди, Завдання, Доставки, Дзвінки, Фінанси

### Умови тригерів

**По етапу воронки:**
```
stage.name == 'Назва етапу'
stage.name == 'Узгодити' and funnel.title == 'Продажі'
```

**По полю:**
```
field.назва_поля == 'значення'
```

### Дії тригерів
- Створити завдання, призначити відповідального, авторозподіл
- Перемістити на інший етап / у іншу воронку
- Завершити угоду (виграно/програно) / архівувати клієнта
- Скопіювати/заповнити поле
- Надіслати SMS/Email клієнту
- Надіслати повідомлення в месенджер
- Згенерувати документ
- Провести фінансову операцію
- Створити ТТН
- Створити нову угоду (для допродажів)
- Вихідний Webhook (відправити дані на зовнішній URL)

### Приклади тригерів
1. Нова угода → Завдання «Зателефонувати»
2. Заповнено ТТН → SMS клієнту з трекінгом
3. Оплата повна → Перейти на «Відправлено»
4. Rozetka/Prom нове замовлення → Авторозподіл + завдання
5. Перехід на «Відправка» → SMS клієнту (ім'я + ТТН)
6. Статус НП «Доставлено» → Угода «Виграно»

## 9. API

**Base URL:** `https://api.keepincrm.com/v1/`  
**Автентифікація:** Header `X-Auth-Token: {api_key}`  
**Формат:** JSON  

**API ключ:** Налаштування → Профіль компанії → API (тільки для власника)

### Основні ендпоінти

| Метод | URL | Дія |
|-------|-----|-----|
| POST | /v1/agreements | Створити угоду |
| GET | /v1/agreements | Список угод |
| GET | /v1/agreements/{id} | Деталі угоди |
| PUT | /v1/agreements/{id} | Оновити угоду |
| POST | /v1/counterparties | Створити контрагента |
| GET | /v1/counterparties | Список контрагентів |

### Приклад PHP — створення угоди
```php
$data = [
    'title' => 'Нове замовлення',
    'stage_id' => 1,       // ID першого етапу воронки
    'source_id' => 2,      // ID джерела
    'client' => [
        'name' => 'Іван Іванов',
        'phone' => '+380501234567',
        'email' => 'ivan@example.com',
        'lead' => true
    ],
    'products' => [
        [
            'sku' => 'SKU-001',
            'title' => 'Товар 1',
            'price' => 1200,
            'amount' => 2,
            'currency' => 'UAH'
        ]
    ]
];

$ch = curl_init('https://api.keepincrm.com/v1/agreements');
curl_setopt_array($ch, [
    CURLOPT_POST => true,
    CURLOPT_POSTFIELDS => json_encode($data),
    CURLOPT_HTTPHEADER => [
        'Content-Type: application/json',
        'X-Auth-Token: ' . $apiKey
    ],
    CURLOPT_RETURNTRANSFER => true
]);
$response = json_decode(curl_exec($ch), true);
```

**Документація:** https://keepincrm.com/category/api  
**PHP приклади:** https://keepincrm.com/php-example  
**Swagger:** в системі після авторизації (розділ API)

## 10. WEBHOOK

### Типи
- **Вхідний (Inbound):** Зовнішній сервіс → KeepinCRM
- **Вихідний (Outbound):** KeepinCRM → Зовнішній сервіс

Налаштування: **Налаштування → Інтеграції → Webhook**

### Вхідний Webhook (прийом лідів з форм)

**Підтримувані джерела:**
- WordPress Contact Form 7, WooCommerce
- Facebook Lead Ads
- Weblium, Wix форми
- Viber через TurboSMS
- Google Forms
- Будь-який HTTP сервіс

**Процес:**
1. Створити вхідний webhook → отримати унікальний URL
2. Вставити URL в зовнішній сервіс
3. Налаштувати маппінг полів (JSONPath → поле CRM)

**Визначення JSONPath:**
Сервіс: https://www.site24x7.com/tools/jsonpath-finder-validator.html

### Вихідний Webhook

- При будь-якій події в системі → відправка JSON на зовнішній URL
- Налаштовується через Тригери або напряму
- Документація для угод: https://keepincrm.com/send-from-deals

1. Налаштування → Інтеграції → Маркетплейс Prom → «+»
2. Ввести API-токен (Prom → Налаштування → Управління API-токенами)
3. Обрати джерело (попередньо створити в Управління → Джерела)
4. Зв'язати товари: по артикулу або по Prom ID

### Синхронізація товарів
- Генерація **XLS-файлу** для імпорту на Prom
- Поля XLS: артикул, ціна, опис, наявність/формула, залишки, назва, одиниця, Prom ID
- **Формула наявності** — поле-формула, повертає «+» (є) або «–» (немає)
- Для **різновидів** — кастомні поля: «ID групи різновидів» + «Значення характеристики»
- Автооновлення на Prom — **раз на 4 години**
- Прайс-лист треба зв'язати зі складом (по назві або артикулу)

### Підв'язка статусів Prom → KeepinCRM
- Кожен статус Prom → відповідний етап воронки

### Автоматична відправка ТТН на Prom
При переході угоди на певний етап → автоматично надсилається ТТН на Prom

## 15. ІНТЕГРАЦІЯ ROZETKA — ДЕТАЛЬНО (з відеоматеріалів)

### Підключення
1. Налаштування → Інтеграції → Маркетплейс Розетка → «+»
2. Назва → Логін/Пароль від кабінету Розетки

### Дві частини інтеграції
**Частина 1 — Прийом замовлень:**
- Замовлення надходять в CRM
- Передача статусів
- Передача ТТН в Rozetka

**Частина 2 — Передача товарів:**
- Через XML-файл
- Поля: назва UA/RU, опис UA/RU, бренд, залишки, ціна, стара ціна, промо-ціна, URL

### Особливості
- Запитання по товарах → можна перенаправити в «Завдання»
- Автоматична зміна статусу «Новий» → «Обробляється» з затримкою (покращення статистики)
- Синхронізація товарів Rozetka → CRM: описи та характеристики (без залишків!)
- Підв'язка статусів Rozetka → внутрішні етапи воронки

## 16. ІНТЕГРАЦІЯ MONOBANK — ДЕТАЛЬНО (з відеоматеріалів)

## 17. ІНТЕГРАЦІЯ CHECKBOX — ДЕТАЛЬНО (з відеоматеріалів)

### Підключення
1. Налаштування → Інтеграції → Checkbox → «+»
2. Ввести дані касира: PIN-код та ключ ліцензії каси
3. Дані підтягуються з Checkbox / податкової
4. Надати доступ менеджерам (окремо для кожного)

### Фіскалізація чека
1. Картка угоди → «Документи» → обрати документ
2. Вибрати касира → зберегти
3. Чек фіскалізується автоматично
4. Поля: назва товару (обов'язково **українською**), тип оплати (готівка/безготівкова)

### Порівняння ПРРО
- **Checkbox:** краще для multichannel з CRM, 149 грн/каса/міс
- **Вчасно.Каса:** краще для Prom/Rozetka, 150 грн/каса/міс (Prom компенсує бонусами)

## 18. ПРОГРАМА ЛОЯЛЬНОСТІ myBot — ДЕТАЛЬНО
