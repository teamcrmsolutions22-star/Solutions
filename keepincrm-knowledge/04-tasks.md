# KeepinCRM — Завдання

> Документація: https://keepincrm.com/tasks-description  
> Автоматичні завдання: https://keepincrm.com/create-tasks

---

## Призначення завдань

Завдання в KeepinCRM — інструмент планування роботи менеджерів. Кожне завдання прив'язується до конкретного контрагента або угоди.

---

## Структура завдання

| Поле | Опис |
|------|------|
| Назва | Що потрібно зробити |
| Тип | Дзвінок, зустріч, email, інше |
| Дата і час | Дедлайн або запланований час |
| Відповідальний | Хто виконує |
| Пріоритет | Висока/середня/низька |
| Опис | Деталі завдання |
| Контрагент | Прив'язка до клієнта/ліда |
| Угода | Прив'язка до замовлення |
| Статус | Нове / В роботі / Виконано |

---

## Типи завдань

- Дзвінок (Call)
- Email
- Зустріч (Meeting)
- Задача (Task)
- Нагадування

---

## Перегляди завдань

- **Список** — таблиця всіх завдань
- **Календар** — за днями/тижнями/місяцями
- **Мої завдання** — тільки завдання поточного менеджера
- **По угодах** — в контексті кожної угоди

---

## Фільтри завдань

- По відповідальному
- По типу
- По пріоритету
- По статусу
- По даті
- По угоді/контрагенту

---

## Автоматичне створення завдань

Документація: https://keepincrm.com/create-tasks

Через тригери можна автоматично:
- Створювати завдання при переміщенні угоди на певний етап
- Призначати відповідального
- Встановлювати дедлайн (наприклад, +1 день від дати створення)
- Використовувати авторозподіл між менеджерами

---

## Нотатки (Notes)

Окрім завдань є система нотаток (хронологічних записів):
- Прив'язуються до картки контрагента або угоди
- Відображаються в хронологічному порядку
- Можна прикріплювати файли
- Є згадки (@менеджер)
