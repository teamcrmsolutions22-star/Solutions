# KeepinCRM — Угоди та Воронки продажів

> Документація: https://keepincrm.com/work-with-deals  
> Воронки: https://keepincrm.com/deal-funnel  
> Категорія: https://keepincrm.com/category/agreements

---

## Концепція угод

**Угода** — це замовлення або проект, пов'язаний з клієнтом/лідом. Угоди рухаються по **воронці продажів** через **етапи**.

При реєстрації акаунту автоматично створюється 1 воронка зі стандартними етапами.

---

## Налаштування воронок

**Шлях:** Налаштування → Типи даних → Угоди

- Можна створити **необмежену кількість воронок**
- Кожна воронка має власні **етапи**
- Етапи мають колір та назву
- Можна налаштувати **фінальні статуси** (Виграно / Програно)

### Приклади воронок
- Продажі (основна)
- Оптові замовлення
- Сервіс / Ремонт
- Постачання

---

## Картка угоди

### Основна інформація
- **Назва угоди** / номер
- **Контрагент** (клієнт або лід)
- **Відповідальний** менеджер
- **Воронка та етап**
- **Дата** створення, оновлення
- **Сума** угоди
- **Джерело**
- **Теги**

### Вкладки картки угоди
- **Товари** — перелік товарів/послуг в угоді
- **Фінанси** — оплати, заборгованість, аванс
- **Доставка** — ТТН, статус доставки
- **Документи** — згенеровані рахунки, акти
- **Завдання** — завдання по цій угоді
- **Нотатки** — хронологія
- **Чати** — листування з клієнтом

---

## Товари в угоді

- Додавання товарів з каталогу або вручну
- Кількість, ціна, знижка, сума
- Вибір типу ціни (роздріб, опт, тощо)
- Автоматичне списання зі складу при підтвердженні
- Бронювання товарів

## Воронка продажів — робота з угодами

### Переміщення угод
- **Вручну** — drag & drop або зміна поля "Етап"
- **Автоматично** — через тригери

### Фільтри в угодах
- По воронці
- По етапу
- По відповідальному
- По даті
- По сумі
- По джерелу
- По тегах
- По кастомних полях

### Відображення
- **Канбан** (по етапах)
- **Список/таблиця**
- **Контроль угод** — спеціальний вид для менеджера (що потребує уваги)

---

## Контроль угод

Спеціальний розділ, що показує угоди які:
- Довго без активності
- Прострочені завдання
- Без відповідального
- Без контакту

Документація: https://keepincrm.com/category/agreements

---

## Кастомні поля для угод

Налаштування: **Налаштування → Типи даних → Угоди → Поля**

Типи полів:
- Текст, число, дата
- Список (dropdown)
- Прапорець
- Посилання
- Файл

## API для угод

### Створення угоди (POST)
```
POST https://api.keepincrm.com/v1/agreements
X-Auth-Token: {ваш_api_ключ}
Content-Type: application/json
```

Приклад (PHP):
```php
$data = [
    'title' => 'Назва угоди',
    'stage_id' => 1,        // ID етапу воронки
    'source_id' => 1,       // ID джерела
    'client' => [
        'name' => 'Ім\'я клієнта',
        'email' => 'client@example.com',
        'phone' => '+380501234567',
        'lead' => true
    ],
    'products' => [
        [
            'sku' => 'ART-001',
            'title' => 'Назва товару',
            'price' => 100,
            'amount' => 2,
            'currency' => 'UAH'
        ]
    ]
];

$ch = curl_init('https://api.keepincrm.com/v1/agreements');
curl_setopt($ch, CURLOPT_POST, 1);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($data));
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'Content-Type: application/json',
    'X-Auth-Token: ' . $apiKey
]);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
$response = curl_exec($ch);
```

Повна документація PHP прикладів: https://keepincrm.com/php-example  
Swagger API: через розділ API в системі

## Тригери для угод (автоматизація)

### Умови тригерів (приклади)
```
stage.name == 'Виграно'
stage.name == 'Узгодити документи' and funnel.title == 'Продажі'
```

### Доступні дії тригерів для угод
- Створити завдання
- Змінити відповідального
- Перемістити в іншу воронку/етап
- Надіслати SMS/Email
- Згенерувати документ
- Провести фінансову операцію
- Створити ТТН
- Скопіювати/заповнити поле

Детальна документація: https://keepincrm.com/main-triggers
