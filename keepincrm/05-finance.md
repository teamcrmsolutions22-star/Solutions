## Блок Фінансів

### Що входить до блоку Фінансів

- **Доходи** — оплати від клієнтів по угодах
- **Витрати** — витрати компанії
- **Банківські транзакції** — після підключення monobank/MonoPay
- **Управлінські звіти** — аналіз чистого доходу

---

## Контроль оплат по угодах

В кожній угоді у вкладці **Фінанси**:

| Показник | Опис |
|----------|------|
| Сума угоди | Загальна сума |
| Аванс | Передоплата |
| Оплачено | Отримані кошти |
| Заборгованість | Сума до сплати |

### Проведення оплати
1. Відкрити угоду → вкладка Фінанси
2. Додати оплату (дата, сума, рахунок)
3. Система автоматично розраховує залишок

---

## Банківські транзакції (monobank)

Після інтеграції з monobank:
- З'являється вкладка "Банківські транзакції"
- Відображаються всі операції по картах/рахунках
- Можна прив'язати транзакцію до угоди
- Автоматичне відображення оплат

## Документи (Рахунки, Акти, Договори)

Документація: https://keepincrm.com/work-with-documents

### Шаблони документів
Налаштовуються в: **Налаштування → Шаблони документів**

- Рахунок на оплату
- Акт виконаних робіт
- Договір
- Накладна
- Будь-який кастомний документ

### Змінні в шаблонах

Шаблони підтримують автопідстановку даних:
```
{{client.name}} — ім'я клієнта
{{client.phone}} — телефон
{{deal.title}} — назва угоди
{{deal.total}} — сума угоди
{{deal.date}} — дата угоди
{{company.name}} — назва компанії
{{company.address}} — адреса компанії
```

### Генерація документів
1. Відкрити угоду → вкладка Документи
2. Вибрати шаблон
3. Система автоматично підставить дані клієнта та угоди
4. Зберегти або надіслати клієнту

### Автоматична генерація через тригери
При переміщенні на певний етап → автоматично генерується документ.

## ТТН (Транспортна Накладна)

### Підтримувані служби доставки
- **Нова Пошта** — https://keepincrm.com/integratsiya-z-servisom-nova-poshta
- **УкрПошта** — https://keepincrm.com/ukrposhta-integration
- **Justin** — https://keepincrm.com/justin-integration
- **Meest** — https://keepincrm.com/meest-integration

### Створення ТТН (Нова Пошта)
1. Відкрити угоду → вкладка Доставка
2. Натиснути "Створити ТТН"
3. Система підтягне дані клієнта (місто, відділення, ПІБ)
4. Вказати: вага, оголошена вартість, спосіб оплати доставки
5. Підтвердити — ТТН створиться в кабінеті НП
6. Роздрукувати або відправити SMS клієнту

### Автоматизація ТТН
Через тригери:
- Автоматичне надсилання SMS з номером ТТН при його створенні
- При зміні статусу посилки — зміна статусу угоди
- Відправка ТТН в Rozetka при відправленні

## Створення ТТН (відео-інструкція з бази знань)

З завантаженого матеріалу (KeepinCRM_створення_ТТН.md):

### Поля при створенні ТТН в KeepinCRM:
- Метод доставки: відділення / поштомат / кур'єр
- Платник за доставку: відправник / одержувач
- Опис вантажу (наприклад: "одяг", "взуття", "електроніка")
- Відділення відправки (з акаунту НП)
- Вид вантажу: посилка / вантаж
- Платник комісії за післяплату: відправник / одержувач
- Форма оплати: готівка / безготівкова

> Дані міста та відділення одержувача підтягуються автоматично з картки клієнта
