## 1. Огляд системи

**Тип:** Комплексна ERP/CRM для малого та середнього бізнесу України. Розробляється з 2017 року.
**Юзери:** 170 000+ на місяць (в одних матеріалах) / 200 000+ (у FAQ сайту) — ⚠️ маркетингова невідповідність, вимагає верифікації.
**Мови:** 16, включно з українською.
**Документація:** 250+ технічних матеріалів.
**Моделі розгортання:** Хмара платна, Хмара безкоштовна (без ліміту по строку), Коробка (Self-Hosted).

### Конфігурації (версії)

| Версія | Призначення |
|--------|-------------|
| **PROJECT** | Завдання, Kanban, Ганта, тайм-трекінг |

## 3. REST API

### Основні параметри

| Параметр | Значення |
|----------|----------|
| Тип | REST API |
| Версія | 2.5 (актуальна) |
| Формат | JSON |
| Auth | Статичний API Key (Bearer / X-Api-Key) |
| Base URL (коробка) | `https://your_domain/api/` |
| Base URL (хмара) | `https://your_subdomain.perfectum.ua/api/` |
| Пагінація | `?limit=N&offset=N` або `?page=N` |
| Формат дат | `Y-m-d` (напр. `2026-05-28`) |
| Документація | Вбудована в UI: Налаштування → Системні → Ключі API |

### Rate Limits

| Обмеження | Значення |
|-----------|----------|
| Мікро-ліміт (хмара) | 20 запитів / 3 секунди |
| Добовий ліміт (хмара) | **200 запитів × кількість юзерів** (напр. CRM-50 = 10 000/добу) |
| Коробка | Індивідуальне налаштування на кожен API-ключ |

**⚠️ Критично для інтеграцій:** при активних сценаріях хмарний добовий ліміт вичерпується швидко. Рішення — коробкова версія або кешування даних на стороні інтеграції.

### Генерація API-ключа

Шлях: *Налаштування → Системні → Ключі API*

Налаштування ключа:
- Білий список IP-адрес
- Ліміти та інтервали запитів
- XSS-валідація
- Права доступу: read/write по кожному модулю окремо
- Термін дії: необмежений (до ручного видалення)

### Ключові ендпоінти

```
# Клієнти
GET    /api/clients             — список (пагінація, фільтрація)
POST   /api/clients             — створити
PUT    /api/clients/{id}        — оновити
DELETE /api/clients/{id}        — видалити

# Замовлення
GET    /api/orders              — список
POST   /api/orders              — створити (специфікація, доставка, сума)
PUT    /api/orders/{id}         — змінити статус / поля

# Товари
GET    /api/items               — номенклатура, залишки, категорії
POST   /api/items               — створити (SKU, ціна, категорія)

# Рахунки та оплати
GET    /api/invoices
POST   /api/invoices
POST   /api/payments            — зафіксувати оплату за рахунком

# Складські операції
# Прихід, переміщення — через складський модуль API

# Кастомні поля
# Передаються у масиві custom_fields у стандартних CRUD-запитах
```

Підтримувані сутності для кастомних полів через API: Клієнти, Контакти, Реквізити, Ліди, Проекти, Склад, Кандидати, Завдання, Підтримка, Рахунки.

**⚠️ Не задокументовано:** завантаження файлів/вкладень через API; отримання списку кастомних полів через API.

### HTTP-коди помилок

| Код | Причина |
|-----|---------|
| 400 | Некоректна структура; помилка полів `staff_assignee`, `rel_id`, `rel_type` |
| 401/403 | Невірний ключ; IP не в білому списку; відсутні права на модуль |
| 422 | Невірний формат дати або відсутні обов'язкові поля |
| 429 | Перевищено мікро-ліміт або добовий ліміт |

## 4. Вебхуки

**Модуль:** Webhooks (нативний)
**Шлях налаштування:** *Налаштування → Додатки → Веб-хуки → Налаштування Веб-Хуків*
**Лог:** *Журнал операцій веб-хуків*

| Параметр | Значення |
|----------|----------|
| Доступні модулі | Склад (Товари), Замовлення, Клієнти, Контакти |
| Події | Створення, Зміна, Видалення |
| Методи | POST, PUT, PATCH, GET, DELETE |
| Формати | `json` або `form-data` |
| Headers | Необмежена кількість кастомних (ключ-значення) |
| Динамічний URL | Шорткод `{{id}}` замінюється на ID сутності |
| Безпека | Статичний токен у кастомних Headers |
| HMAC-підпис | **Не задокументовано ⚠️** |
| Retry при помилці | **Не задокументовано ⚠️** (вважається one-shot) |

### Приклад payload (Склад / Товар):

```json
{
  "module": "warehouse",
  "event": "update",
  "staff_id": "12",
  "item_id": "450",
  "sku": "ART-9988",
  "name": "Назва товару",
  "purchase_price": "1200.00",
  "sale_price": "1800.00",
  "warehouse_id": "2",
  "quantity_minimum": "5",
  "weight": "1.50"
}
```

## 7. Автоматизація

### Конструктор бізнес-процесів

**Тригери:** Створення / Зміна / Видалення сутності  
**Умови:** Перевірка значень будь-яких системних та кастомних полів  
**Дії:**
- Моментальні: призначення відповідального, зміна статусу, створення завдання
- Відкладені: дія через N годин/хвилин (залежить від Cron)

**Журнал бізнес-процесів** — перегляд історії виконання.

**BPMN-модуль** (версія 1.6, травень 2026) — окремий модуль для візуальної побудови процесів у нотації BPMN.

### Cron (Планувальник)
Критично для: розсилки рахунків, нагадувань про дедлайни, оновлення статусів НП/УП, закриття тікетів.  
**Оптимальний інтервал:** кожні 5 хвилин.  
**Типова помилка:** Cron не налаштований → не працюють відкладені дії та оновлення доставки.

## 11. Дебаг та типові помилки

- Лог вебхуків: *Журнал операцій веб-хуків*

### Типові помилки

**401/403 Unauthorized**
- Невірний ключ, IP не в білому списку, немає прав на модуль
- Рішення: перевірити токен, очистити IP для тестів, увімкнути права модуля

**400 Bad Request при редагуванні завдань/лідів**
- Причина: невірний `staff_assignee`, `rel_id`, `rel_type`
- Рішення: передавати відповідно до документації API v2.4/2.5

**429 Too Many Requests**
- Перевищено мікро або добовий ліміт → тимчасове блокування IP
- Рішення: backoff strategy + кешування; для зняття лімітів → коробкова версія

**Бізнес-процес не спрацьовує**
- БП на паузі / не налаштований Cron / надто складні умови
- Рішення: зняти паузу, перевірити Cron, спростити тригер для тестування

**Помилка імпорту >300 записів**
- Хмарне обмеження
- Рішення: виключно "Відкладений імпорт" для файлів >300 рядків
