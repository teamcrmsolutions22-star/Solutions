# Бухучёт и налоги Украины для IT-компании по внедрению CRM (структура из двух ФОП)
**Deep Research отчёт · по состоянию на 2025–2026 · дата: 2026-06-04**

> Контекст: ФОП №1 — 3 группа ЕН, 5% без НДС (разработка, внедрение, поддержка CRM); ФОП №2 — 2 группа. Три направления: (1) разработка ПО, (2) услуги внедрения/поддержки, (3) перепродажа лицензий иностранных вендоров (Salesforce, Microsoft, HubSpot).

> ⚠️ **Дисклеймер:** это не индивидуальная налоговая консультация. Сайты-первоисточники (zakon.rada.gov.ua, tax.gov.ua) технически блокировали автозагрузку, поэтому нормы подтверждены через индексированные выдержки авторитетных изданий (ДПС-региональные, Дебет-Кредит, Factor, Liga, Бухгалтер 911, НБУ, Big-4). Ставки и спорные вопросы перед применением сверьте по тексту ПКУ и получите ІПК (индивидуальную налоговую консультацию по ст. 52 ПКУ).

## 1. Executive summary + срочные action items

1. **Самый большой риск — перепродажа лицензий через ФОП-единщика.** В зависимости от договора платёж за ПО квалифицируется как (а) услуга/товар, (б) **роялти** или (в) посредничество. Если это роялти — такой доход **в принципе не может облагаться единым налогом** (это пассивный доход физлица → НДФЛ 18% + ВЗ), а при выплате иностранному вендору возникает **налог на репатриацию 15%**. Прямого разъяснения ДПС именно по кейсу «единщик-реселлер SaaS» нет — это зона неопределённости.

2. **2 группа непригодна для корпоративных клиентов.** ФОП 2 группы вправе оказывать услуги только населению и плательщикам ЕН. Услуги юрлицу на общей системе → ставка 15% на этот доход + принудительный переход на общую систему. Для B2B CRM-проектов нужна **3 группа**.

3. **Доход единщика при перепродаже = ВСЯ сумма поступления**, а не маржа (п. 292.1 ПКУ). Учитывать только вознаграждение допустимо **исключительно** при корректно оформленном агентском/комиссионном договоре (п. 292.4 ПКУ) с подтверждающей первичкой.

4. **Структура из двух связанных ФОП несёт риск «дробления бизнеса».** Общий адрес/клиенты/управление/IP — основания для проверки ДПС и переквалификации. Минфин в 2026 готовит ужесточение (понятие «налоговое злоупотребление», аннулирование ЕН задним числом до 7 лет) — пока законопроект НЕ принят.

5. **Свежие изменения 2025–2026, требующие действий:** военный сбор для 3 гр. — 1% дохода (с 01.01.2025); обязательная уплата ЄСВ с 2026; квартальная объединённая отчётность с 2026 (первый отчёт до 10.05.2026); полные штрафы за работу без РРО/ПРРО — с 01.08.2025.

**🔴 Срочные action items:**
- [ ] **Перепродажу лицензий** оформить юридически осознанно: выбрать модель (реселлинг от своего имени / агентский договор / уход в юрлицо) и получить **ІПК** по конкретным договорам с вендорами.
- [ ] Проверить, не оказывает ли **ФОП 2 группы** услуги юрлицам/общесистемщикам — если да, срочно переоформить на 3 группу.
- [ ] Убедиться, что оплата приходит **только на IBAN безналично** — иначе обязателен РРО/ПРРО (штрафы с 01.08.2025).
- [ ] Проверить начисление **военного сбора 1%** у ФОП 3 гр. за все кварталы 2025.
- [ ] Оценить риск «дробления» двух ФОП: разные клиенты, адреса, IP, реальная деловая цель; подготовить обоснование.
- [ ] При выплатах иностранным вендорам проверить обязанность удержать **налог на репатриацию 15%** и применимость конвенции (для США).

## 5. Сравнение режимов и триггеры перехода

| Режим | Налоги | Лимит/условия | Кому подходит |
|---|---|---|---|
| **ФОП 2 гр. ЕН** | до 20% МЗП фикс. + ВЗ 10% МЗП + ЄСВ 22% | Лимит 2026: 7,21 млн; **только население/единщики** | НЕ для B2B CRM |
| **ФОП 3 гр. 5% без НДС** | 5% оборота + ВЗ 1% + ЄСВ 22% | Лимит 2026: 10,09 млн; B2B без ограничений | **Оптимально сейчас** |
| **Общая система (ФОП)** | НДФЛ 18% + ВЗ 5% + ЄСВ 22% (с чистого дохода) | НДС при обороте >1 млн | Только при высоких расходах |
| **Дія.City (ТОВ)** | Корп.: 9% на выведенный капитал ИЛИ 18% прибыль; спецы: НДФЛ 5%+ВЗ 5%+ЄСВ | ≥90% IT-дохода, ≥9 сотрудников, ЗП ≥€1200 | При масштабе/найме |
| **ТОВ на ЕН 3 гр.** | 5% оборота + ВЗ 1%; дивиденды НДФЛ 9%+ВЗ 5% | Лимит как у 3 гр. | Когда нужна защита от дробления |

**Триггеры перехода с упрощёнки:**
- Приближение к лимиту дохода 3 гр. (~10 млн грн/год) → Дія.City даёт неограниченный оборот.
- Рост числа ФОП в структуре (10–20) → высокий риск дробления → ТОВ/Дія.City.
- Активная перепродажа лицензий (риск роялти) → юрлицо с правом дистрибуции.
- Найм команды + ЗП ≥€1200 + ≥9 человек → Дія.City выгоднее по нагрузке.

> ⚠️ Неопределённости: ставка ВЗ для специалистов Дія.City (источники расходятся 1,5% vs 5%); лимит закупок ТОВ у единщиков (≤20% с 2025 — один источник) — сверить по ПКУ.

## 7. Открытые вопросы для очной консультации (бухгалтер/юрист + ІПК ДПС)

1. **Перепродажа лицензий Salesforce/Microsoft/HubSpot:** как именно квалифицируются ваши конкретные договоры — поставка услуги, роялти или посредничество? От этого зависит, законен ли вообще ЕН для этого дохода. **Главный вопрос для ІПК.**
2. **Налог на репатриацию по конвенции Украина–США:** точная ставка для роялти вендорам из США (15% базовая или ниже по конвенции) — требует чтения текста конвенции + справки о резидентности.
3. **Можно ли единщику быть дистрибьютором ПО** с перепродажей прав без потери ЕН — официального разъяснения нет, нужна ІПК.
4. **Структура двух ФОП:** оценка риска дробления для вашей конкретной конфигурации (2 гр. + 3 гр.) — отдельной практики нет, нужна оценка юриста.
5. **Применима ли ст. 208-1 (е-услуги)** к каждому конкретному вендору (зарегистрирован ли он в украинской системе НДС на е-услуги).
6. **Целесообразность перехода на Дія.City/ТОВ** с учётом ваших оборотов, числа сотрудников и планов по перепродаже лицензий.

## 8. Бухгалтерский учёт (дополнение)

> **Главное:** для вашей текущей структуры (два ФОП на едином налоге) полноценного бухгалтерского учёта по закону **нет**. Полные правила НП(С)БО ниже применимы **только к юрлицу (ТОВ)** — на будущее, при росте/переходе. Для ФОП сейчас актуальны лишь учёт доходов (раздел 8.1) и управленческий учёт (раздел 8.5).

### 8.1. Что обязан вести ФОП-единщик (ваш случай сейчас)

ФОП-единщик **НЕ ведёт** бухучёт по НП(С)БО: нет баланса, финотчётности, плана счетов, амортизации, учётной политики. Обязателен только **учёт доходов в произвольной форме** (ст. 296 ПКУ):
- 1–2 группы и 3 группа без НДС (ваш случай) — помесячный учёт **доходов** в произвольной форме (бумага/электронно), в ДПС не регистрируется;
- 3 группа с НДС — учёт доходов **и расходов**.
- Доход — **кассовым методом** (дата поступления денег, п. 292.6 ПКУ).

Источники: ст. 296 ПКУ (i.factor.ua/ukr/law-24/section-132/article-33028); Factor НіБУ №42/2025.

### 8.2. Когда бухучёт становится обязательным (переход на ТОВ)

Все юрлица (включая ТОВ на едином налоге) обязаны вести бухучёт и подавать финотчётность — **Закон №996-XIV**. Категория предприятия определяется по 2 из 3 показателей за предыдущий год:

| Категория | Активы (€) | Чистый доход (€) | Численность |
|---|---|---|---|
| Микро | до 350 тыс. | до 700 тыс. | до 10 |
| Малое | до 4 млн | до 8 млн | до 50 |
| Среднее | до 20 млн | до 40 млн | до 250 |
| Крупное | свыше 20 млн | свыше 40 млн | свыше 250 |

- Типичная CRM-ТОВ = **микро или малое** → применяет **НП(С)БО** (не МСФЗ), **аудит НЕ обязателен**.
- МСФЗ и обязательный аудит — только для средних/крупных и предприятий общественного интереса.
- Отчётность микро/малых: упрощённые формы по **НП(С)БО 25** (№1-мс/№2-мс для микро; №1-м/№2-м для малых). Годовая — **до 28 февраля** (за 2025 → до 2 марта 2026).
- **Дія.City:** бухучёт по общим правилам + дополнительно **Звіт про відповідність** с независимым заключением аудитора (не полный аудит) в Минцифры до 1 июня.

Источники: Закон 996-XIV (zakon.rada.gov.ua/laws/show/996-14); Factor НіБУ №102/2025; НП(С)БО 25 (z0161-00); mof.gov.ua. ⚠️ Пороги категорий подтверждены для отчётности за 2025; на 2026 Factor сообщает о новых критериях — сверить.

### 8.3. Признание выручки по НП(С)БО 15 (для ТОВ)

| Направление | Метод признания | Норма |
|---|---|---|
| Перепродажа лицензий | В момент продажи (переход рисков/выгод покупателю) | п. 8 |
| Внедрение/интеграция (проект) | **По степени завершённости** на дату баланса | п. 10–11 |
| Разработка ПО (долгосрочно) | По степени завершённости («percentage of completion»), НЗП на сч. 23 | п. 10–11 |

### 8.4. Лицензии на ПО: НМА, расходы или товар (НП(С)БО 8)

- **НМА** (субсч. 127) — лицензия для собственного использования сроком >1 года; **амортизируется** в течение срока полезного использования (метод и срок — в приказе об учётной политике; по умолчанию прямолинейный; можно применять налоговые сроки — мин. 2 года для группы 6 ПКУ). НМА с неопределённым сроком не амортизируется.
- **Товар** (сч. 281) — лицензии, купленные **для перепродажи** (ваш случай реселлинга): не НМА, а запас; доход — по п. 8 НП(С)БО 15, себестоимость — Дт 902.
- **Расходы периода / сч. 39** — короткий срок использования или роялти за право пользования (без перехода исключительных прав).
- **Собственная разработка** — капитализация на сч. 154→127 только на этапе разработки при выполнении критериев п. 7; этап исследований — расходы периода.
- **Резерв под гарантийную поддержку** — субсч. 473 (НП(С)БО 11, п. 13–14), методика в учётной политике (обычно % от выручки).

Источники: НП(С)БО 8 (приказ №242 от 18.10.1999); НП(С)БО 11 (приказ №20 от 31.01.2000, z0085-00); Factor; cursor-audit.com; yefimov-partners.com (31.03.2025).

### 8.5. Управленческий учёт (полезно уже сейчас, при ФОП)

Не регламентируется законом, но критичен для финмодели:
- **Маржа раздельно по потокам:** (1) перепродажа лицензий — низкая маржа (себестоимость = закупка у вендора); (2) услуги внедрения/поддержки — высокая маржа (себестоимость = ФОТ + накладные). Смешивать нельзя — искажает юнит-экономику.
- **Подписки = deferred revenue:** годовая предоплата поддержки — это **обязательство**, признаётся равномерно (1/12 в месяц), а не «прибыль» в момент оплаты (по аналогии со сч. 69 «Доходи майбутніх періодів»).
- **Расхождение налогового и управленческого учёта — это норма:** у ФОП кассовый метод даёт «всплеск» дохода в месяц оплаты подписки; для реальной месячной прибыльности и MRR ведите **параллельный учёт по начислению**, распределяя подписочную выручку по периодам.

Источники: НП(С)БО 9, 15, 16; сч. 69 (ibuhgalter.net/articles/594); Factor; Дебет-Кредит.
