## 1. EXECUTIVE SUMMARY

**What it is:** Kommo (formerly amoCRM) is a cloud-based, messenger-first CRM system that positions itself as "the #1 AI CRM for messenger-based sales." Operated by QSOFT LLC, originally founded as amoCRM in 2004 by Mikhail Tokovinin, Egor Kovalchuk, and Denis Mitrofanov. Rebranded to Kommo in late 2022.

**Who it's for:** Small and medium-sized businesses (SMBs) whose primary sales channels are messaging apps — WhatsApp, Instagram, Telegram, TikTok, Facebook Messenger. Strong fit for: e-commerce, real estate, healthcare, tourism/travel, financial services, call centers, and service businesses relying on high-volume chat-based communication.

**Core differentiator:** Unlike traditional CRMs (Salesforce, HubSpot, Pipedrive) which are pipeline-first, Kommo is messenger-first — every feature is designed around sales happening inside WhatsApp, Instagram DMs, Telegram, and TikTok. The platform adds a visual sales pipeline on top of a unified messaging inbox. Key differentiator is the native AI agent (not a third-party add-on) that handles sales, qualification, and support conversations 24/7 across all messaging channels from within the CRM.

**Scale:** Available in 50+ countries, 21+ industries, 9+ languages, 200+ integrations.

**Key quotes:**
- *"The #1 AI CRM for messenger-based sales"* — kommo.com
- *"Automate conversations on WhatsApp, Instagram, and TikTok while managing your entire pipeline from one platform"*
- *"The world's first messenger-based CRM system designed to help small businesses take their sales to a different level"*

## 5. TECHNICAL REFERENCE

### API Base URL

```
https://{your-account-subdomain}.kommo.com/api/v4/
```

Example: `https://mybusiness.kommo.com/api/v4/leads`

### Authentication — OAuth 2.0

**Authorization Endpoint:**
```
https://www.kommo.com/oauth
```

**Token Endpoint:**
```
https://{subdomain}.kommo.com/oauth2/access_token
```

**Token Types:**
- **Access Token:** JWT; lifespan = 24 hours
- **Refresh Token:** Lifespan = 3 months; new refresh token generated on each access token refresh; revoked if integration unused for 3 months

**Token Request Parameters:**
```
client_id, client_secret, grant_type, code (or refresh_token), redirect_uri
```

**Integration Types:**
- **Private:** Single-account use; no marketplace listing required
- **Public:** Multi-account; must pass Kommo's security audit; listed in Marketplace

### Rate Limits

- **Maximum:** 7 requests per second per account
- **Exceeded limit:** HTTP 429 Too Many Requests
- **Repeated violations:** IP address blocked → HTTP 403 on all responses

- **Maximum webhooks per account:** 100

**Webhook Event Types:**
- Lead: added, edited, deleted, restored, responsible user changed, status changed, note added
- Contact: added, edited, deleted, restored, responsible user changed, note added
- Company: added, edited, deleted, restored, responsible user changed, note added
- Task: added, edited, deleted, responsible user changed
- Customer: added, edited, deleted, responsible user changed, note added
- Incoming lead: added, edited, deleted
- Chat events (separate format at kommo.com/developers/content/chats/webhooks-format/)

## AI Функції — Повна документація (YouTube)

### AI Rewriter

7 режимів переписування тексту (доступні з картки ліда одним кліком):

1. **Correct grammar and spelling** — виправляє граматику, орфографію, пунктуацію
2. **Make it professional** — надає діловий, офіційний тон
3. **Make it friendly** — робить повідомлення дружнім
4. **Make it witty** — конверсаційний, ненапружений стиль
5. **Make it longer** — розгортає короткі нотатки у детальний текст
6. **Make it shorter** — скорочує довгий текст до ключових тез
7. **Make it simpler** — спрощує складні формулювання

Додатково: кнопка **Undo changes** — скасовує останню зміну.

**Suggestion tool (проактивний режим):** Ko AI стежить за тоном і довжиною тексту під час введення і пропонує виправлення ще до відправки.

### Conversation Summary

- Кнопка **Summarize** в нижній частині кожної розмови
- Генерує короткий огляд: ключові точки + невирішені питання
- Підходить для передачі розмови іншому менеджеру
- Резюме можна **закріпити** (pin) або **видалити** після використання

### AI Suggested Replies

**Умови роботи:**
- Відповіді генеруються на основі **останнього повідомлення клієнта** + **knowledge base** компанії
- Без завантаженої knowledge base функція не працює

**Налаштування knowledge base:** Settings → Communication Tools → AI Suggested Replies → увімкнути toggle → **Add new knowledge source**

**Типи джерел:**

**URL-джерело:**
- Вводиться URL сторінки або сайту
- Опція: індексувати тільки конкретну сторінку або **всі підключені сторінки** (до **1 500 веб-сторінок** на одне URL-джерело)
- Недоступні для парсингу: сторінки за JavaScript, з логіном/паролем, капчею, документи-посилання
- AI обробляє тільки **текст**, не зображення
- Оновлення: відкрити джерело → кнопка **Update**

**Текстовий файл:**
- Потрібно вказати **мову документу**
- Шаблон: кнопка **Download template**
- Оновлення: завантажити новий файл, видалити попередній

### AI Agent — можливості

- Відповідає на повідомлення **24/7** без участі людини
- Ставить уточнюючі запитання, розуміє наміри клієнта
- **Автоматично збирає дані ліда**: ім'я, локація, телефон, тип послуги
- **Автоматично створює профіль ліда** у CRM
- **Записує на зустріч**: перевіряє доступні слоти, адаптується до побажань клієнта, підтверджує час
- **Бронює у CRM-календарі** — без ручного внесення, без подвійних записів
- **Автоматичний follow-up**: через 24 год нагадує про майбутній візит
- Відповідає на типові запитання: години роботи, ціни на послуги
- **Розпізнає наявних клієнтів** і при потребі **передає розмову людині** (handoff)
