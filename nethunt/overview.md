# NetHunt CRM — Общий обзор

## Что такое NetHunt

NetHunt CRM — облачная система управления взаимоотношениями с клиентами, встроенная напрямую в Gmail и Google Workspace. Разработана украинской командой. Данные хранятся на серверах Google в США.

**Главная идея:** менеджер не покидает почту — CRM живёт внутри inbox.

**Целевая аудитория:** SMB (1–100 человек), B2B sales-команды, SaaS, digital agencies, недвижимость, стартапы. Роли: основатели, руководители продаж, BDR/SDR, маркетологи, Customer Success.

**Слоган:** *"Organize. Automate. Grow."*

## Архитектура системы

Иерархическая объектно-реляционная модель:

```
Workspace (рабочее пространство)
└── Folders (папки — контейнеры данных)
    └── Records (записи с уникальным Record ID)
        ├── Fields (поля разных типов)
        └── Timeline (лента активности)
```

### Базовые папки (из коробки)
- **Contacts** — контакты: лиды, клиенты, партнёры
- **Deals** — сделки
- **Companies** — компании
- **Tasks** — задачи

Папки полностью кастомизируемые — можно создавать под любой бизнес-процесс (рекрутинг, тикеты, проекты и т.д.)

### Record ID
Уникальный системный идентификатор каждой записи. Критически важен при двусторонней синхронизации и импорте из внешних таблиц — исключает дублирование.

### Timeline (Лента активности)
Все изменения, комментарии, звонки, файлы и переписка хронологически консолидируются в таймлайне записи.

### Presets (Пресеты)
Предопределённые значения полей по умолчанию при создании записи. Пример: любая новая сделка автоматически получает `Probability = 50%`.

## Типы полей

| Тип | Описание |
|-----|----------|
| Текст, число, дата | Стандартные типы |
| Checkbox | Да/нет (например: Newsletter subscribed) |
| Dropdown | Выпадающий список с предустановленными значениями |
| Tag / Picklist | Мультиселект — несколько значений |
| Страна | Встроенный список стран с поиском по коду |
| Соцсети | Специализированное поле для соцсетей |
| Related Records | Связь с записями другой папки |
| **Formula** | Автоматические расчёты или конкатенация строк (First Name + Last Name → Full Name) |
| **Related Summary** | Агрегация числовых значений из связанных записей (сумма выигранных сделок по компании) |
| **Statistical** | Поведенческая статистика: Last Email, Total Emails, Links click count, Last Interaction Date, время на текущей стадии сделки |

### Важные ограничения
- Текстовый поиск требует **точного синтаксического соответствия** — «square foot» и «SQFT» — разные значения
- Стандартизированные метрики рекомендуется делать через **dropdown/picklist**, а не свободный текст
- Клонирование полей: иконка в виде двух страниц — быстрое дублирование структуры

---

## Связи между папками (Related Records)

- Дочерняя запись отображается внизу родительской карточки, родительская — вверху дочерней
- **Критически важно:** связи должны быть **строго односторонними** (Contacts → Companies)
- Циклическая (двусторонняя) линковка **ломает Workflows** — роботы уходят в бесконечный цикл

## Snippets (Ярлыки / превью в инбоксе)

Динамические метки, показывающие ключевые поля записи прямо в списке писем Gmail — без открытия карточки.

**Настройка:** `Settings → Folder → навести мышь на поле → иконка закладки`

**Рекомендация:** не более 2–3 снипетов. Показывать: должность, тип контакта, лид-источник.

---

## Views (Представления)

Сохранённые фильтры — главный инструмент работы с данными.

- **List view** — табличный список
- **Card view** — карточки, группируемые по любому полю (Kanban)
- Можно делать личными или общими для команды
- Сохранение: «Save as» (не «Save» — иначе перезапишет текущий)

### Примеры готовых Views
| Название | Фильтр |
|----------|--------|
| Not contacted for 2 days | Last Contact Date > 2 дня назад |
| Lost — reason: missing features | Status = Lost + Reason = Missing Features |
| Lost — reason: no budget | Status = Lost + Reason = No Budget |
| Forecast view | Сделки, закрывающиеся скоро |
| Lead Source view | Карточки по источнику лида |
| Newsletter subscribers | Checkbox Newsletter = true |
| Campaign tags | Тег с названием кампании |

## Безопасность

- Данные хранятся на серверах Google (США)
- С 2019 года Google проводит ежегодный **Google Security Assessment** для всех приложений с доступом к Gmail API
- NetHunt ежегодно проходит независимые аудиты безопасности

---

## Критическое ограничение

Пользователи **без Google Workspace** (Outlook, обычный Gmail и др.) теряют:
- Прикрепление файлов к записям
- Синхронизацию Google Calendar
- Массовые email-рассылки
- Автоотправку писем через Workflows
- Синхронизацию Google Contacts
- Интеграцию с Google Looker Studio

Система **функционально нецелесообразна** без корпоративного Google Workspace.

## Плюсы и минусы

### Плюсы
- Нет переключения контекста — всё в Gmail
- Нативный LinkedIn-парсер (1 клик = запись в CRM)
- Гибкая кастомизация без программистов
- Нативные WhatsApp и Instagram без посредников
- Быстрый старт (минуты)

### Минусы
- Требует Google Workspace — Outlook не поддерживается
- API только с Business Plus ($68/мес)
- Встроенная аналитика ограничена — сложные отчёты только через Looker Studio
- Нет бесплатного плана (freemium)
- Сложная первичная настройка связей и Workflows — часто требует поддержки вендора
- Лимиты автоматизаций даже на высоких тарифах
- Проблемы с привязкой писем при одинаковой теме у нескольких клиентов
