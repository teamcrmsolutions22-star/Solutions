## Конкретные функции, показанные в видео

### Snippets (превью в списке писем)
- Настройка: иконка закладки у поля в настройках папки
- Показывают должность, тип контакта, лид-источник без открытия карточки
- Максимум 2–3 снипета

### Email Tracking
- Включение: `Settings → Preferences → Email open tracking + Link click tracking`
- В окне Compose: иконка "i" (статистика открытий) и "указательный палец" (клики)
- В папке Sent: зелёная иконка с временным промежутком = «письмо открыто X времени назад»

### Автолинкинг писем к компаниям
- Для контактов — привязка по email-адресу
- Для компаний — привязка по **доменной части** адреса

### Views (сохранённые фильтры) — примеры из видео
- «Not contacted for 2 days» — через статистическое поле Last Contact Date
- «Lost customers — reason: missing features» → рассылка о новом функционале
- «Lost customers — reason: no budget» → промо-рассылка
- «Lead source» — карточный вид по источнику
- «Country view» — карточный вид по стране
- «Newsletter subscribers» — по checkbox-фильтру
- «Campaign tags» — кто получил какую кампанию
- «Forecast view» — кто вот-вот закроется

### Создание представления:
1. Настроить фильтры в папке
2. Нажать **«Save as»** (не «Save» — иначе перезапишет текущий)
3. Задать имя и видимость (личное / для всей команды)

### Google Calendar интеграция
- Синие иконки NetHunt на событиях
- Отключить иконки (без потери привязки): `Preferences → Show record linking icon on calendar overview`

### Google Contacts Sync
- `Settings → Google Contacts Sync → Gmail-лейбл → целевая папка → Sync`
- Синхронизация в реальном времени

## Use Cases из видео

### Отдел продаж (BDR/SDR)
- Захват лидов из веб-форм автоматически
- LinkedIn-скрапинг одним кликом
- Приоритизация инбокса по снипетам (должность + тип контакта без открытия письма)
- Фильтр лидов по открытию писем (открыл / не открыл) → приоритизация звонков
- Drip-кампании со стопом при ответе
- Прогнозирование выручки через Forecast view
- Реактивация потерянных клиентов через targeted рассылку

### Customer Success (практика NetHunt CS-команды)
- Начало дня: inbox + ключевые Views
- Dispatch лидов через Workflow (round-robin)
- Онбординг: автоматические письма при регистрации, стоп при ответе
- Мониторинг активности: «неактивные пользователи» = красный флаг
- Follow-up каждые 3–6 месяцев через Last Contact Date
- Forecast view на еженедельных совещаниях
- @-mentions в таймлайне как основная внутренняя коммуникация без переключения в чат
- Сбор lost reasons → фидбек в продукт
- Уведомления о платежах в Google Chat через Workflow

### Маркетинг
- Сегментация базы через Views перед кампанией
- Тег с названием кампании — отслеживание кто получил что
- Анализ лид-источников: карточный вид по Lead Source
- Реактивационные кампании по сегментам

### Реальная недвижимость
- База прошлых клиентов с напоминаниями на дни рождения (задача каждые 365 дней)
- Ежемесячные newsletters с информацией о рынке
- Программа реферралов: поле «кто вас порекомендовал» в веб-форме
- Сегментация по территории для деловых поездок

## Email-маркетинг: детали из видео

### 5 типов drip-последовательностей
1. **Constant Content** — регулярный контент (newsletter, обновления продукта)
2. **Post Free-Trial** — серия для тех, кто не купил после трайла (лучшая для SaaS)
3. **Welcome/Onboarding** — самый высокий open rate, снижает unsubscribes и spam complaints
4. **Event Sequence** — приглашения на мероприятия
5. **Cart Abandonment** — для e-commerce: брошенная корзина → follow-up

### Сегментация перед рассылкой
- Checkbox `Newsletter: subscribed/unsubscribed` — базовый обязательный фильтр
- Tag-поле с названием кампании → кто получил что
- Views-фильтры: CEO в Канаде, триальные, потерянные по конкретной причине

### Пошаговая настройка drip в Workflows
1. Workflows → Add new → имя
2. Триггер: Record created / поле изменилось
3. Split Path: ветка A (письма), ветка B (ждать ответа), ветка C (ждать смены поля)
4. В ветке A: Send Email → Wait N days → следующее письмо → повторить
5. В конце каждой ветки: End Workflow
6. Activate

## Репортинг из видео

### User Activity Report
*(только Business и Advanced)*

Настройка:
1. Reports (левый сайдбар) → User Activity → Create
2. Выбрать метрики: emails sent/received, campaigns, calendar events, comments, call logs, files
3. Выбрать папку, пользователей, период
4. Сохранить → добавить виджет на Dashboard

Формат: первое число = текущий период, второе = предыдущий + % изменения.

### Отчёты кейса Recom
Ежедневные отчёты для клиентов: звонки в день, забронированные встречи, задачи, отправленные письма.

## Кейсы клиентов

### Recom (аутсорс BDR)
- **Контекст:** помогает компаниям находить клиентов; несколько клиентов одновременно, у каждого — своя pipeline
- **Проблема:** невозможно управлять 100+ проспектами по нескольким клиентам
- **Решение:** дублирование pipeline под каждого клиента, task reminders, фильтр по открытию писем
- **Любимая фича:** task reminders («lifesavers» для BDR)
- **Результат:** 30 → 150 клиентов (×5), +20–25 лидов в месяц от автоматизации

### AMO Digital Media (Growth Hacking Agency, Колумбия)
- **Контекст:** масштабирование стартапов через growth hacking и контент
- **Проблема:** данные разбросаны по каналам, нет истории отношений с клиентом
- **Решение:** консолидация всех баз, воронка, email campaigns
- **Любимая фича:** массовые рассылки + отслеживание открытий
- **Результат:** более близкие отношения с клиентами, рост продаж

### NetHunt CS-команда (внутренний кейс — Анастасия, Head of CS)
- 50–60 задач в день у одного менеджера в активный период
- 85–90% точность данных в CRM
- Workflow для dispatch входящих лидов между менеджерами
- Forecast view на еженедельных совещаниях команды
- @-mentions как основной инструмент внутренней коммуникации

## Стиль и позиционирование канала

- **Тон:** неформальный, но конкретный. «Easy peasy», «Jackpot!», «Let's get growing»
- **Формула каждого видео:** проблема → решение → шаги в интерфейсе
- **Главный дифференциатор:** «Всё происходит внутри Gmail — без переключения вкладок»
- **Призыв к действию:** 14-дневный триал + специальный план для стартапов упоминается в каждом видео
- В каждом видео — прямая демонстрация интерфейса, не теоретические схемы
