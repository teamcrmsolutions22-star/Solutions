# OneBox — Огляд, позиціонування, OS vs MVP

> Джерело: YouTube-транскрипти OneBox (автоматичні субтитри), дистиляція 2026-06-09

---

## Що таке OneBox

OneBox (OneBox OS / Ванбокс) — це платформа для автоматизації бізнесу, побудована як **набір взаємодіючих застосунків (apps)**. Кожен застосунок — окрема функціональна одиниця (CRM, склад, фінанси, документи, пошта, тощо), що обмінюється даними з іншими застосунками через внутрішній механізм shering даних.

Основний сайт: `crm-onebox.com`

---

## OneBox OS vs OneBox MVP (попередні версії)

| Параметр | OneBox MVP / старі версії | OneBox OS |
|---|---|---|
| Архітектура | Монолітна | Набір незалежних застосунків |
| Нові інтеграції та функції | Не розробляються | Тільки тут |
| Паралельна обробка | Немає (черга) | Так — усі ядра процесора |
| Мінімальні вимоги до сервера | 2 ядра, 2 ГБ RAM | 4+ ядра (рекомендовано) |
| Кількість застосунків | ~150–180 | 270–390+ (зростає) |
| Права доступу | Складний, роздроблений інтерфейс | Групи прав з візуальним інтерфейсом |
| Налаштування інтерфейсу | Обмежене | Повністю кастомізовані вкладки, сітка, мобільна версія |
| Відраслеві рішення (конфігурації) | Відсутні | Є каталог (~11+ офіційних, сотні партнерських) |
| Швидкість глобального пошуку | Повільна, послідовна | Паралельна по всіх застосунках |
| Мобільна робота | Погана | Кожен застосунок можна встановити як окрему PWA-іконку |
| Підтримка та нові функції | Лише критичні баги | Увесь розвиток продукту |

**Ключовий меседж:** Будь-які нові можливості, інтеграції та виправлення з'являються тільки в OneBox OS. Старі версії отримують лише критичні виправлення.

## Архітектура: концепція застосунків (apps)

- OneBox OS — це **не єдина програма**, а **набір застосунків** (apps / "опок"), кожен з яких є незалежним модулем.
- Застосунки можуть розташовуватись на **різних серверах** і навіть в різній інфраструктурі.
- Взаємодія між застосунками: один застосунок надсилає дані до іншого, той отримує їх і запитує «що з ними робити».
- **Shering (обмін даними):** з будь-якого списку можна виділити записи та відправити їх в інший застосунок (наприклад, з пошти → контакти → платежі → документи).
- **Паралельний пошук:** запит одночасно відправляється у всі встановлені застосунки, результати збираються і відображаються разом — незалежно від кількості бізнес-процесів.

### Типи застосунків

| Тип | Опис |
|---|---|
| **Нативні (steam-apps)** | Розробляє команда OneBox або авторизована компанія |
| **Wake-up apps** | "Fake app" — сторінка-заглушка для валідації попиту; кнопка «повідомити про появу» генерує ліди |
| **Workflow apps** | Бізнес-процеси або їх групи, запаковані в застосунок; при встановленні розгортають налаштовані процеси |
| **iFrame apps** | ⚠️ потребує перевірки — анонсований, але статус реалізації не уточнений |
| **Конфігураційні (config apps)** | Повноцінне відраслеве рішення: ставиш одну «app» — отримуєш готово налаштований бокс під нішу |

- Будь-який партнер може розмістити свій застосунок у каталозі.
- Дублікати однотипних застосунків без суттєвих відмінностей не допускаються.
- Якщо поставщик застосунку не реагує на запити — OneBox публічно сповіщає про це і може передати застосунок іншому партнеру.

## Ціноутворення та ліцензування

### Хмара (Cloud)

- Базова ставка: **~$25/користувач/місяць** (усі застосунки доступні).
- Тариф $0: безкоштовний базовий рівень.
- Тариф $375: ⚠️ потребує перевірки точної назви та умов.
- Індивідуальний тариф — для великих клієнтів.
- Планується **тарифікація «за даними»** (pay-per-data): OneBox безкоштовний до певного порогу даних; наприклад, ~$1 за 1000 контактів, ~$1/міс за кожен бізнес-процес понад перші 10. ⚠️ Це анонсована модель, точні цифри наведено як приклад.

### Коробка (On-premise / Boxed)

- Клієнт **не купує софт** — купує **активаційний ключ**; сам софт надається безкоштовно.
- Аналогія: як Windows — ключ купується окремо від дистрибутива.
- Ключ прив'язаний до кількості користувачів і терміну (рік).
- Перший рік: ~$1000–$1200 за ключ (⚠️ ціни на момент запису; можуть бути скориговані).
- Наступні роки: ~$120/рік; для тих, хто уклав договір раніше — ~$50/рік.
- Вічних ключів немає.
- **Юридично:** OneBox продає ключ, а не ПЗ. Після встановлення на сервер клієнта — компанія OneBox знімає з себе відповідальність за збереження даних і працездатність системи.
- Клієнт сам забезпечує резервне копіювання або замовляє окремо.

### Перехід з коробки на хмару

- Існуючим коробковим клієнтам: **до 6 місяців хмари безкоштовно**.
- Технічно: OneBox вилучає ліцензії і видає еквівалентну кількість активаційних ключів для OS.

### Мінімальні вимоги для on-premise OneBox OS

- Мінімум: **4 ядра процесора** (проти 2 ядер для MVP).
- OS ефективно використовує всі доступні ядра; більше ядер = пропорційно швидша робота.
- Тактова частота одного ядра важливіша для деяких операцій; загальний паралелізм — для пакетної обробки.

## OneBox Network

**OneBox Network** — застосунок для обміну даними між різними інсталяціями OneBox (хмарними або коробковими).

### Можливості

- Передача між боксами: **бізнес-процеси, документи, платежі, контакти**.
- Синхронізація коментарів у реальному часі (~1–2 секунди затримки).
- При отриманні даних — автоматичне створення відповідного бізнес-процесу (налаштовується: який БП запустити при отриманні процесу / контакту / документа / платежу).
- Тонкі налаштування: відповідність полів між боксами, передача продуктів процесу.
- Якщо у контрагента немає OneBox — він отримує посилання для завантаження документа або запрошення встановити OneBox.

### Сценарій використання

- Холдинг: центральний бокс + окремі бокси по філіалах/містах → локальний адмін ніколи не отримає доступ до даних інших боксів (фізично різні машини).
- Клієнт і постачальник: замість Excel та email — відправка замовлення з усіма продуктами напряму в OneBox постачальника.
- Двостороннє листування в рамках одного процесу між різними компаніями.

## Партнерська мережа

OneBox не спілкується з кінцевими клієнтами голосом — вся підтримка відбувається через партнерів. Це свідома модель масштабування через дистрибуторську мережу.

### Типи партнерів

| Тип | Роль |
|---|---|
| **Дистрибутор** | Продає ліцензії (хмара та коробки); є реселером хмарних тарифів; може робити власну націнку |
| **Інтегратор** | Впроваджує, налаштовує, надає підтримку клієнтам |
| **Sales Partner** | Продажний партнер; допомагає з підбором та покупкою |
| **Маркетинговий партнер** | Суміжні послуги (⚠️ деталі не розкриті) |

- Ролі Дистрибутора та Інтегратора можуть поєднуватися.
- Дистрибутори закуповують хмарні тарифи/ключі за нижчою ціною і перепродують з надбавкою; партнерські ціни нижчі за публічні на сайті.
- Партнеру видається спеціальний кабінет для покупки ключів і управління клієнтами.
- Стати партнером: реєстрація на `crm-onebox.com`, мінімальна модерація (для базового доступу до документації), договір потрібен для статусу інтегратора / дистрибутора.
- Партнер, що розміщує застосунок, **не може залишатись непомітним** — він завжди публічно вказаний як поставщик.

### Підтримка клієнтів

- **Product support** (оновлення, виправлення багів) — включений у всі тарифи.
- **Customer support** (відповіді на запитання) — **не включений** у тариф; клієнт звертається до партнера.
- Форум OneBox: запитання можна задавати; до дати X команда OneBox відповідає безкоштовно.
- OneBox з клієнтами **не розмовляє голосом** — якщо хтось телефонує і каже, що є «співробітником OneBox» (не партнером) — це неправда.

## Ключові функціональні особливості OneBox OS

### Продуктивність
- Паралельна обробка на рівні ядер процесора: імпорт прайсів, виконання бізнес-процесів, виконання пошуку — все паралельно.
- Приклад: виконання 100 прайс-площадок паралельно замість 45-годинної черги → 15–20 хвилин.
- Пересчет цін по правилам — у реальному часі у фоні, без блокування таблиць.
- Пошук по 16 млн контактів — менше 3 секунд.

### Інтерфейс
- Стандартизований: фільтри ліворуч, пошук завжди на одному місці, шестерня (⚙️) → налаштування конкретного застосунку.
- Кастомні вкладки (tabs) у картках: скільки завгодно вкладок, власна сітка, мобільна сітка, дія по кліку на вкладку.
- Кожен застосунок можна встановити як окрему іконку на смартфон (PWA-подібний механізм, QR-код).

### Права доступу
- Групова модель: створюєш групу прав, додаєш співробітників.
- Візуальний інтерфейс — відмічаєш, які застосунки доступні.
- За замовчуванням: співробітник бачить тільки свої дані (де він менеджер/автор).
- Можна надати права на налаштування окремих застосунків без прав адміна.
- Копіювання налаштувань між групами прав.

### Бізнес-процеси та автоматизація
- Дії/тригери по статусах процесів.
- Масові розсилки (email, Viber, SMS, push) з відстеженням (відправлено / доставлено / прочитано) — без зовнішніх сервісів розсилок.
- Автоматизовані тригери додавання контактів у розсилки.

### Фінанси
- Cashflow (фактичний) + Прогноз cashflow (на основі очікуваних платежів).
- Взаєморозрахунки: конструктор зобов'язань, схожий на 1С; можна налаштувати кілька копій для різних юросіб.
- Фінансовий формат: від'ємні числа у дужках (стандарт GAAP/IF-123).
- Планується 4 стандартних фінзвіти (cashflow, P&L, баланс, капітал) за міжнародним стандартом GAAP.

### Документи
- При виборі шаблону — миттєвий preview з підставленими змінними.
- Формати: PDF, XLS, XML, DOC, HTML.
- Колекція шаблонів документів: вбудована бібліотека типових документів (накладні, акти, інвойси, договори) — завантаження в 1 клік.
- Файлове сховище: єдине сховище всіх файлів у вигляді провідника (папки, перетягування).

### Холдингові компанії
- Вбудований функціонал ієрархії компаній: холдинг → дочірні компанії.
- Блоки інтерфейсу: «Структура холдингової компанії» та «Дочірні компанії».
- Підтримка кольорового маркування за значенням поля.
- Імпорт ієрархії через Excel.
- Відображення ієрархії також у картці процесу (бізнес-процесу).

## Цільові сценарії використання

- Компанії будь-якого розміру, що хочуть гнучко налаштувати CRM+ERP під свій бізнес.
- Холдинги з філіальною мережею (OneBox Network для децентралізованої роботи).
- Інтернет-магазини та e-commerce (інтеграція з майданчиками, ТТН, склад).
- Компанії, що шукають відраслеве рішення «з коробки» (конфіг-застосунки від партнерів).
- B2B з постачальниками, що також використовують OneBox (обмін документами через Network).

## Конкурентне позиціонування

- Ніхто з клієнтів не використовує навіть 100 з 270+ застосунків — OneBox позиціонує себе не як «комбайн для всіх», а як **платформу конкретних застосунків** під конкретні задачі.
- Просування: не OneBox як такий, а просування конкретних застосунків через контент/відео.
- На сайті на кожному застосунку буде: опис, скриншоти, відеопрезентація → продажник надсилає клієнту 3 посилання на потрібні застосунки замість організовувати демо.
- OneBox вкладає значні кошти у розробку: ~$375 000 на розробку тільки за один рік (⚠️ цифра з транскрипту, рік не уточнений).

## Оновлення з MVP/старої версії на OneBox OS

- Клієнт повинен **особисто** підтвердити оновлення через кабінет (кнопка на сайті).
- Процес напівавтоматичний: скрипт робить backup, переносить дані, автоматично встановлює потрібні застосунки.
- Після оновлення права доступу скидаються — потрібно вручну налаштувати групи прав.
- Інтернет-магазин: залишається хоча б 1 ліцензія на старому боксі + встановлюється застосунок `OneBox Shop` на новому боксі.
- Коробковим клієнтам для оновлення потрібен окремий контракт.
