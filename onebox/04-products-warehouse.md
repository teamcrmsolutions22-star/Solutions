## 1. Робота з карточкою продукту

### Створення продукту

- Три способи створення: через **Глобальне меню → Продукти → Додати продукт**, через список **Продукти списком → Новий продукт**, через **Продукти категоріями → Новий продукт**.
- Обов'язкове поле — тільки **Найменування**. Решта полів (ціна, закупівельна ціна, артикул, опис тощо) — опціонально.
- Поле **РРЦ** (рекомендована роздрібна ціна) заповнюється автоматично при пересчоті цін; вручну недоступне.
- **Зачеркнута ціна** відображається на сайті (для інтернет-магазину на базі OneBox), якщо значення не дорівнює нулю.
- Галочки **«Синхронізувати ціну з постачальниками»** та **«Синхронізувати наявність з постачальниками та складами»** — включати для автоматичного пересчоту цін (при включенні закупівельна ціна та наявність блокуються від ручного редагування).
- Блоки в карточці можна **згортати** — стан зберігається при наступному відкритті.

### Категорії товарів

- Категорії створюються в **Глобальне меню → Продукти → Категорії → Додати категорію**.
- Обов'язкове поле — назва категорії; батьківська категорія — опціонально (підтримується вкладеність).
- Один товар може належати лише до однієї категорії.

### Вкладки карточки продукту

- **Постачальники** — прив'язка одного або кількох постачальників з ціною, РРЦ, галочкою наявності у постачальника.
- **Фільтри** — характеристики товару (розмір, колір, сезонність тощо). Фільтри створюються в **Продукти → Фільтри та характеристики**.
- **Аналоги продукту** — список аналогів, що використовуються при заміні (функція «Замінити продукт на аналоги»).
- **Одиниці виміру** — прив'язка кількох одиниць виміру з коефіцієнтами перерахунку.
- **Спеціальні ціни** — фіксовані спеціальні ціни для конкретних клієнтів на даний товар.
- **Склади** — залишки по складах і комірках, журнал складських операцій.

## 2. Налаштування інтерфейсу карточки продукту

### Основи налаштування

- Перейти в **Шестерня (⚙️) → Інтерфейс карточки продукту** з будь-якого місця в розділі Продукти.
- Інтерфейс побудований на **секціях** (штрихова рамка). У кожну секцію можна додавати один або кілька блоків.
- Блоки переміщуються між секціями та всередині секцій перетягуванням (drag-and-drop).

### Типи блоків

| Блок | Особливості |
|------|-------------|
| **Загальна інформація про продукт** | Стандартний набір полів — не редагується (поля зашиті); є одна настройка «Показувати повний шлях категорії» |
| **Універсальний блок** | Довільний набір полів; кілька таких блоків можна додавати; порядок полів змінюється всередині блоку; часто використовується замість «Загальної інформації» |
| Інші блоки (характеристики, акції, упаковка, постачальники тощо) | Стандартні, мають власні поля |

### Таби (вкладки)

- Додаються кнопкою **«+ Додати таб»** → вказується назва.
- Кожен таб показує тільки ті блоки, що розміщені на ньому.
- Для таба можна задати **дію при натисканні** (наприклад, «Склади → Переглянути баланс продукту» — перехід без виведення блоків).
- Порядок табів регулюється числовим полем **сортування** (менше = вище).
- Таби можна перейменовувати та видаляти.

### Різні інтерфейси для різних категорій

- За замовчуванням (ID категорії = 0) — загальний інтерфейс.
- Щоб налаштувати інтерфейс для конкретної категорії: в URL налаштувань замінити `0` на ID потрібної категорії.
- ID категорії видно в адресному рядку браузера при відкритті категорії.

### Мобільна версія

- Перемикання між Desktop і Mobile — іконка **«Сітка»** поруч із шестернею.
- У мобільній версії доступна одна секція (не кілька колонок).

## 4. Одиниці виміру товарів

### Де налаштовується

- У карточці товару є стандартне поле **«Одиниця виміру»** — виводиться у блоці «Загальна інформація про продукт» або через Універсальний блок.
- Довідник одиниць виміру: **Налаштування системи → Інше → Довідник одиниць виміру** (або через шестерню у списку продуктів).
- Нова одиниця виміру додається через кнопку **«+ Додати одиницю виміру»** → вказується назва → зберегти.

### Кілька одиниць виміру для одного товару

- У карточці товару на вкладці **«Одиниця виміру»** можна прив'язати кілька одиниць + задати **коефіцієнт** для кожної.
- Коефіцієнт відповідає за автоматичний **перерахунок кількості** при зміні одиниці виміру.
- Формула перерахунку: `Нова кількість = (поточна кількість × коефіцієнт поточної ОВ) / коефіцієнт нової ОВ`.
  - Приклад: 1 штука (коеф. 1) → упаковки (коеф. 200) = 1×1/200 = 0,005 упаковки.
  - Зворотно: 1 упаковка (коеф. 200) → штуки (коеф. 1) = 1×200/1 = 200 штук.

### Відображення в процесах (замовленнях)

- Колонка **«Одиниця виміру»** виводиться в блоці «Продукти таблицею»: **Налаштування інтерфейсу → Продукти таблицею → Налаштування → ввімкнути поле «Одиниця виміру»**.
- Щоб дозволити змінювати одиницю виміру прямо в замовленні: у тій же настройці натиснути шестерню поруч із полем → ввімкнути **«Дозволити змінювати одиницю виміру для товарів у цьому замовленні»**.
- У випадаючому списку доступні тільки ті одиниці, що задані на вкладці «Одиниця виміру» карточки товару.

## 5. Дробність товарів

### Що таке дробність

- **Дробність** — число, кратне якому можна виконувати складські операції з товаром.
  - Дробність = 5 → допустимо: 5, 10, 15, 20 … (не можна: 3, 6, 8 тощо).
  - Дробність = 0,1 → допустимо: 0,1; 0,2; 1,1 тощо.
- За замовчуванням при створенні товару дробність = **1**.

### Де задається

- У карточці товару — стандартне поле **«Дробність»** (виводиться у блоці **«Опція замовлення»** або через Універсальний блок).

### Де застосовується

- При **додаванні товару в процес** — кількість автоматично округлюється до кратного значення.
- При **оприходуванні** на склад — якщо введено некратне значення, система видає помилку і скидає до кратного.
- При **переміщенні** між складами.
- При **продажу** зі складу (дію «Продати вміст процесу зі складу») — система не виконає складську операцію, якщо кількість не кратна дробності.
- У **паспортах продуктів** (виробництво / специфікація) — дробність складових визначає мінімальний крок списання матеріалів.

## 6. Налаштування цін у карточці товару

### Цінові рівні від кількості

- У карточці товару поруч з ціною продажу є кнопка **«Налаштування цін»** → спливаюче вікно з таблицею «Кількість від → Ціна».
- Приклад: від 3 шт. — 95 грн/шт.; від 5 шт. — 90 грн/шт.; від 10 шт. — 80 грн/шт.
- Аналогічне вікно доступне й для кожного **рівня цін** (наприклад, «Оптова ціна»).
- Налаштування можна **імпортувати** через Excel-файл (колонки: «Налаштування цін: кількість від» та «Налаштування цін: ціна»).

### Як застосовується при додаванні товару в процес

- Якщо товар додається кнопкою **«Додати в корзину»** напроти назви — додається зі стандартною ціною, незалежно від кількості.
- Якщо натиснути **«+»** (розгорнути) поруч із товаром у вікні пошуку — відображається список варіантів «від N штук → ціна X», при кліку товар додається з відповідними кількістю і ціною.
- Після додавання ціна не змінюється автоматично при ручній зміні кількості в процесі.

### Автоматичний перерахунок ціни за кількістю

Два дії на етапі або процедурі бізнес-процесу:

**1. «Оновити ціну продажу продукту процесу згідно налаштування кількості в карточці товару»**
- Без додаткових налаштувань.
- Спрацьовує **при переході на етап**.
- Перераховує ціну продажу всіх товарів у процесі відповідно до налаштувань кількості.

**2. «Виконати перерахунок ціни продажу продукту залежно від кількості товару»**
- Є три галочки:
  - **«Враховувати лише відмічені продукти»** — пересчет тільки для обраних рядків.
  - **«Запускати дію при редагуванні процесу»** — спрацьовує при кожному збереженні, не тільки при переході на етап.
  - **«Враховувати рівень цін продукту процесу»** — при пересчоті враховує рівень цін, встановлений клієнту або процесу (а не стандартну ціну).

## 7. Функціонал спеціальних цін

### Де налаштовується

- **З карточки контакту**: вкладка **«Спеціальні ціни»** (зелена вкладка) → задати скидку (%) на категорію товарів або фіксовану ціну на конкретний продукт.
- **З карточки продукту**: вкладка **«Спеціальні ціни»** → вказати контакт (клієнта), спеціальну ціну та валюту. Це дзеркало тих самих даних.

### Логіка роботи

- **Скидка на категорію** (%) — застосовується до всіх товарів цієї категорії для даного клієнта. Підтримується мультивибір категорій.
- **Фіксована ціна на продукт** — перевищує пріоритетом скидку на категорію і рівень цін (price level): якщо задана фіксована спеціальна ціна на товар, вона застосовується завжди, незалежно від рівня цін клієнта.
- При додаванні товару в процес система автоматично підставляє спеціальну ціну:
  - Колонка «Скидка» залишається порожньою (0%).
  - Ціна продажу відразу задана зі спеціальною ціною.
  - Додаткову скидку можна внести вручну поверх спеціальної ціни.
- Пріоритет: **Фіксована спеціальна ціна на товар > Скидка на категорію з урахуванням рівня цін**.

### Приклад

| Ситуація | Результат |
|----------|-----------|
| Рівень цін «Ціна 1» = 100 грн, скидка на категорію 20% | Ціна продажу = 80 грн |
| Фіксована спеціальна ціна на товар = 500 грн | Ціна продажу = 500 грн (ігнорує рівень цін і скидку категорії) |

## 8. Перерахунок цін та наявності

### Що таке «Пересчет цін і наявності»

- Окремий застосунок у OneBox (встановлюється з маркетплейсу застосунків).
- Дозволяє автоматично формувати ціну продажу (та рівні цін) шляхом **автоматичної націнки** на закупівельну ціну постачальника або ціну оприходування на склад.
- Також автоматично **встановлює наявність** товару.

### Умови спрацьовування для товару

1. У карточці товару включені галочки: **«Перераховувати ціну з постачальниками»** та/або **«Перераховувати наявність з постачальниками та складами»**.
2. Товар є в наявності у постачальника або на складі (або ввімкнена настройка «Перераховувати для товарів, яких немає в наявності»).
3. Створено відповідне правило пересчоту.

### Базова настройка карточки товару

- **Галочка «Перераховувати ціну»**: закупівельна ціна та РРЦ блокуються від ручного редагування.
- **Галочка «Перераховувати наявність»**: галочка «Продукт у наявності» блокується.
- Постачальники прив'язуються на вкладці **«Постачальники»** з ціною та галочкою наявності.

### Правила пересчоту

- Перше правило завжди **базове** (видалити не можна) — застосовується для всіх товарів без явного правила.
- Нові правила: типи — **звичайне, стандартне, стандартне з відхиленням, індивідуальне, загальне**; відрізняються пріоритетом.
- Параметри правила:
  - **Ціна від / до** — діапазон закупівельної ціни постачальника, під який підпадає правило.
  - **Тип націнки** — відсоток або сума.
  - **Значення націнки** — відсоток або сума; можна взяти з додаткового поля товару.
  - **Скидка** — знімається з уже нарахованої націнки (відсоток або сума).
  - **Валюта** — правило діє тільки для постачальників із цією валютою.
  - **Склад, комірка, модель, бренд** — додаткові фільтри.
  - **Мінімальна / максимальна націнка**.
  - **«Не змінювати ціну продажу»** — правило не змінює ціну (актуально для наявності).
  - **Категорія** — правило діє тільки в обраній категорії.
  - **Приоритет** — цифрове поле; при конфлікті двох правил перемагає більш пріоритетне.

### Додаткові рівні цін (marketplace)

- Галочка у «Прочих настройках» застосунку: **«Враховувати додаткові ціни у правилах пересчоту»** → у кожному правилі з'являються поля націнки для кожного рівня цін.
- Приклад: Ціна продажу +20%, Marketplace1 +30%, Marketplace2 +40%.

### Пріоритет вибору ціни

- Настройка у **«Прочих настройках застосунку» → «Пріоритет вибору ціни»**:
  - **«Найвигідніша ціна»** (за замовчуванням) — обирається джерело з найнижчою закупівельною ціною.
  - **«Спочатку постачальник»** або **«Спочатку склад»**.

### РРЦ (рекомендована роздрібна ціна)

- Якщо у постачальника задано РРЦ і вона більша за розраховану ціну — автоматично застосовується РРЦ.
- Відключити: **Прочі настройки → «Відключити РРЦ»**.

### Запуск пересчоту

| Спосіб | Як |
|--------|----|
| По одному товару | В карточці товару: **«Відправити в пересчот цін та наявності» → «Перерахувати ціни для продуктів»** або **«… і переглянути детальний лог»** |
| Масово (з переглядом) | У застосунку «Пересчот цін» → кнопка **«Перерахувати ціни»** → таблиця з поточними та майбутніми цінами → **«Застосувати ціни»** |
| Масово (фон) | Кнопка **«Перерахувати всі ціни у фоні»** — без попереднього перегляду |
| Автоматично за розкладом | **Налаштування системи → Автоматика «Раз на годину»** → дія **«Пересчот цін»** з зазначенням часу (наприклад, 06:00 та 17:00) |

### Перерахунок у реальному часі

- Пересчот спрацьовує автоматично у фоні при зміні даних постачальника або зміні балансу товару на складі (якщо відповідні галочки ввімкнено у «Прочих настройках»).
- При масовому запуску система автоматично розбиває процес на потоки (по кількості ядер сервера) — паралельне виконання.

### Важлива поведінка при відсутності наявності

- Якщо товару немає у постачальника і немає на складі, пересчот **обнуляє** закупівельну ціну і ціну продажу (якщо не включені захисні галочки).
- Захист: **«Перераховувати для товарів, яких немає в наявності»**, **«Не обнуляти закупівельну ціну продукту при пересчоті наявності»**, **«Не обнуляти ціну продажу при пересчоті цін і наявності»**.

## 9. Термін придатності продуктів

### Встановлення терміну придатності

- Термін прив'язується не до товару загалом, а до **конкретної партії оприходування** на склад.
- При оприходуванні (Склади → Оприходування продуктів) у таблиці корзини є колонка **«Годен до»**.
- Якщо колонка не відображається: **Шестерня → Налаштування сторінки оприходування → блок «Таблиця корзини продуктів» → увімкнути поле «Годен до»**.
- Формат дати: `РРРР-ММ-ДД ГГ:ХХ`.

### Де переглядати термін придатності

1. **Карточка товару → вкладка «Склади»** — колонка «Годен до» у журналі складських операцій.
2. **Блок пошуку продуктів у процесі** — налаштування блоку → увімкнути **«Виводити термін придатності замість дати приходу товару»** → відображається дата терміну придатності по кожній партії.
3. **Звіт «Про терміни придатності товарів»** (Глобальне меню → Звіти) — фільтри по даті, постачальнику, складу, категорії.

### Автоматичне повідомлення про закінчення терміну

- **Налаштування системи → Автоматизація → «Раз на день» → дія «Термін придатності продуктів»**.
- Налаштування дії:
  - **«Нагадати за N днів»** — кількість днів до закінчення.
  - Назва бізнес-процесу (задачі), що буде створена.
  - Виконавець задачі.
  - Тип бізнес-процесу.
  - **«Додавати продукти до процесу»** — якщо ввімкнено, в задачу додається товар, що закінчується.
  - **«Враховувати тільки продукти в наявності на складі»**.
- Логіка: якщо до закінчення терміну залишилося менше N днів → перевіряє, чи вже є відкрита задача по цій партії → якщо ні, створює нову.

## 10. Заміна продуктів на аналоги

### Налаштування аналогів

- В карточці товару є вкладка **«Аналоги продукту»**.
- Аналоги додаються через пошук або кнопку **«Вибрати/Знайти/Створити»**.
- Зв'язок двосторонній: якщо товар А — аналог товару Б, то в карточці Б також відображається А.

### Дія «Замінити продукт на його аналоги»

- Розміщується на **процедурі** або **етапі бізнес-процесу**.
- Логіка за замовчуванням: якщо товар є на складі — заміна не виконується; якщо товару немає — замінює на перший підходящий аналог, у якого є наявність.

### Налаштування дії

| Налаштування | Опис |
|-------------|------|
| **«Ціну аналога проставляти як у товару, що замінили»** | Ціна після заміни залишається як у вихідного товару |
| **«Замінити тільки ID продукту в процесі, дані продукту не змінювати»** | Змінюється тільки ID (тобто ідентифікатор товару), всі інші поля (назва, розмір тощо) залишаються від вихідного |
| **«Враховувати тільки відмічені продукти»** | Заміна тільки для відмічених рядків (галочки в процесі) |
| **«Враховувати додаткові поля складу при пошуку продукту»** | Враховує фільтри/атрибути складу при пошуку аналога |
| **«Примусово замінити на 1 аналог незалежно від наявності на складі»** | Заміна виконується завжди, без перевірки наявності |

## 11. Комірки складів

### Що таке комірки

- **Комірки** — конкретне місце зберігання на складі (аналог полиці у шафі).
- Типово використовуються на великих складах для точного відстеження місця кожного товару.

### Створення комірок

- **Налаштування системи → Інше → (конкретний склад) → Налаштування комірок складу**.
- Поля комірки: **Назва** (обов'язково), штрихкод, батьківська комірка (ієрархія), порядок сортування, опис, менеджер комірки, локація (широта/довгота).
- Для швидкого додавання: вписати назву і натиснути «Додати».
- Підтримується **ієрархія** (дерево комірок): комірка може бути вкладена в іншу.
- **Дерево комірок всіх складів**: Склади → вкладка **«Дерево комірок усіх складів»**.

### Відображення в інтерфейсі процесу

- Колонка **«Комірка»** в блоці «Продукти таблицею»: **Налаштування інтерфейсу → Продукти таблицею → увімкнути поле «Комірка»**.

### Оприходування на комірку

- Склади → Оприходування → вибрати **Склад**, потім у полі **«Комірка»** обрати потрібну (показуються тільки комірки обраного складу).

### Продаж з комірки

- У процесі в колонці **«Склад»** обрати склад, у колонці **«Комірка»** обрати комірку.
- Дія **«Продати вміст процесу зі складу»** виконає продаж з вказаного складу та комірки.

### Переміщення між комірками

- У настройках дії **«Перемістити вміст процесу на інший склад»** є галочка **«З комірки переміщується товар»**:
  - **Вимкнено (за замовчуванням)**: поле «Комірка» у процесі — це куди переміщується.
  - **Ввімкнено**: поле «Комірка» — звідки беремо; куди — через поля **«Склад для переміщення»** та **«Комірка для переміщення»** (виводяться в Універсальному блоці процесу).

### Перевірка заповненості комірки

- Дія **«Перевірити, якщо продукти у комірці»**: бере ID комірки з вказаного додаткового поля процесу, перевіряє — якщо є товар, видає задану помилку.
- Застосування: перешкоджає розміщенню другого товару в комірку, розрахованою на один.

## 12. Вибір складу для продажу продукту в замовленні

### Як визначається склад для продажу

- Дія **«Продати вміст процесу зі складу»** (на етапі або процедурі) має поле **«Склад (постачальник)»** — звідки виконується списання.
- **Пріоритет**: склад, вказаний безпосередньо у товару в процесі (колонка «Склад»), **важливіший**, ніж склад у налаштуваннях дії.

### Ручне встановлення складу кожному товару

- У процесі є колонка **«Склад»** — можна вказати для кожного рядка окремо.

### Масове встановлення складу всім товарам одночасно

Схема автоматизації через дії на процедурі:

1. Додати **додаткове поле процесу** типу «Випадаючий список» (наприклад, «Склади ДПП») зі значеннями, що відповідають назвам складів.
2. Для кожного складу додати дію **«Вирахувати і записати значення в додаткове поле, якщо умова виконується»**: умова — якщо поле «Склади ДПП» = «Склад № 1», то записати ID складу (наприклад, 5) у додаткове поле продукту процесу «Результат».
3. Дія **«Копіювати поля продуктів процесу»**: копіює значення з поля «Результат» у поле **«ID складу продукту»**.
4. Менеджер обирає значення з «Склади ДПП» → запускає процедуру → усім товарам автоматично встановлюється обраний склад.

## 13. Використання співробітників як склад

### Увімкнення функції

- **Налаштування системи → Інше → Налаштування складу → «Дозволити використовувати співробітника як склад»** → зберегти.
- Зміни набувають чинності протягом ~1 години.

### Доступні операції зі складом-співробітником

| Операція | Через глобальне меню | Через БП (дії на етапі) |
|----------|---------------------|------------------------|
| Оприходування | ✅ (Склади → Оприходування → поле «Склад» → вибрати співробітника) | ✅ (дія «Оприходувати вміст процесу на склад») |
| Переміщення | ✅ | ✅ (дія «Перемістити вміст процесу на інший склад») |
| Списання | ✅ | — |
| **Продаж** | ❌ | ❌ |

- **Продаж зі складу-співробітника неможливий** — система видає помилку «Вибраний склад не призначений для продажу».

### Типовий сценарій

1. Перемістити товари зі складу → на склад-співробітника (видача інструментів/матеріалів на об'єкт).
2. Співробітник виконує роботу.
3. Перемістити товари зі складу-співробітника → на загальний склад (повернення).

## 14. Налаштування відображення таблиці продуктів

### Де налаштовується

- **Продукти (список) → Шестерня → Колонки таблиці продуктів**.

### Що можна налаштувати

**Ліва частина (колонки):**
- Галочками вибираються потрібні колонки (стандартні поля та додаткові поля продуктів).
- Порядок колонок змінюється перетягуванням.
- Кожній колонці можна призначити **колір підсвічування** (з готової палітри або власний колір).

**Права частина (додаткові настройки):**
- **«Показувати записів на сторінці»** — мінімум 10, максимум 100.
- **«Сортування за замовчуванням»** — поле та напрямок (від A до Z / від Z до A).
- **«Фіксований заголовок таблиці»** — шапка залишається видимою при вертикальному прокручуванні.
- **«Фіксувати першу колонку при горизонтальній прокрутці»** — перша колонка «прилипає» ліворуч.

### Особисті vs загальні налаштування

- **«Зберегти»** — зберігає тільки для поточного користувача (особисті налаштування).
- **«Зберегти для всіх користувачів системи»** — застосовується для всіх, у кого немає власних особистих налаштувань.
- **«Очистити особисті налаштування»** / **«Очистити всім особисті налаштування»** — скидання.

### Збережені списки

- Можна зберегти поточні налаштування під іменем (кнопка «Зберегти як новий список»).
- Перемикання між збереженими списками — через назву на верхній панелі.
- Видалення списку — хрестик поруч із назвою.

## 15. Налаштування інтерфейсу вікна пошуку продукту

### Де з'являється кнопка «Вибрати/Знайти/Створити продукт»

- На вкладці **«Продукти»** процесу (увімкнюється в налаштуваннях БП → вкладки меню).
- У блоці **«Продукт зі списку»** (додається в інтерфейс процесу).

### Де налаштовується

- **Налаштування системи → Інше → «Налаштування інтерфейсу вікна пошуку продукту»**.

### Блоки/фільтри, що виводяться у вікні пошуку

| Блок | Функція |
|------|---------|
| **Вивести фрейм (iFrame)** | Додає вкладку з URL, задається назва вкладки та URL |
| **Скинути фільтр** | Кнопка «Очистити» — скидає всі активні фільтри |
| **Вибір наявності** | Фільтр: всі / в наявності / не в наявності / в наявності на складі / не в наявності на складі |
| **Додаткові поля продуктів** | Мультивипадаючий список — вибір ДПП для фільтрації |
| **Фільтр товарів** | Вибрати конкретні фільтри (кольори, розміри, матеріали тощо) |
| **Зовнішній код, Артикул, Бренд** | Фільтри за цими полями |
| **Категорії** | Фільтр за категорією; настройка **«Відображати категорії деревом»** |
| **Таблиця продуктів** | Основна таблиця з товарами (детально нижче) |

### Налаштування блоку «Таблиця продуктів»

- **«Відображати тільки продукти згідно доступних списків продуктів»**.
- **«Вибір кількох товарів (тільки для процесів і складських операцій)»** — з'являється чекбокс навпроти кожного товару; потім кнопка «Додати вибрані».
- **«Вибрати кількість (тільки для процесів і складських операцій)»** — з'являється поле введення кількості; працює тільки разом із «Вибором кількох товарів».
- **«Відображати загальну кількість залишків на складах»** — замість розбивки по складах — сумарний залишок.
- **«Відображати ціну продуктів відносно кількості»** — іконка «А» з підказкою: показує ціну при покупці від N штук (із налаштувань цін карточки товару).
- **«Відображати деталізовану інформацію по складських залишках»** → підсписок:
  - **«Відображати вільний залишок продукту на складі»**.
  - **«Відображати баланс розділеними записами»** — баланс у розрізі кожного оприходування (розмір, партія тощо).
  - **«Відображати баланс по складу, вибраному перед переміщенням або списанням»**.
- **Вибір складів** — вибрати склади, інформацію по яких відображати у вікні пошуку.
- **Налаштування відображення більше інформації по складських залишках** — назви полів (зарезервовано, кількість, комірка, вартість, характеристики, вільний залишок, склад) можна перейменувати.

## 16. Налаштування інтерфейсу превʼю продукту

### Що таке превʼю

- Спливаюче вікно, що з'являється при **наведенні курсора на назву товару** у списку продуктів або у блоці пошуку продукту в процесі.
- Мета — швидкий перегляд ключових полів без відкриття карточки.

### Де налаштовується

- **Налаштування системи → Інше → «Налаштування превʼю продукту»**.

### Що можна налаштувати

- Вибір полів для відображення (стандартні + ДПП): відмічаємо потрібні, змінюємо порядок перетягуванням.
- Завжди відображаються назва та категорія (не залежить від налаштувань).
- **Перейменування полів** — нова назва відображається тільки в превʼю (не впливає на карточку).
- **Поле «Зображення»** — відображає основне фото товару.
- **Шаблон для друку штрих-коду** та **шаблон для друку цінника** — при виборі шаблону в превʼю з'являються посилання «Штрих-код» і «Цінник»; натискання відкриває друк за обраним шаблоном.
  - Шаблони беруться з **Налаштування системи → Прочее → «Шаблони цінників»**.

## 17. Автоматичне додавання продуктів у процес

### Дія 1: «Додати продукт у процес»

- Розміщується на **етапі бізнес-процесу** → спрацьовує при переході на етап.
- Налаштування:
  - Вибір одного або кількох товарів з кількістю (кнопка «+ Додати ще продукти до списку»).
  - **Кількість з поля**: кількість береться зі значення конкретного ДПП процесу.
  - **Найменування з поля**: назва береться з ДПП процесу.
  - **«Забороняти додавати продукти, якщо вони вже є в процесі»**:
    - Увімкнено → товар додається тільки один раз.
    - Вимкнено → кожен перехід на етап збільшує кількість (або додає новий рядок).
  - **«Додавати продукт окремим рядком»**: кожен перехід → новий рядок (не збільшення кількості).
  - **«Додавати продукти в батьківський процес»**.
  - **«Не додавати продукти при певному значенні ДПП»**: умова — якщо ДПП = значення → не додавати.

### Дія 2: «Додати продукт у процес на основі додаткового поля процесу»

- Спрацьовує **після редагування процесу** (не при переході на етап).
- Умова: у процесі є ДПП з типом **«Продукт»**; якщо поле заповнено — вказаний товар додається в таблицю продуктів процесу.
- Налаштування:
  - Вибір ДПП з типом «Продукт».
  - **«Видалити всі продукти процесу перед додаванням»**.

### Комбінована схема (умовне додавання)

1. Дія **«Вирахувати і записати значення в ДПП, якщо умова виконується»**: якщо певне поле = 1 → записати ID потрібного товару в ДПП типу «Продукт».
2. Дія **«Додати продукт у процес на основі ДПП»**: бере товар з того ДПП і додає в процес.

→ Таким чином, залежно від виконання умов, у процес автоматично додається потрібний товар.
