# OneBox — Контакти

> Джерело: YouTube-транскрипти OneBox (автоматичні субтитри), дистиляція 2026-06-09

---

## Типи контактів

### Стандартні типи (за замовчуванням)
- **Не визначено** — тип за замовчуванням при створенні
- **Чоловік** (ідентифікатор ключа: `1`)
- **Жінка** (ідентифікатор ключа: `2`)
- **Компанія** (ідентифікатор ключа: `3`)

### Створення власних типів
- Шлях: шестеренка у правому верхньому куті → **Типи контактів**
- Натиснути `+` → вказати **Назву** та обов'язковий **Ключ** (латиницею, без спецсимволів)
- Опціонально: приховати поля ФІО для типу (наприклад, для юридичних осіб), зробити тип прихованим
- Нові типи відразу з'являються у випадаючому списку в картці контакту

### Вплив типу на інтерфейс
- Зміна типу в картці контакту **динамічно перебудовує інтерфейс** без збереження
- Кожному типу налаштовується **окремий інтерфейс** (детальніше — у розділі нижче)
- При імпорті XLS потрібно вказувати **ключ** типу, не назву

## Налаштування інтерфейсу картки контакту

### Пріоритет інтерфейсів (від вищого до нижчого)
1. **Інтерфейс групи** з більшим ідентифікатором (якщо контакт у кількох групах — застосовується інтерфейс групи з більшим ID)
2. **Інтерфейс групи** (будь-якої)
3. **Інтерфейс типу** (чоловік / жінка / компанія тощо)
4. **Інтерфейс за замовчуванням** (тип = «не визначено», не в жодній групі)

### Доступ до налаштувань
- Відкрити картку контакту → шестеренка → **Інтерфейс карток контакту**
- Або: у групах контактів (шестеренка → **Групи контактів**) відкрити групу → іконка **Інтерфейс**
- У рядку адреси: `?id=0` — за замовчуванням; `-1` — чоловік; `-2` — жінка; `-3` — компанія; `-5` і т.д. — власні типи

### Структура редактора інтерфейсу
- Макет складається з **секцій** (обрамлені штриховою лінією) та **блоків** всередині них
- Блоки можна переміщати між секціями (перетягування), видаляти (×), додавати (+)
- Кожен блок має власні **налаштування** (кнопка з шестеренкою на блоці)
- Підтримується налаштування як для **десктопної**, так і для **мобільної** версії (перемикач)
- Після змін обов'язково натиснути **Зберегти**

### Вкладки (таби) в картці контакту
- У налаштуваннях інтерфейсу можна додавати **таби** (вкладки): `+` → вказати назву та сортування
- Сортування визначає порядок відображення табів
- Кожному табу — окрема сітка з блоками
- Таб може виконувати **дію** при натисканні (наприклад, перехід до розділу системи) — замість блоків вказується цільова URL-дія

### Інтерфейс для груп
- Для кожної групи налаштовується окремий інтерфейс через розділ **Групи контактів**
- Можна **скопіювати** інтерфейс з іншої групи (доступні лише групи з вже налаштованим інтерфейсом)
- Якщо в групи немає власного інтерфейсу — застосовується інтерфейс за замовчуванням

### Комбінування групи та типу
- У налаштуваннях блоку можна вказати **«Показувати тільки для типу»** (наприклад, «чоловік»)
- Це дозволяє в межах одного інтерфейсу групи показувати різні блоки залежно від типу контакту

## Налаштування інтерфейсу вікна додавання контакту

### Де знаходиться
- **Варіант 1:** Глобальне меню → Налаштування системи → Прочее → пошук «добавление контакта»
- **Варіант 2:** Додаток «Контакти» → шестеренка → **Налаштування інтерфейсу додавання контакту**

### Вікно додавання контакту
Відкривається при введенні клієнта в процес та натисканні «Додати новий контакт».

### Стандартні поля та їх порядок (незмінний для стандартних полів)
Фіксований порядок: Стать → ФІО → Компанія → Посада → Телефон → Email → Skype/Чат → Адреса (Країна, Область, Район, Місто, Вулиця, Будинок, Квартира) → Джерело → Група

Стандартні поля не можна перемістити за межі відведеного місця — порядок зарезервований.

### Додаткові поля
- Відображаються у вікні тільки якщо в налаштуваннях самого поля встановлено галочку **«Відображати у вікні додавання користувача»**
- Порядок додаткових полів можна змінювати перетягуванням у налаштуваннях інтерфейсу

### Ключові налаштування
| Налаштування | Опис |
|---|---|
| **Обов'язкові до заповнення** | Вибір полів мульті-списком; система не збережеться без заповнення виділених полів |
| **Відображати тільки для типу** | Поле з'являється лише для вказаного типу контакту (наприклад, «Компанія») |
| **Дозволити без контактних даних** | У додатку «Контакти» → шестеренка → Прочие налаштування → «Дозволити додавання користувача без контактних даних» |

### Логіка валідації
- За замовчуванням система вимагає **телефон або email**
- Якщо включено «Дозволити без контактних даних» — перевірка знімається, якщо тільки поле не зроблено обов'язковим вручну
- При незаповнених обов'язкових полях — помилка без перезавантаження сторінки (введені дані зберігаються)

## Блок «Контакт у довіднику» в картці контакту

### Призначення
Виводить рядки з **довідника** (конструктор об'єктів), де поточний контакт згадується у полі типу «Мульти-список контактів».

**Приклад:** довідник «Договори» з колонками Номер договору, Поручитель, Борг, Відсоток, Клієнт. Поле «Клієнт» має тип «Мульти-список контактів». Блок виводить всі рядки, де поточний контакт є клієнтом.

### Умова роботи
У довіднику **обов'язково** повинно бути хоча б одне поле з типом **«Мульти-список контактів»** — по ньому система знаходить пов'язані рядки.

### Налаштування блоку (ліва панель)
| Параметр | Опис |
|---|---|
| **Довідник** | Вибір довідника з конструктора об'єктів |
| **Поля довідника для відображення** | Мульти-список колонок, які виводяться в блоці |
| **Поле для пошуку контактів** | Поле типу «Мульти-список контактів» — по ньому фільтруються рядки |
| **Поля, доступні для редагування** | Дозволяє редагувати значення прямо з блоку, не відкриваючи довідник |

### Налаштування блоку (права панель)
| Параметр | Опис |
|---|---|
| **Назва блоку** | Довільний заголовок, наприклад «Договори» |
| **Кількість записів для відображення** | Число рядків видимих спочатку; решта — по кнопці «Завантажити ще» |
| **Показувати блок для** | Мульти-список груп контактів; блок видимий лише контактам з вказаних груп |

### Скільки блоків можна додати
Таких блоків у картці контакту можна додати **необмежену кількість** — по одному для кожного потрібного довідника.

## Вкладка «Історія» в картці контакту

### Вимога: встановити застосунок
Перед роботою необхідно встановити застосунок **«Логи користувачів»** (шукати «лог» у каталозі застосунків OneBox OS).

### Відкриття історії
Відкрити картку контакту → іконка «Відправити в» → **Логи користувачів** → «Показати логи»

### Що показує таблиця логів
| Колонка | Зміст |
|---|---|
| **Клієнт** | ФІО контакту |
| **Автор** | Хто вносив зміни (порожньо — автоматична зміна системою/автоматизацією) |
| **Дата і час** | Коли внесено зміну |
| **Поле** | Яке поле змінилося |
| **Старе значення** | Значення до зміни |
| **Нове значення** | Значення після зміни |
| **URL** | З якого розділу/сторінки зроблено зміну |

### Фільтрація в логах
- **За полем** — показати зміни тільки по конкретному полю (наприклад, email)
- **За датою** (від / до)
- **За автором** — зміни конкретного співробітника
- **Показувати системні зміни** — зміни від автоматизацій, імпортів (колонка «Автор» порожня)

### Налаштування відображення таблиці логів
Шестеренка (права частина) → **Колонки таблиці історії змін контактів**:
- Включити/виключити колонки галочками
- Змінити порядок колонок перетягуванням
- Вказати кількість записів на сторінці
- Вказати сортування

### У яких налаштуваннях вимкнути запис логів
Шестеренка → **Прочі налаштування застосунку** → можна вимкнути запис для: категорій продуктів, контактів, продуктів, процесів.

## Панель фільтрів контактів

### Відкриття
Додаток «Контакти» → всі контакти → кнопка **«Фільтрувати»** (ліворуч угорі)

### Можливості фільтрації
- Фільтри комбінуються між собою (AND-логіка)
- Доступні фільтри: тип, група, дата створення, підписка на розсилку та інші
- Результати оновлюються після натискання кнопки **«Фільтрувати»**

### Налаштування панелі фільтрів
Шлях: шестеренка → **Панель фільтрації контактів**
- **Видалити** непотрібний фільтр: натиснути × на блоці
- **Додати** фільтр: кнопка `+` → вибрати зі списку
- **Змінити порядок**: перетягнути блок фільтра на нове місце
- Після змін — **Зберегти**

### Додаткові поля як фільтри
- Блок «Додаткове поле контакту» → у налаштуваннях блоку вибрати конкретне поле
- Доступні опції: розбивати значення по крапці з комою, дозволити вибір кількох значень (для випадаючих списків)

### Налаштування для різних ролей
- Вгорі є поле вибору **Ролі** (за замовчуванням — «Всі ролі»)
- Для кожної ролі можна налаштувати **окрему** панель фільтрів

### Збережені фільтри
- Після фільтрації → кнопка **«Зберегти фільтр»** внизу → вказати назву
- Збережені фільтри відображаються в нижній частині панелі фільтрів
- Вибір збереженого фільтру — застосовується миттєво без натискання «Фільтрувати»
- Збережені фільтри **особисті**: в іншого менеджера своїх збережених фільтрів немає
- Редагування/видалення: кнопка «Редагувати» → галочка «Видалити» → Зберегти

## Унікальні поля контактів

### Стандартні унікальні поля (за замовчуванням)
| Поле | Примітка |
|---|---|
| **Телефон** | Дублювання неможливе; можна вимкнути |
| **Email** | Дублювання неможливе; можна вимкнути |
| **Назва компанії** | Дві компанії з однаковою назвою — неможливо |
| **Зовнішній ID** | Для зв'язку з ID з інших систем |

### Де вимкнути унікальність телефону/email
Налаштування системи → Прочее → пошук «дублів» → **Налаштування оформлення таблиць бізнес-процесів**:
- «Дозволити дублікати email»
- «Дозволити дублікати телефонів»

### Унікальність у додаткових полях контактів
При створенні додаткового поля є галочка **«Тільки унікальне значення»** — система не дасть двом контактам мати однакове значення цього поля.
