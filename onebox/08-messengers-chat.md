## Viber через сервіс E-chat

- **Що це:** Компонент-додаток `echat` — інтеграція з Viber через сторонній сервіс echat.
- **Ключова перевага:** Підключається звичайний номер телефону (не публічний акаунт); можна писати клієнту **першим**.
- **Що дає інтеграція:**
  - Відправка та отримання Viber-повідомлень з OneBox.
  - Перетворення вхідного повідомлення на чат або процес (автоматична дія раз на хвилину).
  - Масова розсилка Viber-повідомлень по вибраних контактах.
  - Відправка файлів та зображень через чат/задачу.
  - Відправка повідомлень через дію на етапі бізнес-процесу.
  - Перегляд історії повідомлень у розділі `Події`.
- **Налаштування:**
  1. Встановити додаток `echat` в OneBox.
  2. У особистому кабінеті echat підключити інтеграцію `inbox`, вказати номер телефону в міжнародному форматі → натиснути `Додати`.
  3. Відкрити Viber на смартфоні (з потрібним номером) → відсканувати QR-код у кабінеті echat.
  4. Після підключення скопіювати **API ключ** та **номер телефону** з кабінету echat.
  5. У налаштуваннях додатку OneBox (шестерня → `Налаштувати інтеграцію`) вставити ключ і номер → `Підключити`.
- **Автоматизація:**
  - `Перетворити повідомлення в процес` — вказати БП, статус, відповідального.
  - `Перетворити повідомлення в чат` — вказати на кого створювати чат та ролі учасників.
  - ⚠️ Потребує перевірки: для використання чатів необхідно окремо встановити додаток `Чат`.

## Facebook Messenger

- **Що це:** Вбудована інтеграція — підключається напряму без сторонніх сервісів, без додаткової верифікації.
- **Що дає інтеграція:**
  - Миттєве отримання вхідних повідомлень (без затримок).
  - Відповідь через: додаток Facebook Messenger, задачу, чат, блок коментарів у процесі, агрегатор месенджерів (мультичат).
  - Перетворення вхідних повідомлень на чати або процеси.
  - Підтримка смайлів; обмін файлами.
  - Відстеження статусу прочитання повідомлення.
  - Отримання аватару, імені акаунту та ID користувача Facebook.
  - Відправка повідомлень через дію на етапі БП.
- **Налаштування:**
  1. Встановити додаток `Facebook Messenger` в OneBox.
  2. Переконатися, що у браузері дозволені спливаючі вікна для сайту.
  3. У додатку → шестерня → `Налаштувати підключення` → `Додати сторінку`.
  4. Авторизуватися у Facebook → вибрати потрібну бізнес-сторінку → надати всі дозволи → `Готово` → `OK`.
  5. Оновити сторінку — інтеграція активна.
  - Можна підключити кілька Facebook-сторінок.
- **Автоматизація:**
  - `Перетворити спілкування Facebook Messenger в процес`.
  - `Перетворити повідомлення в Facebook Messenger в чат`.
- **Агрегатор месенджерів (мультичат):** окремий додаток `Агрегатор Messenger multichat` — блок `Загальний чат клієнта` у процесі, підтримка смайлів, індикатор прочитання.

## Instagram Direct

- **Що це:** Вбудована інтеграція — без сторонніх сервісів, підключається за кілька кліків.
- **Що дає інтеграція:**
  - Миттєве отримання та відправка Direct-повідомлень.
  - Імпорт голосових повідомлень, зображень.
  - Отримання Stories, в яких відмічена ваша сторінка.
  - Видалення скасованих повідомлень у реальному часі (синхронізується з OneBox).
  - Перетворення вхідних повідомлень на чати або процеси.
  - Відправка повідомлень через дію на етапі БП.
  - Автоматичне створення або пошук картки контакту за ідентифікатором Instagram Direct.
- **Налаштування:**
  1. Встановити додаток `Instagram Direct` в OneBox.
  2. В Instagram (з телефону): `Налаштування → Конфіденційність → Повідомлення → Увімкнути доступ до повідомлень`.
  3. Переконатися, що спливаючі вікна дозволені у браузері.
  4. У додатку → шестерня → `Налаштування застосунку` → `+` (Додати сторінку) → авторизація через Facebook → вибрати бізнес-акаунт Instagram → надати всі 4 дозволи → `Готово` → оновити сторінку.
  - Можна підключити кілька Instagram-сторінок.
- **Налаштування поля для пошуку контакту:** шестерня → `Інші налаштування застосунку` → вибрати `Додаткове поле контакту для пошуку клієнта за назвою акаунту Instagram`.
- **Автоматизація (Автоматика раз на хвилину):**
  - `Перетворити повідомлення Instagram Direct в чат`.
  - `Перетворити повідомлення Instagram Direct в процес` — аналогічні налаштування: БП, статус, відповідальний, джерело.

## Telegram: Конструктор бота

- **Що це:** Додаток `Конструктор Telegram Bot` — візуальний no-code конструктор для побудови Telegram-ботів, пов'язаних з процесами OneBox.
- **Що дає:**
  - Побудова інтерактивного бота з кнопками та переходами між блоками.
  - Прийом та обробка замовлень (аналог інтернет-магазину через Telegram).
  - Відображення каталогу товарів за списками продуктів.
  - Відображення списку замовлень клієнта та інформації про замовлення.
  - Перемикання статусу бізнес-процесу безпосередньо з бота.
  - Додавання коментарів до замовлення через бота.
  - Оформлення замовлення з заповненням полів процесу (ФІО, телефон, спосіб доставки, адреса).
- **Налаштування:**
  1. Встановити додаток `Конструктор Telegram Bot` з сайту `1b.f`.
  2. Створити бота через `@BotFather` у Telegram: `New Bot` → задати ім'я та username (закінчується на `_bot` або `bot`) → отримати **API ключ**.
  3. У OneBox → додаток → `+` → вказати назву, тип `Telegram`, вставити API ключ → `Додати бота`.
  4. Перейти в `Налаштувати` → побудувати схему бота з блоків.
- **Типи блоків конструктора:**
  - `Текстовий блок` — відображає текст + кнопки для переходів.
  - `Product List` — список продуктів з обраного списку товарів.
  - `Order List` — список замовлень клієнта з обраного БП.
  - `Order View` — детальна інформація про замовлення (з підтримкою змінних, кнопок зміни статусу та додавання коментарів).
  - `My Order` (Create Order) — блок оформлення замовлення; кожна кнопка = одне поле для заповнення (ФІО, телефон, спосіб доставки тощо).
  - `Навігація` — нижнє меню (завжди видиме); один блок на бота.
  - `Паттерн` — набір кнопок для переходів.
- **Логіка побудови:** блоки з'єднуються стрілками від кнопок (аналогічно до побудови БП). Один блок — стартовий. Доступні для переключення статуси — тільки ті, де в БП увімкнено галочку `Доступен для переключения в кабинете`.

## Telegram: Перетворення повідомлень у чати

- **Що це:** Функція перетворення вхідних Telegram-повідомлень на **Єдині чати** OneBox (а не лише на процеси).
- **Вимоги:**
  - Встановлений SSL-сертифікат (HTTPS) на сервері OneBox.
  - Підключена інтеграція з Telegram (в OneBox Market: вказати токен бота).
  - У налаштуваннях інтеграції Telegram увімкнути галочку **«Використовувати webhook для отримання повідомлень»** (без HTTPS галочка не збережеться).
  - Увімкнути системну настройку **«Дозволити додавання користувачів без контактних даних»** (Налаштування системи → Прочее), оскільки з Telegram приходить лише нік без телефону/email.
- **Налаштування автоматики:**
  - Налаштування системи → Автоматизація → Автоматичні дії раз на хвилину → `Перетворити повідомлення Telegram в чат`.
  - Параметри дії: вибрати `Telegram токен`; вказати `На кого створювати чат`; за потреби — `Додати учасників вибраних ролей у чат`.
- **Можливості чату:**
  - Миттєвий обмін повідомленнями (webhook, без затримок).
  - Відмічення (`@`) сотрудників у чаті.
  - Прикріплення файлів і зображень.
  - Шаблонні коментарі.
  - Смайли, теги для вставки коду.
  - `Перетворити в процес` — вся переписка переноситься до Ленти коментарів нового процесу. Чат після цього зникає зі списку.
- **Сценарій використання:** клієнт пише в бот → створюється чат (не процес) → консультація → якщо клієнт готовий до угоди → перетворити чат у процес потрібного типу.

## Перетворення листів у чати (Email → Chat)

- **Що це:** Функція створення чату на основі вхідного email-листа.
- **Вимоги:** Підключена інтеграція з поштою (iMAP/SMTP).
- **Налаштування:**
  1. OneBox Market → `Інтеграція з email` → вказати: `email`, `IMAP Host`, `IMAP Port`, `IMAP username`, `Password`, `SMTP`-параметри; галочка `Активна`.
  2. Налаштування системи → Автоматизація → Автоматичні дії раз на хвилину → `Перетворити лист у чат`.
  3. Параметри дії: `Email одержувача` (яку поштову скриньку парсити); `На кого створювати чат` (відповідальний співробітник).
  - Для кількох поштових скриньок — кілька окремих дій.
- **Як виглядає чат:**
  - В `Єдиному чаті` з'являється чат, у назві — тема листа.
  - Іконка поштового конверту поруч із назвою.
  - Учасники: клієнт (відправник листа) + відповідальний співробітник.
  - Відповідь з чату → відправляється саме як лист (email), а не коментар.
- **Перетворення в процес:** кнопка `Перетворити в процес` → вся переписка зберігається в Ленті коментарів нового процесу.

## Вбудований поштовий клієнт

- **Що це:** Вбудований модуль роботи з поштою (IMAP/SMTP), інтегрований з контактами та процесами OneBox.
- **Можливості:**
  - Повний функціонал поштового клієнту: написати, відповісти, переслати, позначити як прочитане.
  - Автоматичне створення картки контакту для кожного нового листа.
  - Відправка документів та файлів зі сховища OneBox як вкладень.
  - Планування відправки листа на конкретну дату/час.
  - Збереження відправлених листів у розділі `Події`.
- **Підключення пошти:** Додаток `Пошта` → шестерня → `Налаштування підключення до поштових скриньок` → вибрати сервіс (Gmail, ukr.net, корпоративна пошта тощо) → вказати email, пароль, IMAP/SMTP параметри.

## Відправка email з процесів

- **Три способи відправки:**

### 1. Блок `Лента коментарів`
- Натиснути `Написати email` у блоці.
- Email клієнта процесу підтягується автоматично; можна вказати копію (CC).
- Тема листа за замовчуванням = `Назва процесу #ID`.
- Фіксований обратний адрес email налаштовується в `Налаштування інтерфейсу → Лента коментарів → Обратный адрес для писем`.

### 2. Блок інтерфейсу `Написати лист`
- Окремий блок, що додається до інтерфейсу процесу.
- Поля: email одержувача, тема, відправник, дата відправки, HTML-оформлення, вкладення.

### 3. Дія на етапі БП: `Відправити сповіщення по email`
- Автоматична відправка при переході на етап.
- Параметри: одержувачі (клієнт, відповідальний, автор, менеджер клієнта, конкретний співробітник — мультивибір); обратний email; тема; текст з підтримкою змінних (`{order_id}` = ID процесу, `{order_client_name}` = ФІО клієнта тощо); затримка відправки (через N хвилин/годин); часове вікно відправки.
- ⚠️ Email-адреса у полі `Обратный адрес` повинна бути підключена в системі (інтеграція з поштою або сервісом розсилок).
