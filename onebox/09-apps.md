# OneBox — Застосунки (apps)

> Джерело: YouTube-транскрипти OneBox (автоматичні субтитри), дистиляція 2026-06-09

OneBox OS побудований за модульним принципом: функціональність надається через **застосунки (apps)**, які встановлюються окремо з каталогу `1b.ua`. За замовчуванням встановлено базовий набір застосунків; решту підключають за потреби.

---

## Встановлення застосунків

### Встановлення на хмарний OneBox OS

1. Перейдіть на сайт `1b.ua` → розділ **Застосунки**.
2. Знайдіть потрібний застосунок (пошук або фільтр за категоріями).
3. Відкрийте картку застосунку → натисніть **Встановити** → **Встановити в хмарі**.
4. У спливаючому вікні оберіть свій хмарний OneBox OS → **Встановити**.
5. Дочекайтесь завершення; після встановлення з'явиться повідомлення про успіх.
6. Застосунок стане доступним у списку всіх застосунків (робочий стіл).

### Налаштування застосунків

- Кожен застосунок має власні налаштування за іконкою **шестерні** у правому верхньому куті.
- Типові сторінки налаштувань: акаунти, категорії, відображені поля, права доступу тощо.
- Пункт **Інші налаштування застосунку** — загальні параметри, що впливають на роботу конкретного застосунку.
- Після будь-яких змін обов'язково натискати кнопку **Зберегти**.

### Права доступу до застосунків

- Для доступу співробітника до застосунку або посилання-Links необхідно відкрити застосунок **Права доступу**, обрати групу і дозволити відповідний пункт.

## Огляд ключових застосунків

### CRM

**Призначення:** Управління угодами у вигляді канбан-дошки зі стовпцями-статусами. Ідеально підходить для відділів продажів та проєктів із лінійною воронкою.

**Ключові можливості:**
- Перегляд процесів по колонках (один стовпець = один етап бізнес-процесу); перетягування картки між етапами.
- Налаштування відображуваних бізнес-процесів: шестерня → **Відображувані процеси** → обрати БП та статуси.
- Створення нового процесу прямо з потрібного етапу: навести мишу на етап → **Запустити процес**.
  - Якщо процес створюється не на стартовому статусі — у форму створення обов'язково додати поле **Статус**.
- Налаштування відображуваних полів для кожного БП: шестерня → **Налаштування відображуваних полів**.
- За замовчуванням показуються тільки процеси, де поточний користувач є відповідальним. Фільтр **Відповідальний** дозволяє переглядати процеси інших менеджерів.

---

### GTD Calendar

**Призначення:** Комбінація списку справ і класичного календаря — кожен день є окремим списком завдань. Підходить для планування задач на день без прив'язки до конкретного часу.

**Ключові можливості:**
- Режими відображення: **день / тиждень / місяць** — переключаються вручну.
- На відміну від CRM, кожна колонка відповідає не статусу, а **дню** (дні тижня або дні місяця).
- Налаштування відображуваних БП і полів: шестерня → **Відображувані процеси**.
- Створення процесу на конкретну дату: навести на потрібний день → **Додати задачу**. Якщо дата не поточна — у форму створення обов'язково додати поле **Запланувати**.
- Перенесення процесу на інший день — перетягуванням у потрібну колонку (змінює поле «Запланувати»).
- Фільтрація за відповідальним дозволяє переглядати задачі інших менеджерів.

---

### Документи

**Призначення:** Формування документів за шаблонами з автоматичною підстановкою полів із замовлень/процесів. Результат — HTML-файл, що редагується прямо у OneBox.

**Ключові можливості:**
- Шаблон документа створюється через шестерня → **Шаблони документів** → **+**. Вкладка **Редактор** — візуальний WYSIWYG-редактор; вкладка **Перегляд** — назва та прив'язка.
- Підтримуються **змінні** (перетягуються з лівої панелі в текст): підставляють дані процесу (номер, клієнт, сума тощо). Змінні типу `row:*` (для рядків продуктів) обов'язково розміщати всередині таблиць.
- Прив'язка шаблону: до процесу, до складської операції або до комірки складу.
- Готові шаблони (рахунок-фактура тощо) — встановлюються окремим застосунком **Колекція шаблонів документів**.
- Формування документа з процесу: у замовленні → **Відправити в** → **Документи** → **Створити документ** → обрати шаблон → **Зберегти**.
- Документ можна надрукувати або зберегти як PDF.
- Перегляд усіх документів процесу: **Відправити в** → **Документи** → **Знайти документи**.

### Воронка

**Призначення:** Аналітичний інструмент. Відображає шлях клієнта від першого контакту до купівлі — кількість і суму процесів на кожному етапі БП.

**Ключові можливості:**
- Встановлюється як окремий застосунок; відображає окрему воронку для кожного бізнес-процесу.
- Кожен сегмент воронки = один етап БП. Послідовність відповідає порядку в конструкторі Workflows.
- Поруч з назвою кожного етапу — **сума** і **кількість** процесів на ньому.
- Панель фільтрів: фільтрація за менеджером, контактом, датою створення тощо — для поглибленого аналізу.
- Дозволяє бачити, на якому етапі відсіюється найбільше клієнтів.

### Глобальний пошук

**Призначення:** Швидкий пошук по всіх сутностях системи (контакти, процеси, задачі тощо) з одного рядка введення.

**Ключові можливості:**
- Пошук по імені, номеру телефону, email та іншим налаштованим полям.
- Результати охоплюють кілька типів сутностей одночасно (контакти + процеси + ...).
- Налаштування застосунків-джерел для пошуку: шестерня → **Застосунки, що беруть участь у пошуку** → додати потрібні (задачі, ліди, проєкти тощо) → **Зберегти**.
- Тонке налаштування полів для пошуку за кожним типом сутностей: поля по яких шукати (email, телефон, посада тощо) задаються окремо для кожного застосунку.
- Перевага фільтрів панелі: комбінування кількох умов. Перевага глобального пошуку: швидкість, не потрібно відкривати окремий застосунок.

### Links

**Призначення:** Створення «віртуальних застосунків» — ярликів (посилань) на робочому столі OneBox. Зручно для збереження відфільтрованих списків, зовнішніх сервісів, корпоративних чатів тощо.

**Ключові можливості:**
- Кожен Links = назва + URL + (опційно) іконка. Після створення посилання з'являється у списку застосунків для всіх (або обраних) користувачів.
- Іконку можна обрати з вбудованого каталогу або завантажити власний файл.
- Видалення посилання: відкрити Links → поставити галочку → **Видалити**.
- **Важливо:** для доступу співробітника до конкретного Links — додати дозвіл у застосунку **Права доступу**. Якщо у користувача немає прав на бізнес-процес, що є цільовою сторінкою посилання — дані не відображатимуться (навіть якщо URL доступний).

### Workflows

**Призначення:** Додаток для **створення та редагування бізнес-процесів** у вигляді графічних блок-схем. Є основою всієї архітектури OneBox — всі задачі, замовлення, ліди, проєкти — це бізнес-процеси.

**Ключові можливості:**
- Список усіх БП, згрупованих по групах (Задачі, Замовлення, Ліди, Проєкти тощо). Кожна група = окремий застосунок у списку.
- Кожен БП — блок-схема: **етапи** (прямокутники) з'єднані **стрілками** (переходи між статусами).
- Редагування: перейменування етапів, додавання/видалення, проведення або видалення стрілок, задання кольорів етапам. Зміни зберігаються автоматично.
- Стартовий етап — обов'язковий, не можна видалити, лише перейменувати.
- **Автоматика на етапах:** при переході на кожен етап можна налаштувати дії (відправити SMS/email, списати товар зі складу, виписати документ, створити підпроцес тощо). Дії додаються через клік на етап → **Дії** → **+**.
- Група БП = окремий застосунок: процеси групи «Замовлення» відображаються у застосунку «Замовлення», групи «Задачі» — у застосунку «Задачі» тощо.
- Підтримується імпорт/експорт БП з файлу.

### Стікери

**Призначення:** Додавання візуальних стікерів (зображень) у коментарях до процесів і карток контактів. Використовується для корпоративного спілкування або як візуальний сигнал у автоматиці.

**Ключові можливості:**
- Після встановлення застосунку **Стікери** — завантажити зображення: застосунок → **+** (у правому куті) → вибрати файли з комп'ютера → **Зберегти**.
- Підтримувані формати: **GIF, JPEG, PNG**. Рекомендується завантажувати зображення великого розміру для чіткості.
- Стікери доступні в будь-якому місці, де можна залишити коментар: у процесах (замовленнях, задачах), у чатах, у блоці «Нотатка» карточки контакту.
- Використання в автоматиці: у дії **Додати коментар до процесу** вказати ідентифікатор стікера — при переході на потрібний етап БП стікер додається автоматично.
- Ідентифікатор стікера з'являється у полі введення після вибору стікера (перед збереженням).
- Видалення стікерів: поставити галочку → **Видалити**.

## Взаємодія застосунків

- Застосунки можуть **відправляти сутності один одному**: наприклад, замовлення → Документи (сформувати документ), замовлення → Платежі (провести платіж), процес → Очікувані платежі тощо. Це відбувається через кнопку **Відправити в** всередині запису.
- Кількість застосунків у каталозі `1b.ua` — понад 300+, включно з **галузевими рішеннями** (конфігурації, готові під конкретну нішу: автосервіс, нерухомість, е-commerce тощо). ⚠️ потребує перевірки актуальної кількості.
- Партнери можуть створювати власні застосунки та відраслеві рішення і публікувати їх у каталозі.
- Нові функції та інтеграції (GPS-відстеження, Instagram Direct, е-документообіг тощо) виходять тільки у OneBox OS — попередні версії не оновлюються.
