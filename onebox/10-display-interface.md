# OneBox — Способи відображення та інтерфейс

> Джерело: YouTube-транскрипти OneBox (автоматичні субтитри), дистиляція 2026-06-09

---

## Огляд способів відображення

OneBox дозволяє вибрати спосіб відображення **окремо** для кожного типу бізнес-процесів (заказів, задач, проектів, лідів тощо). Перемикання — у верхньому лівому куті розділу.

---

## Воронка продажів

**Призначення:** аналітика конверсії — відображає, скільки процесів пройшло через кожен етап, де відбуваються «втрати».

### Що показується
- Список бізнес-процесів, згрупованих по воронках (кожен бізнес-процес — окрема воронка)
- По кожному етапу: кількість процесів + загальна сума
- Якщо по бізнес-процесу немає даних — воронка пуста

### Поведінка
- Стартовий етап = 100 % вхідних процесів
- Кожен наступний етап — менше процесів (ті що відсіялись по шляху)
- Відображає кількість, а не лише поточний стан (процеси, що «пройшли через» етап)

### Налаштування видимості етапів
Кожна воронка має кнопку налаштувань (права сторона). При натисканні — чекбокси з переліком етапів: можна **приховати** непотрібні етапи.

### Панель фільтрів
Стандартна панель фільтрів (вибір бізнес-процесу, дата тощо) — відображаються воронки лише вибраних бізнес-процесів.

**Коли застосовувати:** аналіз якості продаж, пошук «вузьких місць» у воронці.

---

## Обернена воронка

**Призначення:** визначити, скільки процесів потрібно запустити на стартовому етапі, щоб отримати заданий результат на завершальному.

### Де знаходиться
Глобальне меню → **Маркетинг** → **Обернена воронка**

### Три кроки налаштування
1. Вибрати **бізнес-процес** і **завершальний етап** (можна кілька)
2. Вказати **інтервал дат** створення процесів
3. Вказати **потрібну кількість** процесів на завершальному етапі

### Результат
Система розраховує (на основі наявної статистики): скільки процесів потрібно на стартовому етапі для досягнення потрібної кількості на завершальному.

**Приклад:** якщо для 1 закритого замовлення потрібно 8 на стартовому — конверсія 12,5 %.

**Коли застосовувати:** планування обсягу лідів / заявок для досягнення планового результату.

## Таблиця контактів

### Два підтипи

#### «Таблиця» (стандартна)
- Виглядає схоже на «Список», але є ключова відмінність
- **Редагування полів прямо зі списку**: подвійний клік на білій ячейці → ввести значення → Enter або клік на іншій ячейці
- Сірі ячейки — заблоковані для редагування (системні поля)
- Налаштування колонок: правий клік на шапці → налаштування відображення таблиці

#### «Класична таблиця»
- Таблиця з чіткими межами між ячейками
- **Не підтримує** редагування полів безпосередньо зі списку
- Додаткові налаштування відображення:
  - Колір меж (від світлого до чорного)
  - Розмір шрифту
  - Вертикальне вирівнювання тексту (верх / низ / по центру)
  - Горизонтальне вирівнювання тексту (ліво / право / центр)

### Загальне для обох підтипів
- Налаштування списку колонок: галочками включити/виключити, перетягнути для зміни порядку, швидкий пошук поля
- Показники «Фіксований заголовок» та «Фіксувати першу колонку» — **не працюють** в режимі «таблиця» (тільки в «списку»)

## Канбан

**Призначення:** візуальне розподілення процесів по етапах бізнес-процесу; виявлення «затримок» на певних етапах.

### Відображення
- Колонки = етапи бізнес-процесу
- Картка = процес
- Над кожним етапом: кількість процесів і % від загальної кількості
- Закриті процеси за замовчуванням приховані; кнопка **«Відобразити закриті»** показує їх

### Обов'язковий крок
Для відображення процесів **необхідно вибрати бізнес-процес** у панелі фільтрів.

### Дії прямо з Канбану
- **Перетягування** процесу між колонками = зміна етапу
- **Дзвінок / лист клієнту** — іконки на картці (якщо є клієнт з телефоном/email)
- **Створення нового процесу** на конкретному етапі: навести на колонку → «Додати процес»; при створенні можна змінити бізнес-процес і статус

### Поля, що відображаються на картках
Налаштування системи → Бізнес-процеси → вибрати БП → **Інтерфейс у календарі** → «Показувати поля у відображенні Канбан» — вибрати мульті-списком.

### Управління видимістю колонок
Кнопка шестеренки поруч з назвою бізнес-процесу → відзначити чекбокси потрібних етапів → **Зберегти**.

**Коли застосовувати:** оперативне управління задачами, Scrum/Agile-підходи, продажі.

## Діаграма Ганта

**Призначення:** візуалізація плану-графіку проекту: ієрархія процесів, терміни виконання, відсоток виконання.

### Структура відображення
- **Ліва частина:** дерево процесів (батьківський → підпроцеси)
- **Права частина:** часова шкала з відрізками для кожного процесу
- Колір відрізку = колір поточного статусу процесу
- Червона шкала = відсоток виконання (з додаткового поля)

### Два режими відображення дат
| Режим | Налаштування |
|---|---|
| **Від «Запланувати» до «Виконати до»** | Галочка активна (за замовчуванням рекомендований) |
| **Від дати створення до «Запланувати»** | Галочка вимкнена |

Шлях: Налаштування системи → Прочее → пошук «gant» → блок **«Зовнішній вигляд»**.

### Налаштування способу відображення
Налаштування системи → Прочее → **«Налаштування способів відображення процесів»** → вибрати тип БП → Ганта:

| Налаштування | Опис |
|---|---|
| **Додаткові поля процесу** | Вибрати поля для відображення в лівій колонці |
| **Поле для відображення у %** | Вибір поля (числове), значення якого відображається червоною шкалою |
| **Додатково виводити інформацію** | Ще поля в лівій частині (наприклад, Відповідальний) |
| **Відображати в кольорі статусу** | Забарвлення відрізку за кольором поточного етапу |
| **Не виходити за межі дати батьківського процесу** | Підпроцес не може мати дату пізніше батьківського |
| **Заблокувати переміщення на діаграмі** | Забороняє перетягування дат мишкою |

### Пропорційне зміщення дат підпроцесів
Налаштування системи → Прочее → «gant» → **«При зміні дати пропорційно змінювати дати підпроцесів»**:
- **Включено:** підпроцеси зміщуються пропорційно (наприклад, батьківський +2 дні → підпроцеси +2 дні)
- **Вимкнено:** підпроцеси отримують **точно таку ж** дату, що й батьківський

### Дії з діаграмою
- Перетягування відрізку змінює дату «Запланувати» або «Виконати до» процесу (якщо режим від «Запланувати»)
- При активних підпроцесах зміщення залежить від налаштування вище

**Коли застосовувати:** планування проектів, відстеження дедлайнів, управління ресурсами.

## Відображення по часу

**Призначення:** розподіл процесів на денній/тижневій сітці по запланованому часу; зручне планування робочого часу.

### Зовнішній вигляд
- Рядки = часові слоти (інтервал налаштовується)
- Колонки = дні тижня (поточний день або тиждень)
- Кожен процес = блок у відповідному часовому слоті (за датою «Запланувати»)

### Налаштування
Налаштування системи → **Бізнес-процеси** → вибрати БП → **Інтерфейс у календарі**:

| Параметр | Опис |
|---|---|
| **Показувати у відображенні по часу** | Мульті-список полів на картці процесу |
| **Інтервал часу** | Початок і кінець робочого дня (наприклад, 9:00–18:00) |
| **Крок** | 60, 30, 10 хвилин (за замовчуванням 60 хв) |
| **Поле для редагування** | Одне додаткове поле, яке можна редагувати прямо зі списку |

⚠️ потребує перевірки: у відео зазначено, що налаштування інтервалу часу є «наскрізним» для всіх бізнес-процесів, але при зміні в одному БП — змінюється і в інших.

### Відображення по сотрудниках
У розділах задач (і будь-якому іншому типі БП) можна бачити навантаження кожного співробітника окремо — зручно для планування нарад, груп, зустрічей.

**Коли застосовувати:** планування зустрічей, контроль навантаження менеджерів, scheduling.

## Сітка (Grid)

**Призначення:** відображення процесів у вигляді сітки по годинах за день або тиждень — аналог calendar-view.

### Відмінність від «По часу»
- Візуально схожий, але більш «квадратний» — кожен слот в сітці
- Відображення за **один день** або за **тиждень** (перемикач)
- Можна побачити вільні «вікна» дня та запланувати в них нові процеси

### Вивід інформації на блоках
Налаштування системи → Бізнес-процеси → вибрати БП → Інтерфейс у календарі → **«Показувати в сітці»** — вибрати поля мульті-списком.

### Дії
- Клік на `+` у слоті → форма створення процесу (дата/час підставляються автоматично)
- Клік на процес → перехід у картку процесу
- Панель фільтрів працює (вибір бізнес-процесу, відповідального тощо)

**Коли застосовувати:** денний/тижневий розклад, контроль завантаженості команди.

## Таблиця процесів

### Два підтипи

#### «Таблиця» (стандартна)
- Редагування полів безпосередньо зі списку: клік на білу ячейку → ввести/вибрати значення
- Можна міняти статус, бізнес-процес, дату планування тощо без заходу в процес
- Сірі ячейки — заблоковані

#### «Класична таблиця»
- Таблиця з чіткими межами
- Налаштування: горизонтальне вирівнювання, вертикальне вирівнювання, розмір шрифту, колір меж
- Редагування полів зі списку **не підтримується**

### Налаштування доступних для редагування полів
Налаштування системи → Прочее → **«Налаштування способів відображення процесів»** → тип БП → «Таблиця»:

| Параметр | Опис |
|---|---|
| **Поля для блокування редагування** | Список полів, що стають сірими (недоступними) |
| **Відкрити додаткові поля для редагування** | Дозволяє редагувати вибрані доп. поля зі списку |
| **Відображати кнопку друку** | Кнопка збереження процесу у файл |
| **Додаткові поля продукту** | Поля продуктів, що виводяться у ячейці з товарами |

### Налаштування колонок
Правий клік на шапці таблиці → вибрати поля, порядок, кількість записів на сторінці, сортування.

## Таймлайн (Timeline)

**Призначення:** відображення процесів на часовій шкалі (по вертикалі — час, по горизонталі — дні тижня). Розмір блоку = тривалість від «Запланувати» до «Виконати до».

### Зовнішній вигляд
- Вертикаль: часова шкала (за замовчуванням 8:00–23:00, крок 60 або 30 хв)
- Горизонталь: дні тижня (Пн–Нд)
- Блок процесу:
  - **Верхня межа** = дата «Запланувати»
  - **Нижня межа** = дата «Виконати до»
  - Якщо процеси накладаються — один відображається вузькою смужкою; при наведенні мишею розгортається на всю ширину

### Обмеження
Процес можна розтягнути лише в межах **одного дня** (якщо «Виконати до» > 24 години — блок займає одну ячейку, деталі у підказці).

### Налаштування
Налаштування системи → Бізнес-процеси → вибрати БП → Інтерфейс у календарі → **«Показувати поля у відображенні Timeline»**.

Кнопка розгортання налаштувань у самому режимі (права частина):
| Параметр | Опис |
|---|---|
| **Інтервал часу** | Повзунок початок–кінець (наприклад, з 6:00 до 22:00) |
| **Крок** | 60 або 30 хвилин |
| **Висота блоків** | 1–6 рядків тексту + час |

### Дії
- Клік на `+` у слоті → форма створення процесу
- Перетягування нижньої межі блоку — зміна дати «Виконати до»
- Клік на процес → перехід в картку

**Коли застосовувати:** планування зустрічей, розклад дзвінків, переговорні блоки.

## Робочі столи (Desktops)

**Призначення:** організація застосунків системи по тематичних «робочих столах» для швидкого доступу.

### Налаштування (тільки адміністратор)
Перейти до списку всіх застосунків → шестеренка → **«Налаштування desktop»**:
- Додати потрібну кількість робочих столів (`+`)
- Кожному задати **назву** та **колір**
- Видалити зайвий: позначити галочкою → іконка видалення
- Зберегти

### Відображення
- Кожен робочий стіл = кольорова точка на головному екрані
- Кількість точок = кількість столів
- Переключення: клік на точку або **свайп** (затиснути ліву кнопку і перетягнути)

### Переміщення застосунків
- Навести на застосунок → вибрати «Перемістити на» → вибрати стіл
- Переміщений застосунок зникає з поточного столу та з'являється на цільовому

### Сортування застосунків
- Увімкнути **«Режим сортування»** (кнопка у верхній частині; підсвічується синім)
- Перетягнути застосунки в потрібному порядку
- Вимкнути режим сортування

> ⚠️ Без режиму сортування перетягування запускає перемикання між столами.

### Вибране (Favorites) у межах столу
- Кожен стіл має свій список «Вибраного» — незалежно від інших столів
- Навести на застосунок → додати в «Вибране» на поточному столі

### Що можуть робити звичайні співробітники
- **Бачать** всі робочі столи, створені адміністратором
- **Можуть** переміщати застосунки між столами, додавати в «Вибране» — налаштування особисті
- **Не можуть** створювати, видаляти та перейменовувати столи

### Збереження налаштувань для всіх
Шестеренка → **«Зберегти зовнішній вигляд для всіх користувачів»** — застосовується тільки для тих, хто ще не робив власних налаштувань.

### Пошук між столами
Пошук (іконка лупи) — **шукає по всіх столах** одночасно. Знайдений застосунок відкривається незалежно від того, на якому столі він знаходиться.

## Загальні та особисті налаштування меню

### Два рівні налаштувань
Шлях: Налаштування системи → **«Налаштувати меню»** → перемикач **Загальні / Особисті**

| Тип | Хто може | Коли застосовується |
|---|---|---|
| **Загальні** | Тільки адміністратор | Якщо у користувача немає особистих налаштувань |
| **Особисті** | Будь-який співробітник | Замінюють загальні для конкретного користувача |

### Особисті налаштування меню
- Перемкнутись на вкладку «Особисті»
- Можна: приховати непотрібні пункти, змінити їх порядок перетягуванням
- «Скинути особисті налаштування» — повернутись до загальних (і налаштувати заново)

### Важливий нюанс
При перемиканні на «Особисті» — стан меню **копіюється** з загального на момент перемикання. Якщо адміністратор пізніше додасть новий пункт до загального меню — у користувача з особистими налаштуваннями цей пункт **не з'явиться автоматично**. Потрібно вручну скинути особисті налаштування.

### Налаштування положення меню
Ці параметри завжди **особисті** (навіть якщо змінити в загальних — застосовується лише для себе):
- Відображати меню **завжди відкритим** або **завжди закритим**
- Горизонтальне меню **внизу** або вертикальне **ліворуч**

## Панель фільтрів подій

### Де знаходиться
Додаток **«Події»** → кнопка «Фільтрувати»

### Стандартні фільтри
Тип події, Напрямок (вхідні/вихідні/внутрішні), Джерела, Етапи та інші.

### Налаштування панелі
Шестеренка → **«Інтерфейс фільтрів подій»**:
- Видалити непотрібні фільтри (×)
- Додати нові (`+` → вибрати зі списку)
- Змінити порядок: перетягнути блок
- Зберегти

### Налаштування для різних ролей
Вгорі є поле **«Роль»** (за замовчуванням «Всі ролі»). Можна налаштувати окрему панель фільтрів для директора, менеджерів, call-центру тощо.

**Рекомендація:** фільтри, що використовуються найчастіше — розміщувати вище; невживані — видалити.

## Управління полями у списках

**Де застосовується:** списки процесів (закази, задачі, проекти, ліди), контакти, продукти, фінанси.

### Відкриття налаштувань
Іконка з 9 квадратиків (ліворуч угорі) → **«Налаштування відображення полів»**

### Доступні налаштування
| Параметр | Опис |
|---|---|
| **Відображення колонок** | Галочки: синя = відображається, без галочки = прихована |
| **Порядок колонок** | Перетягнути за три точки (`⋮`) |
| **Пошук поля** | Швидкий пошук поля за назвою |
| **Кількість записів на сторінці** | Від 1 до 100 (за замовчуванням 50) |
| **Сортування** | 1-й і 2-й рівень: поле + напрям (A→Z або Z→A) |
| **Фіксований заголовок таблиці** | Шапка залишається видимою при прокрутці |
| **Зберегти для всіх користувачів** | Застосовується тільки для тих, у кого «загальна» таблиця |

### Два типи таблиць: «загальна» та «особиста»
- Якщо користувач **налаштував таблицю під себе** і зберіг (без галочки «Для всіх») — це **особиста** таблиця
- Якщо адміністратор натисне «Зберегти для всіх» — зміни застосуються лише для тих, у кого **загальна** таблиця (без особистих налаштувань)
- Особиста таблиця **не перезаписується** навіть при збереженні для всіх

> ⚠️ Типова помилка: адміністратор думає, що «Зберегти для всіх» перезаписує налаштування менеджерів. Це не так — особисті налаштування захищені.

## Функціонал унікальних полів

**Де налаштовується:** Налаштування системи → Прочее → пошук «уникальность»

### Унікальні поля контактів (детально — у файлі `05-contacts.md`)
Телефон, Email, Назва компанії, Зовнішній ID.

### Унікальні поля продуктів
| Поле | За замовчуванням | Де вимкнути |
|---|---|---|
| **Артикул** | Унікальний (якщо галочка включена) | Прочее → «Унікальність артикулів» |
| **Зовнішній ID** | Унікальний | Прочее → «Унікальність зовнішнього ID» |
| **Штрих-код** | Не унікальний | Прочее → «Унікальність штрих-коду» |
| **Назва товару** | Не унікальна (дублі дозволені) | Прочее → «Дозволити дублікати товарів» |

Також: **«Унікальність штрих-коду у картці товару та фільтрах»** — штрих-код фільтра не може збігатися зі штрих-кодом у картці.

### Унікальне поле: Номер процесу
- За замовчуванням = ID процесу, але можна редагувати
- Прочее → **«Унікальність номера процесу»** — система не дає встановити однаковий номер двом процесам

### Унікальність у додаткових полях
При редагуванні будь-якого додаткового поля (контакту або процесу) є галочка **«Тільки унікальне значення»**. При спробі зберегти дублюване значення — система показує помилку і повідомляє, у якому запису вже є таке значення.
