## 1. Імпорт контактів через XLS

### Підготовка

- Встановити додаток **«Імпорт/Експорт контактів»** через OneBox Market (пошук за словом «імпорт»).
- Підготувати XLS/XLSX-файл без форматування, фільтрів і макросів. Перший рядок завжди сприймається як заголовки і **не імпортується**.
- Кодування файлу: UTF-8 (99 % випадків) або Windows-1251 — вибирається при завантаженні.

### Шлях у системі

`Меню → Додаток «Імпорт/Експорт контактів» → кнопка «Імпортувати»`

Або: `Контакти → Всі контакти → кнопка «Імпорт контактів»` (новіший інтерфейс).

### Підтримувані формати файлу

- XLS, XLSX
- CSV (роздільник: `;`, `,` або табуляція)

### Логіка пошуку існуючого контакту

При імпорті система спочатку шукає існуючий контакт і оновлює його, якщо знайде. Пріоритет пошуку:

1. Внутрішній ID OneBox
2. Зовнішній ID (external_id)
3. Номер телефону
4. Email

Для компаній — додатково по **назві компанії**.

### Сопоставлення колонок

Після вибору файлу і натискання «Розпочати завантаження» відкривається сторінка налаштування колонок. Потрібно вказати:

| Колонка у файлі | Поле OneBox |
|---|---|
| ID | ID контакту OneBox |
| Зовнішній ID | Зовнішній ID |
| ПІБ / Прізвище, Ім'я, По-батькові | Можна окремо або разом |
| Телефон | Основний телефон |
| Додаткові телефони | Додаткові номери (через кому з пробілом: `050 111 22 33, 067 444 55 66`) |
| Email | Email |
| Компанія | Назва компанії |
| Тип/Пол | Ключ типу: `man`, `woman`, `company` |
| Холдингова компанія | Холдингова компанія |
| Кастомне поле | Додаткове поле контакту (в дужках позначено `(до поле)`) |
| Дата народження | Формат: `РРРР-ММ-ДД` (наприклад, `1983-01-11`) |
| Група контакту | ID або назва групи |

> **Важливо:** для поля «Тип/Стать» треба вказувати **ключ**, а не текст. Ключі дивитись: `Налаштування → Прочее → Типи контактів`. Для компанії — ключ `company`, інакше контакт не буде створений як компанія.

### Додаткові налаштування імпорту

- **Не шукати по email і телефону** — відключити пошук дублів за цими полями.
- **Не оновлювати компанії, тільки створювати нові.**
- **Юридичні реквізити** — вибрати формат (Україна тощо), вказати номери колонок для кожного поля реквізитів через кому.
- **Тип адреси** — вибрати тип для завантаження адреси контакту.

### Результат імпорту

У списку відображається статистика кожного імпорту:
- Updated (оновлено)
- Added (додано)
- Skipped (пропущено)
- Errors (помилки)

### Типові помилки

- Якщо у контакту немає контактних даних (телефон або email) і **не включено** налаштування «Дозволити додавання без контактних даних» (`Налаштування системи → Прочее`), контакт не буде створено.
- Якщо вказати компанію і холдингову компанію в одному рядку для фізичної особи — холдингова компанія не підтягнеться. Для правильного імпорту ієрархії компаній — імпортувати холдингові компанії окремим рядком (без ПІБ, тільки назва компанії і тип `company`).
- Якщо у рядку є тільки зовнішній ID та поля для оновлення, але немає даних для створення — і контакт не знайдено — новий не створюється.

## 2. Імпорт продуктів через XLS

### Підготовка

- Встановити додаток **«Імпорт/Експорт продуктів»** через OneBox Market.
- Включити унікальність артикулу та зовнішнього ID: `Продукти → шестерня → Прочі налаштування → Унікальність артикулу + Унікальність поля "Зовнішній ID"`.
- Підготувати XLS/XLSX-файл без зайвого форматування. Перший рядок — заголовки, **не імпортується**.

### Шляхи у системі

Спосіб 1: `Меню → Продукти → Продукти списком → кнопка «Операції з імпорту/експорту» → Імпорт продуктів`

Спосіб 2: `Меню → Продукти → Імпорт і Експорт Excel` (перейти на вкладку «Імпорт»)

Спосіб 3: `Додаток «Імпорт/Експорт продуктів» → кнопка «Імпортувати»`

### Логіка пошуку існуючого товару

Пошук по:
1. Внутрішньому ID
2. Зовнішньому ID
3. Артикулу
4. (опційно) Штрих-коду — якщо увімкнено в «Тонких налаштуваннях»

### Основні поля сопоставлення колонок

| Поле у файлі | Поле OneBox |
|---|---|
| ID | ID продукту |
| Зовнішній ID | Зовнішній ID |
| Артикул | Артикул |
| Назва | Назва товару (обов'язково для створення) |
| Категорія | Категорія продукту (за ID або назвою) |
| Ціна | Ціна продажу |
| Закупівельна ціна | Закупівельна ціна |
| Зображення | Пряме посилання на зображення (при відкритті одразу відкривається картинка) |
| Додаткові зображення | Прямі посилання через `, ` (кома + пробіл) |
| Знижка | Знижка (у відсотках) |
| Додаткове поле | Кастомне поле продукту |

> **Фільтри і постачальники** — колонки сопоставлення **залишати пустими**, налаштовуються окремо в «Тонких налаштуваннях завантаження фільтрів і постачальників».

### Тонкі налаштування (фільтри і постачальники)

**Налаштування пошуку категорії:**
- За ID (рекомендовано — менше ризик помилки)
- За назвою

**Налаштування фільтрів:**

Вказуються **номери колонок** (через кому для декількох фільтрів):

| Налаштування | Приклад |
|---|---|
| Номери колонок з назвами фільтрів | `12,18` |
| Номери колонок зі значеннями фільтрів | `13,19` |
| Доступний у фільтрах (0/1) | `14,20` |
| Характеристика в таблиці (0/1) | `15,21` |
| Надбавка (0/1) | `16,22` |
| Значення надбавки | `17,23` |
| Тип надбавки (`sum`/`percent`) | (опційно) |
| Штрих-коди фільтрів | (опційно) |

**Налаштування постачальників:**

| Поле | Номер колонки |
|---|---|
| ID постачальника | напр. `24` |
| Код товару у постачальника | напр. `25` |
| Ціна постачальника | напр. `26` |

**Додаткові галочки:**
- **Видалити старі значення фільтрів** — перед імпортом очищає всі фільтри товару.
- **Перезаписати значення фільтрів** — знаходить фільтр за назвою і оновлює значення (не дублює).
- **Розділяти значення фільтрів по `;`** — кожне значення записується окремо.
- **Видалити старі бренди** — видаляє бренди перед імпортом.
- **Видалити додаткові фото** — перед імпортом очищає додаткові зображення.

### Шаблони налаштувань

Після першого налаштування імпорту можна зберегти **шаблон** (вказати назву в полі «Шаблон налаштувань»). При наступних імпортах однотипних файлів — просто вибрати шаблон і всі колонки заповняться автоматично.

### Зображення

Зображення завантажуються **не одразу**, а через **годинний крон**. Посилання повинно бути прямим (при переході одразу відкривається картинка).

### Нові фільтри

Якщо у файлі є назва фільтра, якого немає в системі, він **створюється автоматично**.

## 3. Імпорт продуктів з Google Таблиць

### Принцип роботи

Автоматичне дію раз на N хвилин перевіряє Google Таблицю та оновлює/створює товари. Пошук існуючого товару — **по артикулу** (або по назві / штрих-коду, якщо увімкнено відповідне налаштування).

### Підключення Google Таблиць

1. `Меню → Налаштування системи → OneBox Market → Google Таблиці → Налаштувати`
2. Заповнити 4 поля:
   - **Google Таблиці Client ID** (Unique ID сервісного акаунту)
   - **Google Таблиці Email** (email сервісного акаунту)
   - **Файл P12**
   - **Файл JSON**
3. Отримати ці дані: `console.developers.google.com` → Сервісні акаунти → Create Key (JSON + P12).
4. Відкрити доступ до Google-документу для сервісного акаунту з правами **Редактор**.

### Налаштування автоматичного дії

`Налаштування системи → Автоматизація → Автоматичні дії раз на хвилину → «Імпорт продуктів з Google Таблиць»`

| Налаштування | Опис |
|---|---|
| Інтервал (хвилини) | Як часто перевіряти таблицю (наприклад, 5) |
| ID Google Таблиці | З адресного рядка між двома слешами |
| При створенні записати в до-поле | Записати значення у кастомне поле (для фільтрації імпортованих) |
| Сопоставлення колонок | Назва колонки у таблиці → поле OneBox |

**Обов'язково вказати артикул** — саме по ньому шукається існуючий товар.

### Приклад структури Google Таблиці

| артикул | назва | цена | цена закупки | наличие | категорія |
|---|---|---|---|---|---|
| Q157AS | Товар 1 | 100 | 50 | 1 | Дивани |

Наявність: `1` = в наявності, `0` = не в наявності.

### Галочки налаштування

- **Тільки оновлювати, не створювати нові** — оновлення вже існуючих товарів.
- **Шукати по назві замість артикулу.**
- **Шукати по штрих-коду замість артикулу.**
- **Видалити постачальників, які не були оновлені з файлу.**
- **Увімкнути логування** — у розділі `Звіти → Логування` буде лог `product_import_google_spreadsheets`.

## 4. Імпорт залишків на склад через XML

### Принцип роботи

Дія **«Інтеграція залишків на складах XML (Імпорт)»** оновлює залишки: не просто приходує зверху, а **встановлює актуальний залишок**. Якщо в системі товару більше — система списує різницю; якщо менше — приходує.

### Розміщення в системі

`Налаштування системи → Автоматизація → Автоматичні дії раз на годину`

### Джерела XML-файлу

- **Локальна папка** — шлях до файлу на сервері OneBox; кнопка «Перевірити існування файлу».
- **FTP** — FTP директорія, хост, логін, пароль, порт, ознака Passive.
- **URL (зовнішнє посилання)** — пряме посилання на файл.

### Обов'язкові налаштування

- **Шлях до масиву даних** (path to data array) — вказується ієрархія тегів, наприклад `items/item`. Корневий елемент не вказується.

### Приклад структури XML

```xml
<shop>
  <items>
    <item>
      <storage_name>Склад 1</storage_name>
      <storage_code>S1</storage_code>
      <product_id>15</product_id>
      <product_name>Товар А</product_name>
      <count>3</count>
      <price>50</price>
      <currency>UAH</currency>
      <date>2024-01-15</date>
    </item>
  </items>
</shop>
```

Шлях до масиву: `items/item`

### Сопоставлення полів

У лівій колонці — поле OneBox, у правій — назва тегу/атрибуту у XML-файлі:

| Поле OneBox | Тег у XML (приклад) |
|---|---|
| Назва складу | `storage_name` |
| Зовнішній ID складу | `storage_code` |
| ID товару | `product_id` |
| Назва товару | `product_name` |
| Серійний номер | `serial` |
| Дата приходу | `date` |
| Кількість | `count` |
| Вхідна ціна | `price` |
| Валюта | `currency` |

> Якщо значення є **атрибутом** XML-тегу (наприклад, `<item id="1">`), треба відмітити галочку «Атрибут».

### Поведінка щодо складів і товарів

- Якщо **склад не знайдений** по назві або коду — система **створює новий склад**.
- Якщо **товар знайдений** по ID — оновлює залишки по ньому.
- Дія встановлює **актуальний залишок**, а не додає зверху.

### Додаткові налаштування

- **Видалити файл після завантаження** — тільки при FTP.
- **Дописати комірку складу в доступні комірки складу** — для роботи з комірками.
- **Перед оновленням залишків обнулити залишки по продукту** — спочатку обнулити, потім приходувати.
- **Запускати раз на N хвилин**: 15, 30 або 60 хвилин.

## 5. Завантажувач залишків на склад (XLS)

### Принцип роботи

Окремий розділ для ручного або напівавтоматичного оновлення залишків по одному складу. Пошук товару — **тільки по артикулу**. Встановлює **фінальний баланс** (не приходує зверху).

### Шлях у системі

`Меню → Склади → Завантажувач залишків`

### Порядок дій

1. Вибрати **склад**.
2. Вибрати **постачальника** (необов'язково).
3. Завантажити **файл** (XLS/CSV — не потрібен стандартизований шаблон, потрібні лише колонки артикул + кількість).
4. Налаштувати **сопоставлення колонок**:
   - **Артикул продукту** — для пошуку товару.
   - **Поточний залишок** — фінальна кількість, яку треба встановити.
5. Перегляд результату: система показує знайдені та не знайдені товари.
   - Червоний рядок = товар не знайдений; можна вручну зв'язати або видалити рядок.
   - Можна відредактувати через кнопку олівця або «Відкрити червоні поля».
6. Натиснути **«Завантажити всі збіги»**.

### Збереження налаштувань

Система **запам'ятовує** сопоставлення колонок для кожного конкретного складу. При наступному завантаженні для того ж складу — поля заповнюються автоматично.

### Доступні поля для сопоставлення

- Найменування продукту
- Артикул
- Поточний залишок
- Дата списання
- Кількість для списання
- Кількість для оприходування
- Підсумкова кількість

## 6. Імпорт процесів через XLS

⚠️ **Файл транскрипту цього відео має критичні дефекти** (суміш мов, нечитабельний текст). Достовірних технічних деталей витягти не вдалося.

**Що відомо зі збереженого фрагменту:**
- Функція дозволяє імпортувати процеси з Excel-файлу.
- При імпорті можна вибрати колонки для сопоставлення.
- При знаходженні контакту по телефону — система може знайти збіг і прив'язати існуючого клієнта.
- Статус нових процесів — «Новий».

⚠️ Детальніший опис потребує перевірки через офіційну документацію OneBox або повторну дистиляцію джерела.

## 7. Експорт контактів у XLS

### Підготовка

Потрібен той самий додаток **«Імпорт/Експорт контактів»**.

### Порядок дій

1. `Меню → Контакти → Всі контакти` — відфільтрувати потрібні контакти.
2. Відмітити галочками потрібні контакти або «Вибрати всі на всіх сторінках».
3. Натиснути **«Відправити в»** → вибрати «Імпорт/Експорт контактів» → «Експортувати контакти».
4. У вікні налаштувань:
   - Вибрати формат: **XLS** або **XLSX**.
   - Вказати **email** одержувача — на нього прийде лист із посиланням на файл.
   - Вибрати поля для експорту: вручну (`+ Додати ще`) або натиснути **«Заповнити за замовчуванням»**.
5. Натиснути **«Завантажити»**.

### Поля за замовчуванням

ID контакту, ПІБ (разом і окремо: ім'я, прізвище, по-батькові), телефон, додаткові телефони, email, додаткові email тощо.

### Перейменування полів у файлі

У правій колонці налаштувань можна змінити назву поля у вихідному файлі (наприклад, «ID контакту OneBox» → «ID»).

### Важливо: налаштування пошти

Для коректної відправки листа зі посиланням на файл потрібно **підключити інтеграцію з поштою**:
`Додаток «Пошта» → шестерня → Налаштування підключення → Додати нове підключення → вказати email та пароль`

Потім: `шестерня → Прочі налаштування → Зворотня адреса email для листів`.

Без налаштованої пошти лист може не надійти.

Кнопка «Додати значення» — умови типу «поле > значення». Тільки товари, що відповідають умовам, потраплять у файл.

## 11. Експорт процесів у XML (автоматичний)

### Розміщення дії

`Налаштування системи → Автоматизація → Автоматичні дії раз на хвилину (або раз на годину) → «Інтеграція бізнес-процесів XML (Експорт)»`

Можна додавати декілька таких дій одночасно.

### Джерело вивантаження

- **Локальна папка**: шлях + назва файлу (наприклад, `/media/export/orders.xml`). В назві можна використовувати змінну `{date}`.
- **FTP**: директорія, хост, порт, логін, пароль, FTP Passive.

### Ключові налаштування

| Налаштування | Опис |
|---|---|
| Вивантажувати БП починаючи з дати | Якщо не вказано — всі процеси |
| Бізнес-процеси + статуси | Вибрати які БП і статуси вивантажувати |
| Не вивантажувати раніше вивантажені | Система запам'ятовує останній вивантажений процес |
| Тільки процеси, що змінили статус | З моменту останнього запуску дії |
| **Шлях до масиву даних** | **Обов'язкове поле.** Наприклад `orders/order` |
| Шлях до масиву до-полів | Наприклад `additional_fields` |
| Шлях до масиву до-полів продукту процесу | Наприклад `custom` |
| Додаткова назва дії | Відображається в дужках поруч з назвою дії (для ідентифікації) |
| Запускати раз на N хвилин | Наприклад 20 або 30 |

### Правила іменування тегів

- Тільки латиниця, без пробілів.
- Пробіл замінювати на `_` (нижнє підкреслення).
- Не використовувати кирилицю.

### Сопоставлення полів

Ліва колонка — поле OneBox, права — назва XML-тегу (задається вільно):

```
ID процесу         → order_id
Назва процесу      → order_name
Дата створення     → order_date
Запланований       → order_planned
ID клієнта         → client_id
Телефон клієнта    → client_phone
Email клієнта      → client_email
Коментар процесу   → order_desc
```

**Шлях до масиву товарів процесу** (наприклад, `offers/offer`):

```
Назва товару       → prod_name
Кількість          → prod_qty
Ціна               → prod_price
Валюта             → prod_currency
ID товару          → prod_id
```

### Додаткові поля

Вказуються окремою таблицею:

| Ключ до-поля (з системи) | Назва у XML | До чого відноситься |
|---|---|---|
| `region_6` | `region` | Процес |
| `custom_field_1` | `field_name` | Продукт процесу |

### Статичні поля

Можна задати поле з фіксованим значенням для всіх вивантажуваних процесів:

```
Ключ поля: source_name
Значення:  OneBox
```

### Приклад структури вихідного XML

```xml
<shop>
  <orders>
    <order>
      <order_id>313</order_id>
      <order_name>Тест розсилка</order_name>
      <client_phone>+380501234567</client_phone>
      <additional_fields>
        <region>17</region>
      </additional_fields>
      <offers>
        <offer>
          <prod_name>Диван кутовий</prod_name>
          <prod_qty>2</prod_qty>
          <prod_price>4850</prod_price>
        </offer>
        <offer>
          <prod_name>Кутовий стіл</prod_name>
          <prod_qty>1</prod_qty>
          <prod_price>1400</prod_price>
        </offer>
      </offers>
      <source_name>OneBox</source_name>
    </order>
  </orders>
</shop>
```

## 12. Вивантаження процесу в XML з налаштуванням полів (поодинокий)

### Принцип

Замість масового автоматичного вивантаження — формується **окремий XML-файл** для кожного конкретного процесу. Запускається дією на **етапі бізнес-процесу** або **процедурою по кліку**.

### Розміщення

`Налаштування системи → Бізнес-процеси → Етап → Додати дію → «VX: Вивантажити в XML з налаштуванням полів»`

Або аналогічно через **Процедури** (`workflow → процедура по кліку`).

### Основні налаштування

| Налаштування | Опис |
|---|---|
| Джерело вивантаження | Локальна папка або FTP |
| Шлях до файлу | За замовчуванням: `/media/export/order-{order_id}.xml` |
| Назва файлу | Можна використовувати змінну `{order_id}` та `.xml` розширення |
| Шлях до масиву даних | Обов'язково. Наприклад: `orders/order` |
| Шлях до масиву до-полів | Наприклад: `custom` |
| Шлях до масиву до-полів продукту процесу | Опційно |

### Сопоставлення полів

Аналогічно п. 11. Доступні поля:
- ID процесу, зовнішній ID, назва, дата, коментар/опис, назва етапу, телефон/email/ім'я клієнта, сума, спосіб доставки, спосіб оплати.
- Шлях до масиву товарів процесу (наприклад `products/product`): назва, кількість, ціна, ID товару.

### Додаткові галочки

- **Вивантажувати тільки заповнені поля** — порожні теги не з'являтимуться у файлі.
- **Не вивантажувати продукти процесу.**
- **Не додавати коментарі** (локальні вивантаження).
- **Запускати дію після редагування процесу.**

### Результат

Після спрацювання дії в **стрічці коментарів** процесу з'являється повідомлення з посиланням на XML-файл. Клік по посиланню — одразу починається завантаження.

## 14. Експорт складських залишків у XML

### Розміщення дії

`Налаштування системи → Автоматизація → Автоматичні дії раз на годину → «Інтеграція залишків на складах XML (Експорт)»`

### Джерело вивантаження

- **Локальна папка**: шлях до файлу.
- **FTP**: директорія, хост, порт, логін, пароль.

### Обов'язкові налаштування

- **Шлях до масиву даних** (наприклад `storage/storage`).

### Сопоставлення полів

| Поле OneBox | Тег у XML (приклад) |
|---|---|
| ID складу | `storage_id` |
| Назва складу | `storage_name` |
| Зовнішній ID складу | `storage_external_id` |
| ID товару | `product_id` |
| Зовнішній ID товару | `product_external_id` |
| Назва товару | `product_name` |
| Кількість | `count` |
| Кількість у резерві | `reserved` |
| Вхідна ціна | `price` |
| Валюта (ID) | `currency` |
| Дата приходу | `date` |

Порожні поля = не вивантажуються.

### Важлива особливість

Залишки вивантажуються **окремими записами** по кожному приходу (оприходуванню). Тобто якщо один товар оприходовувався двічі (два різних приходи), у XML буде два рядки (два теги `storage`).

### Приклад вихідного XML

```xml
<shop>
  <storage>
    <storage id="5" name="Склад 1" external_id="">
      <product_id>4</product_id>
      <product_name>Тканина</product_name>
      <count>6</count>
      <reserved>0</reserved>
      <price>50</price>
      <currency>1</currency>
      <date>2020-10-15</date>
    </storage>
    <storage id="5" name="Склад 1" external_id="">
      <product_id>4</product_id>
      <product_name>Тканина</product_name>
      <count>1</count>
      <reserved>0</reserved>
      <price>50</price>
      <currency>1</currency>
      <date>2020-10-19</date>
    </storage>
  </storage>
</shop>
```

> Валюта у полі `currency` — це **ID валюти** в системі, не назва. Назви валют дивитись у `Налаштування → Валюти`.

## 15. Передача даних замовлення (процесу) у форматі XLS

### Принцип

Дія на **етапі бізнес-процесу**: при переході процесу на вказаний етап формується **окремий XLS-файл** для цього конкретного процесу.

### Розміщення

`Налаштування системи → Бізнес-процеси → Налаштування БП → Етап → Додати дію → «Передати дані процесу в XLS форматі по FTP»`

### Джерело вивантаження

- **FTP**: директорія, хост, порт, логін, пароль, FTP Passive.
- **Локальна папка**: шлях до файлу.

### Назва файлу

Підтримуються змінні:
- `{date}` — дата вивантаження (рік-місяць-день)
- `{order_id}` — ID процесу

Приклад: `order-{order_id}.xls`

### Поля для вивантаження

Вибираються через `+ Додати значення`. Кожному полю задається назва колонки у XLS. Приклади:

| Поле OneBox | Назва у XLS |
|---|---|
| ID процесу | `order_id` |
| Назва процесу | `order_name` |
| Дата процесу | `order_date` |
| Дата планування | `order_plan_date` |
| ID клієнта процесу | `client_id` |
| Коментар процесу (= Опис) | `order_desc` |
| ПІБ менеджера | `manager_name` |
| ПІБ автора | `author_name` |
| Назва бізнес-процесу | `bp_name` |
| Ім'я етапу | `stage_name` |
| Телефон клієнта | `client_phone` |
| Email клієнта | `client_email` |
| Сума процесу | `order_sum` |
| Спосіб оплати | `payment_name` |
| Спосіб доставки | `delivery_name` |
| Назва товару | `product_name` |
| Кількість товару | `product_qty` |
| Сума по продукту | `product_sum` |
| До-поле процесу | За ключем поля |

> **Коментар процесу** у цьому контексті — це **Опис** процесу (не коментарі в стрічці).

### Результат

Після спрацювання дії у стрічці коментарів з'являється повідомлення із **клікабельним посиланням** на XLS-файл (одразу починається завантаження).

## 16. Передача замовлень між двома OneBox через XML

### Схема

```
OneBox 1 (Джерело)           OneBox 2 (Призначення)
  ↓ Дія «Експорт XML»           ↓ Дія «Імпорт XML»
  XML-файл (FTP або URL)   →   Зчитати файл → Створити процес
```

### Крок 1: Налаштування ExpORT у OneBox 1

`Автоматичні дії раз на хвилину → «Інтеграція бізнес-процесів XML (Експорт)»`

Налаштування — аналогічно п. 11. Рекомендований інтервал — кожні 20–30 хвилин (не щохвилини).

### Крок 2: Налаштування IMPORT у OneBox 2

`Автоматичні дії раз на хвилину → «Інтеграція бізнес-процесів XML (Імпорт)»`

| Налаштування | Опис |
|---|---|
| Тип файлу | XML або JSON |
| Джерело | Зовнішнє посилання (URL на файл з OneBox 1), або локальна папка, або FTP |
| Шлях до масиву даних | Те саме, що і при експорті (наприклад `orders/order`) |
| Сопоставлення полів | Ліва колонка — назва тегу у XML файлі; права — поле OneBox 2 |

### Сопоставлення полів при імпорті

При імпорті — навпаки: ліва колонка це **назва тегу у XML** (те, що задавалось при експорті):

```
order_id   → ID процесу
order_name → Назва процесу
client_phone → Телефон клієнта
...
```

### Поведінка при імпорті

- **Клієнт**: система шукає існуючого клієнта (за телефоном, email тощо). Якщо знайшла — прив'язує; якщо ні — створює нового.
- **Продукти**: система шукає існуючий товар в OneBox 2. Якщо знайшла — підставляє; якщо ні — **створює новий товар**.
- **Менеджер/автор**: якщо ID менеджера не знайдено в OneBox 2 — призначається за замовчуванням.

### Сопоставлення способів доставки/оплати

Якщо назви способів в OneBox 1 і OneBox 2 відрізняються, налаштовується сопоставлення:

```
У файлі (OneBox 1): "Самовивіз"    → В OneBox 2: "Забір з магазину"
У файлі (OneBox 1): "Оплата на карту" → В OneBox 2: "На карту"
```

### Інші налаштування імпорту

- **Шукати процес по ID** — для оновлення існуючих.
- **Шукати процес по External ID** — аналогічно.
- **При знаходженні процесу — оновлювати** (а не тільки створювати нові).
- Запускати раз на годину або раз на N хвилин.

## 17. Експорт налаштувань БП на GitHub

### Призначення

Збереження JSON-конфігурації бізнес-процесу в Git-репозиторій (аналог бекапу, версіонування).

### Підключення

`Налаштування системи → OneBox Market → GitHub → Налаштувати`

Потрібно заповнити 4 поля:

| Поле | Де взяти |
|---|---|
| **Токен доступу** | GitHub → Settings → Developer Settings → Personal Access Tokens → Generate new token (галочки: `repo`, `admin:repo_hook`, `delete_repo`) |
| **Власник репозиторію** | Із «хлібних крихт» репозиторію у GitHub |
| **Ім'я репозиторію** | Із сторінки репозиторію в GitHub |
| **Гілка репозиторію** | Наприклад `main` |

### Порядок дій

1. Отримати Personal Access Token у GitHub.
2. Створити або вибрати існуючий репозиторій.
3. Заповнити 4 поля в налаштуваннях інтеграції, зберегти.
4. `Налаштування системи → Бізнес-процеси` → навести на назву БП → **«Імпорт/Експорт (GitHub)»**.
5. У лівій частині (Експорт) — вказати коментар коміту → натиснути **«Експорт»**.

### Результат

У репозиторії GitHub з'являється JSON-файл з назвою `export_workflow_NN.json`. Зміст — повна конфігурація БП.

### Імпорт назад

У правій частині того ж вікна можна вибрати JSON-файл з репозиторію і **імпортувати** налаштування БП.

## 18. Загальні зауваження та best practices

### Перевірка перед масовим імпортом

Перед завантаженням великого файлу (тисячі рядків) — **завжди спочатку протестуйте на 2–10 записах**. Перевірте результат, переконайтеся у коректності. Тільки потім запускайте повний файл.

Видалити 5 записів після помилкового тесту — набагато простіше, ніж виправляти 5 000.

### Email для отримання файлів експорту

Всі функції XLS/XLSX-експорту, що надсилають посилання на файл поштою (контакти, продукти), вимагають **налаштованої поштової інтеграції** в системі. Без неї лист може не прийти.

`Додаток «Пошта» → шестерня → Підключення → Зворотня email адреса для листів`

### Шаблони налаштувань (імпорт/експорт продуктів)

При повторних операціях з однотипними файлами — обов'язково використовувати **шаблони**. Економить час і захищає від помилок.

### Зображення при імпорті продуктів

Зображення підтягуються **через годинний крон** після імпорту (не миттєво). При великій кількості товарів — поступово, декілька кронів поспіль.

Посилання на зображення повинно бути **прямим** (відкрити у браузері = одразу відкривається файл зображення).

### Іменування XML-тегів

- Тільки латиниця.
- Без пробілів (замінювати на `_`).
- Без кирилиці.

### Декілька однотипних XML-дій

OneBox дозволяє додавати кілька дій «Інтеграція бізнес-процесів XML (Експорт)» одночасно. Щоб не плутатися — заповнюйте поле **«Додаткова назва дії»** (відображається в дужках поруч з назвою).

### Автоматичний експорт платежів у XML

⚠️ Функція автоматичного експорту платежів у XML існує в OneBox, але докладне покриття — у розділі фінансів. Коротко: в автоматизації є дія «Інтеграція платежів XML (Експорт)» за аналогічним принципом (шлях до масиву, сопоставлення полів, локальна папка або FTP).
