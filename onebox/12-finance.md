## Платежі та каса

### Загальна концепція

Застосунок **Платежі та каса** веде всі вхідні та вихідні оплати між вашими юридичними особами і контактами. Показує **баланс кас** — скільки реально грошей на рахунках.

Усі платежі проводяться по **акаунтах (кошельках)** — аналог банківського рахунку або готівкової каси.

### Акаунти (кошельки)

- Створення: шестерня → **Акаунти платежів** → **+** → вказати назву → галочка «Активний» → **Створити акаунт**.
- Кожен платіж прив'язується до конкретного акаунту.
- Акаунти відображаються в налаштуваннях Cashflow та на загальній сторінці фінансів.

### Способи проведення платежу

**1. З картки процесу (замовлення/задачі):**
- Відкрити процес → **Відправити в** → **Платежі та каса** → **Створити платіж і прикріпити до процесу**.
- Заповнити: сума, акаунт (кошелек), тип (вхідний / вихідний), клієнт (підтягується автоматично).
- Зберегти → платіж прив'язується до процесу.

**2. Прямо через застосунок Платежі та каса** — без прив'язки до конкретного процесу.

### Категорії платежів

- Дозволяють групувати платежі всередині одного акаунту (наприклад, «Оренда», «Реклама»).
- Створення: шестерня → **Категорії платежів** → **+** → назва, батьківська категорія (опційно), галочка «Активна» → **Додати запис**.
- При створенні платежу категорія вибирається поряд з акаунтом.

### Перегляд платежів за процесом

- Відкрити процес → **Відправити в** → **Платежі та каса** → **Показати платежі процесу**.

## Застосунок Cashflow

### Призначення

Показує **грошові потоки в майбутньому** на основі **очікуваних платежів**. Ніяких автоматичних прогнозів не робить — дані формуються виключно на основі введених очікуваних платежів.

### Структура звіту

- **Стовпці** — часові **періоди** (день / тиждень / місяць / квартал).
- **Рядки** — категорії платежів.
- **Блок «Очікувані витрати»** — витрати за категоріями на кожен період.
- **Блок «Очікувані надходження»** — надходження за категоріями на кожен період.
- **Баланс** — різниця надходжень і витрат за кожен період.
- **Блок «Готівка (Наявні)»** — поточний баланс кошельків (акаунтів).

### Налаштування відображення

Шестерня → **Параметри відображення**:
- **Період:** день / тиждень / місяць / квартал.
- **Кількість відображуваних періодів** (наприклад, 5 тижнів або 3 місяці).
- **Валюта** для розрахунків (вибір з випадаючого списку).
- **Юридичні особи та кошельки**, по яких відображати дані.

### Додавання очікуваного платежу

**Спосіб 1 — безпосередньо з Cashflow:** натиснути **+** → заповнити форму (сума, тип, категорія, дата, процес).

**Спосіб 2 — з картки процесу:** **Відправити в** → **Очікувані платежі** → **Додати очікуваний платіж** → вказати суму, тип (вхідний/вихідний), дату, категорію.

### Деталізація

- Клік на конкретну суму в таблиці — спливаюче вікно з переліком очікуваних платежів категорії за цей період (сума, дата, посилання на процес).

## Прогноз Cashflow

Прогноз Cashflow — це окремий застосунок (`Прогноз Cashflow`) з ідентичним до Cashflow функціоналом. За змістом транскрипту це фактично один і той самий застосунок, що описується в двох відео. ⚠️ потребує перевірки — можлива різниця у функціоналі або це один застосунок з двома відео.

**Основна логіка:**
- Очікувані платежі заносяться вручну (або через процеси).
- Система обчислює баланс по кожному майбутньому періоду: надходження − витрати = залишок.
- Кумулятивний баланс: поточні гроші на рахунках + очікуваний баланс майбутніх періодів.

## Загальна сторінка фінансів

### Доступ

- Глобальне меню → **Фінанси** → **Загальна сторінка фінансів**.
- Або: шестерня (на сторінці) → **Налаштування інтерфейсу загальної сторінки фінансів**.

### Налаштування інтерфейсу

- Сторінка будується за принципом **блокової сітки**: блоки розміщуються у клітинки сітки перетягуванням або через **+**.
- Після розстановки — натиснути **Зберегти**.
- Інтерфейс може бути різним для різних ролей (менеджер / бухгалтер тощо).
- Якщо сторінка порожня — перейти в налаштування та додати блоки.

### Доступні блоки

| Блок | Зміст |
|------|-------|
| Таблиця фактичних платежів | Список платежів за обраний період з панеллю фільтрів (дата, акаунт) |
| Підсумкові рядки фактичних платежів | Сума вхідних / вихідних / підсумок по юрособі (у базовій валюті) |
| Статистика платежів за період | Залишок на початок, вхідні, вихідні, залишок на кінець |
| Курс валют | Назви валют та їх курси |
| Залишки на бюджетах | Назва, відповідальний, підсумок по бюджетах |
| Історія бюджетів | Кількість нарахувань, списань по кожному бюджету |
| Поточний баланс бюджетів | Графік змін балансу бюджетів |
| Поточний баланс коштів | Графік балансу коштів (по датах) |
| Поточний баланс очікуваних платежів | Графік балансу очікуваних платежів |
| Залишок на рахунках | Баланси кошельків |
| Очікувані платежі | Перелік запланованих платежів |
| Графік неотриманих і неоплачених очікуваних платежів | — |
| Залишок по категоріях | Баланс по категоріях платежів |

**Примітка:** блоки не мають власних налаштувань — вони відображаються у фіксованому форматі. Можна лише вибрати їх та розставити по сітці.

## Вичислення бонусів на основі KPI

### Призначення

Автоматичне нарахування бонусів співробітникам на основі їх KPI (ключових показників ефективності). Підходить для розрахунку зарплат, премій.

### Налаштування

Розташоване у: **Налаштування системи** → **Автоматизація** → **Автоматичні дії раз на годину** → дія **«Обчислити бонус на основі KPI»**.

**Параметри дії:**

| Параметр | Опис |
|----------|------|
| День запуску | Щоденно або конкретне число місяця |
| Час запуску | Від 00:00 до 23:00 |
| Ролі | Для яких ролей нараховувати бонуси |
| KPI | Який показник відслідковувати (наприклад, «Кількість закритих процесів за день») |
| Операція | Додати / відняти / помножити / поділити |
| Фіксоване значення | Множник або константа (наприклад, ×10) |
| Коментар | Текст, що додається до запису про нарахування |
| Перевірка за останні N днів | Захист від пропущених нарахувань |
| Бізнес-процес + статус | Куди записувати бонуси (опційно) |
| Галочка «якщо процес закритий — створити новий» | — |

**Приклад:** KPI «Кількість закритих процесів за день» × 10 = кількість нарахованих бонусів.

### Перегляд нарахувань

- У картці процесу: вкладка **Бонуси** (якщо не відображається — увімкнути через налаштування інтерфейсу процесу).
- У картці співробітника: вкладка **Бонуси** → запис без оператора (нарахування автоматичне).

## Автоматичний експорт платежів у XML-файл

### Призначення

Регулярна (раз на годину) вивантаження платежів з OneBox у XML-файл для передачі до сторонніх систем.

### Налаштування

**Налаштування системи** → **Автоматизація** → **Автоматичні дії раз на годину** → дія **«Інтеграція платежів XML-експорт»**.

Можна додати кілька таких дій (кожна формує окремий XML-файл з різними фільтрами).

**Параметри дії:**

| Параметр | Опис |
|----------|------|
| Кошельки | З яких акаунтів вивантажувати |
| Категорії платежів | Фільтр за категоріями (наприклад, тільки «Оренда», «Реклама») |
| Напрям платежу | Всі / лише вхідні / лише вихідні |
| Тип платежу | Всі / фактичні / лише віртуальні |
| Вивантажувати платежі починаючи з | Дата (якщо не заповнена — вивантажуються всі) |
| Шлях до масиву платежів | Обов'язкове поле. Приклад: `payments/payment`. Визначає структуру XML |
| Джерело (ціль файлу) | Локальна папка або FTP |
| Шлях до файлу | Наприклад: `media/export/payment/` |
| Назва файлу | Латиниця + розширення `.xml`. Підтримується змінна `{date}` (рік-місяць-день) |
| Поля для експорту | Зіставлення полів OneBox → імена XML-елементів |
| Додаткове поле платежу | Записати значення («виванжений») у доп. поле після вивантаження |
| Вивантажувати тільки нові | Галочка + перевірка по додатковому полю |
| Юридичні реквізити клієнта | Вивантажувати доп. поля юрреквізитів (ідентифікатор поля з **Налаштування системи → Інше → Додаткові поля до юридичних реквізитів контакту**) |

**Структура XML (приклад шляху `payments/payment`):**
```xml
<payments>
  <payment>
    <id>46</id>
    <created>2021-02-02</created>
    <sum>1000</sum>
    <category>test2</category>
    <order_id>469</order_id>
    <order_name>Тестове замовлення №1</order_name>
    <!-- ... інші поля -->
  </payment>
</payments>
```

**Запуск:** дія спрацьовує автоматично раз на годину. Результуючий файл доступний за вказаним шляхом (локальна папка) або на FTP.
