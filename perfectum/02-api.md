# Perfectum CRM+ERP — REST API

> REST API v2.5 | Актуально: 2026-05-31

## Базові параметри

| Параметр | Значення |
|----------|----------|
| Версія | 2.5 |
| Формат | JSON |
| Auth | Статичний API Key (`Authorization: [api_key]`) |
| Base URL (коробка) | `https://your_domain/api/` |
| Base URL (хмара) | `https://your_subdomain.perfectum.ua/api/` |
| Пагінація | `?limit=N&offset=N` або `?page=N` |
| Формат дат | `Y-m-d` (напр. `2026-05-28`) |
| Документація | Налаштування → Системні → Ключі API |

## Rate Limits

| Обмеження | Значення |
|-----------|----------|
| Мікро-ліміт (хмара) | 20 запитів / 3 секунди |
| Добовий ліміт (хмара) | **200 запитів × кількість юзерів** |
| Коробка | Індивідуальне налаштування на кожен ключ |

⚠️ При активних сценаріях хмарний ліміт вичерпується швидко. Рішення — коробка або кешування.

## Генерація API-ключа
Шлях: *Налаштування → Системні → Ключі API*
- Білий список IP-адрес
- Ліміти та інтервали запитів
- XSS-валідація
- Права доступу: read/write по кожному модулю
- Термін дії: необмежений (до ручного видалення)

## Ключові ендпоінти

```
# Клієнти
GET    /api/clients             — список (пагінація, фільтрація)
POST   /api/clients             — створити
PUT    /api/clients/{id}        — оновити
DELETE /api/clients/{id}        — видалити

# Замовлення
GET    /api/orders              — список
POST   /api/orders              — створити (специфікація, доставка, сума)
PUT    /api/orders/{id}         — змінити статус / поля

# Товари
GET    /api/items               — номенклатура, залишки, категорії
POST   /api/items               — створити (SKU, ціна, категорія)

# Рахунки та оплати
GET    /api/invoices
POST   /api/invoices
POST   /api/payments            — зафіксувати оплату за рахунком

# Кастомні поля — передаються у масиві custom_fields у CRUD-запитах
# Ідентифікатор: cf_X (X — порядковий номер)
```

Сутності для кастомних полів через API: Клієнти, Контакти, Реквізити, Ліди, Проекти, Склад, Кандидати, Завдання, Підтримка, Рахунки.

## HTTP-коди помилок

| Код | Причина |
|-----|---------|
| 400 | Некоректна структура; помилка полів `staff_assignee`, `rel_id`, `rel_type` |
| 401/403 | Невірний ключ; IP не в білому списку; немає прав на модуль |
| 422 | Невірний формат дати або відсутні обов'язкові поля |
| 429 | Перевищено мікро-ліміт або добовий ліміт |

## Відомі обмеження
- ⚠️ Завантаження файлів/вкладень через API — не задокументовано
- ⚠️ Отримання списку кастомних полів через API — не задокументовано
