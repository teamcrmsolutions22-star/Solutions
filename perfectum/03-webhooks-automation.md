# Perfectum CRM+ERP — Вебхуки та Автоматизація

## Вебхуки

**Шлях:** Налаштування → Додатки → Веб-хуки → Налаштування Веб-Хуків
**Лог:** Журнал операцій веб-хуків

| Параметр | Значення |
|----------|----------|
| Доступні модулі | Склад (Товари), Замовлення, Клієнти, Контакти |
| Події | Створення, Зміна, Видалення |
| Методи | POST, PUT, PATCH, GET, DELETE |
| Формати | `json` або `form-data` |
| Headers | Необмежена кількість кастомних (ключ-значення) |
| Динамічний URL | Шорткод `{{id}}` замінюється на ID сутності |
| Безпека | Статичний токен у кастомних Headers |
| HMAC-підпис | ⚠️ Не задокументовано |
| Retry при помилці | ⚠️ Не задокументовано (вважається one-shot) |

### Приклад payload (Склад / Товар):

```json
{
  "module": "warehouse",
  "event": "update",
  "staff_id": "12",
  "item_id": "450",
  "sku": "ART-9988",
  "name": "Назва товару",
  "purchase_price": "1200.00",
  "sale_price": "1800.00",
  "warehouse_id": "2",
  "quantity_minimum": "5",
  "weight": "1.50"
}
```

## Автоматизація

### Конструктор бізнес-процесів
**Тригери:** Створення / Зміна / Видалення сутності
**Умови:** Перевірка значень будь-яких системних та кастомних полів
**Дії:**
- Моментальні: призначення відповідального, зміна статусу, створення завдання
- Відкладені: дія через N годин/хвилин (залежить від Cron)

**Журнал бізнес-процесів** — перегляд історії виконання.
**BPMN-модуль** (версія 1.6, травень 2026) — окремий модуль для візуальної побудови процесів у нотації BPMN.

### Cron (Планувальник)
Критично для: розсилки рахунків, нагадувань про дедлайни, оновлення статусів НП/УП, закриття тікетів.
**Оптимальний інтервал:** кожні 5 хвилин.
**Типова помилка:** Cron не налаштований → не працюють відкладені дії та оновлення доставки.

## Кастомні поля

**Шлях:** Налаштування → Системні → Налаштовувані поля
**Кількість:** необмежена
**Сутності:** 22 (Компанія, Ліди, Досьє, Клієнти, Контакти, Реквізити, Співробітники, Договори, Завдання, Витрати, Рахунок, Прайси, Пропозиції, Проект, Підтримка, Оголошення, Склад × 5 типів, Замовлення)

**Типи полів:** Текст (1 рядок), Текст (багато рядків), Число, Кількість, Список (1 значення), Список (кілька), Чек-бокс, Дата, Дата+час, Телефон, Колір, Google Maps, Гроші, HTML/WYSIWYG, Файл, Прив'язка до співробітника, Зображення, Гіперпосилання та ін.

**Ідентифікатор у API:** `cf_X` — передаються у масиві `custom_fields`.
**Обчислювані поля:** ⚠️ Не задокументовані в no-code. Реалізуються через плагін або розширений Конструктор.
