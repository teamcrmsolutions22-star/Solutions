# Pipedrive — Вебхуки (Webhooks v2)

## Огляд

Pipedrive передає події в реальному часі через POST-запити на URL розробника.

**Налаштування:** Settings → Tools and apps → Webhooks або через REST API (`POST /v1/webhooks`)

**Поточна версія:** Webhooks v2 (v1 — legacy, не рекомендується, міграція обов'язкова)

**Події:** `added`, `updated`, `deleted`, `merged`  
**Сутності:** Deal, Person, Organization, Activity, Note

## Формат Payload (v2)

```json
{
  "meta": {
    "action": "updated",
    "entity": "deal",
    "entity_id": 12345,
    "company_id": 98765,
    "user_id": 112233,
    "timestamp": 1717184000,
    "correlation_id": "5863a8ad-06f6-4d30-96e1-52f88e564a80"
  },
  "current": {
    "id": 12345,
    "title": "Нова назва угоди",
    "stage_id": 3
  },
  "previous": {
    "title": "Стара назва угоди",
    "stage_id": 2
  }
}
```

- `current` — новий стан сутності
- `previous` — стан до зміни
- При змінах Users — `current` і `previous` передаються масивами

## Безпека

- Підтримується **HTTP Basic Auth** (логін + пароль при реєстрації вебхука)
- Кастомні HTTP-заголовки — не задокументовано ⚠️

## Retry-логіка та Ban System

**Таймаут відповіді:** 10 секунд. Якщо сервер не відповів або повернув не 2XX — вебхук вважається недоставленим.

**3 повторні спроби:**

| Спроба | Затримка |
|--------|----------|
| 1-ша | через 3 сек |
| 2-га | через 30 сек |
| 3-тя | через 150 сек |

Поточний номер спроби передається в полі `attempt` (v2) або `retry` (v1).

**Ban System:**
- Лічильник збільшується **тільки при збої першої спроби** (повтори не рахуються)
- При **10 послідовних збоїв** першої спроби → вебхук **блокується на 30 хвилин**
- Всі події під час блокування **безповоротно втрачаються**
- Після 30 хвилин лічильник скидається, вебхук активується знову

**Автоматичне видалення:** якщо жодна доставка не вдалася протягом **3 послідовних днів** — підписка видаляється назавжди.

## Обмеження

- Максимум **40 активних вебхуків** на одного користувача
- Вихідні вебхуки **НЕ підпадають** під API rate limits
- Лог спроб доставки: Tools and integrations → Webhooks

## Кращі практики обробки вебхуків

```
Отримати POST → негайно відповісти HTTP 200 → передати тіло у фонову чергу
```

**Помилка:** блокувати з'єднання тривалою обробкою → таймаут 10с → Ban System.

**OAuth scopes для вебхуків:** `webhooks:read` / `webhooks:full`

## API для управління підписками

```
POST /v1/webhooks    — створити підписку
GET  /v1/webhooks    — список підписок
```
