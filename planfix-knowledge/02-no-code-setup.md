# Planfix — Налаштування без коду

## Базові сутності

| Сутність | Опис |
|----------|------|
| **Акаунт** | Ізольована екосистема компанії на субдомені (yourcompany.planfix.com) |
| **Проєкт** | Папка для задач, деревоподібна ієрархія, кастомні права |
| **Задача** | Ключова сутність: терміни, чек-листи, виконавці, аудитори |
| **Контакт** | Фізособи та компанії в єдиній таблиці БД |
| **Угода** | Окремого модуля немає — угоди = задачі у відповідних процесах з кастомними полями |
| **Довідники** | Табличні структури для статичної інформації (прайси, реквізити юросіб) |
| **Шаблони** | Заготовки карток з виконавцями, чек-листами, значеннями полів |

## Порядок первинного розгортання

1. Системні параметри: мова, валюта, часовий пояс, графік роботи
2. Структура компанії: відділи, групи, облікові записи співробітників
3. Архітектура даних: довідники та кастомні поля для задач/контактів
4. Бізнес-процеси та статуси для різних типів робіт
5. Шаблони задач та автоматичні сценарії

> **Типова помилка:** спроба перенести логіку попередньої CRM без проектування зв'язків → дублювання полів та конфлікти сценаріїв

## Кастомні поля

- Конфігуруються окремо для задач, контактів, проєктів, користувачів
- **Обчислювані поля (тип 24):** формули як Excel (сума з ПДВ, автогенерація ПІБ)
- **Умовна видимість:** поле «Причина відмови» з'являється лише при статусі «Скасовано»
- **В API:** звернення лише за числовим ID або slug — звернення за назвою не підтримується ✅

## Воронки та статуси

- Кастомних статусів — необмежена кількість
- CRM-воронка = послідовний перехід задачі між статусами
- Можна обмежувати напрямки переходів: «Нова» → лише «В роботі» або «Відмова»
- Кожному статусу — свій колір у планувальнику

## Права доступу (матрична структура)

- Базуються на ролях у задачі (виконавець / аудитор / учасник) та приналежності до груп
- **Field-level security:** приховування/блокування полів залежно від ролі
- **Гостьовий доступ:** клієнти як контакти в особистому кабінеті (бачать лише свої задачі, без внутрішніх коментарів)

## Аналітика та звіти

- Конструктор звітів: задачі, час, контакти, фінансові транзакції
- Інтерактивні дашборди в планувальнику (реальний час)
- Воронка продажів = час перебування задачі в кожному статусі

## Email та комунікації

- Двостороння робота: IMAP/SMTP
- Вхідний лист на спецадресу → автоматично створює задачу
- Відповіді в коментарях → відправляються клієнту як email
- Шаблони листів: змінні `{{Task.Title}}`, `{{Contact.Name}}` + умовна логіка
- Вбудований корпоративний чат

## Часовий облік (Time Tracking)

- Ручне додавання аналітичних записів або інтерактивний таймер
- Звіти для розрахунку ЗП (погодинна ставка) або рентабельності проектів
- Порівняння планового часу з фактичним (контроль SLA)

## Фінансовий облік

- Реалізований через інструмент «Аналітика» (транзакційні записи)
- Доходи, витрати, рахунки, фактичні платежі — з прив'язкою до задачі/проекту/контрагента
- Автоматичний розрахунок планового та фактичного прибутку
