# Planfix — Архітектура та Модель Даних

## Нова архітектура: Objects (актуальна модель)

Центральна концепція нової моделі Planfix — **Object** («цифровий двійник» об'єкта або процесу):
- Приклади: угода, контракт, зустріч, стаття бази знань
- Кожен Object має: форму, набір статусів, скрипти, кнопки
- Стара модель «процеси + шаблони задач» → збережена лише для зворотної сумісності ⚠️

> При роботі з документацією: шукати статті про **Objects**, а не про старі «процеси».

## Основні сутності

Task, Project, Contact, User, Group, Handbook, Comment, File, Tag, **Object**

```
[Акаунт]
  |-- [Object] (нова модель: угода, контракт, будь-яка кастомна сутність)
  |      |-- [Форма + поля]
  |      |-- [Статуси]
  |      |-- [Скрипти + Кнопки]
  |-- [Проєкт] (деревоподібна вкладеність)
  |      |-- [Задача] (батьківські та підзадачі без обмеження рівнів)
  |            |-- [Коментарі]
  |            |-- [Файли]
  |-- [Контакт] (фізособи та компанії в єдиній таблиці)
  |-- [Довідники] (кастомні реєстри даних)
  |-- [Workspaces] (різні інтерфейси для різних ролей)
```

**Важливо:**
- Ліди та угоди = Objects відповідних типів (нова модель) або задачі у процесах (стара модель)
- Контакти фізосіб та компаній зберігаються в єдиній таблиці, зв'язок через поля «Контрагент» або «Список користувачів»

## Версії API

| Версія | Статус | Формат |
|--------|--------|--------|
| **REST API v2.0** (OpenAPI 3.0, v1.5.7) | ✅ Активний, пріоритетний | JSON |
| XML-RPC | Застарілий, але офіційно не deprecated ⚠️ | XML |

> Для всіх нових інтеграцій — лише REST API v2.0

## Кастомні поля в API

### Звернення до полів
- **За числовим ID** або системним ім'ям (slug) через параметр `fields`
- Звернення за відображуваною назвою **не підтримується**

### Отримання метаданих полів
```
GET /customfield/task       # поля задач
GET /customfield/contact    # поля контактів
GET /customfield/project    # поля проектів
```

### Складні типи полів
Для запису у поля типу «Запис довідника» або «Контакт» — передавати об'єкт `{"id": <target_id>}`

### Обмеження
- Типів полів: 0–29 за ідентифікаторами API (включаючи `AI results field`)
- **Ліміт**: до **150 полів у одному наборі**
- В шаблоні: основний набір + один додатковий одночасно
- Групування полів у набори — лише на вищих тарифах (Plan B / Plan X)
