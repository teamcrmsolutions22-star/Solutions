# Planfix — Архітектура та Модель Даних

## Основні сутності

Task, Project, Contact, User, Group, Handbook, Comment, File, Tag

```
[Акаунт]
  |-- [Проєкт] (деревоподібна вкладеність)
  |      |-- [Задача] (батьківські та підзадачі без обмеження рівнів)
  |            |-- [Коментарі]
  |            |-- [Файли]
  |-- [Контакт] (фізособи та компанії в єдиній таблиці)
  |-- [Довідники] (кастомні реєстри даних)
```

**Важливо:**
- Усі ліди та угоди — це задачі певних процесів
- Контакти фізосіб та компаній зберігаються в єдиній таблиці, зв'язок через поля «Контрагент» або «Список користувачів»

## Версії API

| Версія | Статус | Формат |
|--------|--------|--------|
| **REST API v2.0** (OpenAPI 3.0, v1.5.7) | ✅ Активний, пріоритетний | JSON |
| XML-RPC | Застарілий, але офіційно не deprecated ⚠️ | XML |

> Для всіх нових інтеграцій — лише REST API v2.0

## Кастомні поля в API

### Звернення до полів
- **За числовим ID** або системним ім'ям (slug) через параметр `fields`
- Звернення за відображуваною назвою **не підтримується**

### Отримання метаданих полів
```
GET /customfield/task       # поля задач
GET /customfield/contact    # поля контактів
GET /customfield/project    # поля проектів
```

### Складні типи полів
Для запису у поля типу «Запис довідника» або «Контакт» — передавати об'єкт `{"id": <target_id>}`
