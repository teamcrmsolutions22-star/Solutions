# Planfix — REST API: Технічні деталі

## Автентифікація

- **Протокол:** OAuth 2.0 (Bearer Token) ✅
- **Генерація токена:** «Керування акаунтом» → «Доступ до API» → «REST API»
- **Заголовок:** `Authorization: Bearer <access_token>`
- **Увага:** токен успадковує права того співробітника, який його створив — для інтеграцій рекомендується обліковий запис робота з максимальним доступом

## Базова структура запитів

| Параметр | Значення |
|----------|---------|
| Base URL | `https://<account>.planfix.com/rest/` або `https://<account>.planfix.ua/rest/` |
| Формат відповіді | JSON (метадані + масив результатів) |
| Пагінація | `limit` + `offset`, максимум **100 записів** за запит ✅ |
| Фільтрація | Об'єкт фільтрації у тілі POST-запиту (напр. `/task/list`) |
| Silent mode | `?silent=true` — без сповіщень і записів в історії ✅ |

## Rate Limits ✅

| Тип | Значення |
|-----|---------|
| Секундний ліміт | **1 запит/секунду** |
| Добовий ліміт | **20 000 / 50 000 / 100 000** (залежно від тарифу) |
| Free тариф | API повністю заблоковано |
| Заголовок контролю | `X-RateLimit-Remaining` (оновлюється кожні 10 хв) |
| При перевищенні | HTTP 429, код помилки 22 |

## Ключові ендпоінти

### Задачі
```
POST   /task              # створення задачі
GET    /task/{id}         # отримання задачі
POST   /task/{id}         # оновлення задачі
POST   /task              # підзадача: передати parent: {id: <parent_id>}
```

### Проєкти
```
POST   /project           # створення
POST   /project/list      # список
```

### Контакти
```
POST   /contact           # створення
POST   /contact/list      # пошук
```

### Коментарі
```
POST   /task/{id}/comments   # додавання коментаря
DELETE /comment/{id}         # видалення
```

### Файли
```
POST   /file              # завантаження (multipart/form-data) ✅
```
> Максимальний розмір файлу не задокументований ⚠️; обмежений дисковим простором акаунта

### Довідники
```
POST   /directory/{id}/entries   # отримання записів ⚠️
```

### Кастомні поля
```
GET    /customfield/task      # метадані полів задач
GET    /customfield/contact   # метадані полів контактів
GET    /customfield/project   # метадані полів проектів
```

## Пакетні операції

Нативна підтримка bulk-запитів **не задокументована** ⚠️  
→ Використовувати послідовні запити з throttling (1 запит/сек)

## Коди помилок

```json
{
  "error_code": 1,
  "error_message": "Invalid token"
}
```

| Код | Опис |
|-----|------|
| **1** | Невалідний токен |
| **6** | Доступ до методу/об'єкта заблоковано |
| **21** | API на безкоштовному тарифі (HTTP 403) |
| **22** | Перевищено добовий ліміт (HTTP 429) |
| **30** | Помилка парсингу JSON |
| **1000** | Задача не існує |

## HTTP 202 — Паралельне оновлення

Якщо задача оновлюється паралельно іншим сценарієм → HTTP 202  
**Це нормальна логіка Planfix** — запит стає в чергу автоматично, додаткових дій не потрібно ✅
