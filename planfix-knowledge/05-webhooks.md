# Planfix — Вебхуки та події

## Вихідні вебхуки (Outbound)

- Налаштовуються лише через **Автоматичні сценарії** в інтерфейсі адміністратора
- Спрацьовують за умовами (наприклад, лише при переході в статус «Оплачено» + заповнені реквізити)

**Події:**
- Створення, оновлення, видалення задач, контактів, проектів

**Payload:**
- Формат: JSON ✅
- Структура повністю кастомізується адміністратором через змінні системи
- Таймаут очікування відповіді від стороннього сервера **не задокументований** ⚠️

## Вхідні вебхуки (Inbound)

- Через **Chat API** або парсинг вхідних листів на спеціальні поштові адреси
- При отриманні: автоматичне створення задач або додавання коментарів до існуючих об'єктів

## Безпека

- **Shared Secret токен** — додається до URL або передається у JSON-структурі ✅
- Верифікація вхідних запитів від легітимних джерел

## Retry-логіка

- Кількість спроб та часові інтервали **не задокументовані** ⚠️
- **Рекомендація:** будувати ідемпотентні приймачі для уникнення подвоєння операцій

## Логування

- Логи вебхуків та результати сценаріїв: «Керування акаунтом» → «Процеси»
- Для зовнішнього дебагу: webhook.site для аналізу вихідних вебхуків

## Невизначеності

| Питання | Статус |
|---------|--------|
| Webhook HMAC-підпис (SHA256) | Не задокументований ⚠️ |
| Кількість retry-спроб | Не задокументована ⚠️ |
| Таймаут очікування відповіді | Не задокументований ⚠️ |
