# Planfix — Автоматизація та бізнес-логіка

## Сценарії автоматизації

Налаштовуються в інтерфейсі адміністратора в межах процесів задач або контактів.

## Тригери (події запуску)

| Тип події | Опис |
|-----------|------|
| Створення задачі | Новий запис у системі |
| Додавання коментаря | Фільтрація за ключовими словами чи командами |
| Зміна поля | Будь-якого системного або кастомного поля |
| Зміна статусу | Перехід між статусами процесу |

## Умови фільтрації

- Логічні оператори **І / АБО** за значеннями полів
- Перевірка ролей користувачів
- Перевірка робочого календаря
- Наявність зв'язаних об'єктів

## Дії сценаріїв

- Зміна значень полів
- Відправка email або повідомлень у месенджери
- Призначення виконавців
- Створення підзадач за шаблоном
- Виклик вихідного вебхука
- **Надіслати запит до Planfix AI** ✅

## Затримки

- Підтримка відкладеного виконання (таймери) відносно дати створення або зміни об'єкта

## Процеси (воронки)

- Кожен процес задає логіку переходів між статусами
- **Обов'язкові поля при переході:** неможливо перевести задачу в «Угода підписана» без завантаження файлу договору

## Сповіщення

| Канал | Деталі |
|-------|--------|
| Push у браузер | Підтримується |
| Email | З кастомізованим текстом через змінні |
| Telegram-боти | Підтримується |

## Планувальник (Scheduler)

- Автоматичне створення регулярних задач за розкладом
- Приклад: щотижнева задача на формування звітів
