# Planfix — Інтеграції та конектори

| Сервіс | Статус | Механізм |
|--------|--------|----------|
| **Електронна пошта** | ✅ | IMAP/SMTP або OAuth (Gmail, Outlook); листи → задачі |
| **Telegram** | ✅ | Пряма інтеграція; бот-нотифікації + ведення діалогів |
| **Viber** | ✅ | Пряма інтеграція |
| **WhatsApp** | ✅ | Пряма інтеграція |
| **Slack** | ⚠️ | Можлива через зовнішні конектори або API; нативна кнопка не задокументована |
| **Телефонія** | ✅ | Zadarma та інші АТС; фіксація дзвінків, запис, click-to-call |
| **Zapier** | ✅ | Офіційний конектор: стандартні тригери та дії |
| **Make (Integromat)** | ✅ | Офіційний конектор |
| **n8n** | ✅ | Generic HTTP-вузли з Bearer Auth або неофіційні ноди ⚠️ |
| **1С / BAS** | ✅ | Готові партнерські модулі або пряма розробка на REST API ⚠️ |
| **Checkbox (ПРРО)** | ⚠️ | Конфігурації маркетплейсу або кастомні API-сценарії ⚠️ |
| **Google Calendar** | ✅ | Двостороння синхронізація задач та подій |
| **Google Drive** | ✅ | Нативна інтеграція для зберігання/прикріплення файлів |
| **Dropbox** | ✅ | Нативна інтеграція |

## Вбудовані інтеграції месенджерів

- Planfix підтримує ведення діалогів з клієнтами безпосередньо з картки задачі
- Клієнт отримує відповідь у свій канал (Telegram/Viber/WhatsApp)
- Команда веде роботу в єдиній картці задачі (омніканал)
