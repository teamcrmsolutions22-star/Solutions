# Planfix — Дебаг та типові помилки

## Інструменти налагодження

| Інструмент | Де знайти |
|-----------|----------|
| Лог сценаріїв | «Керування акаунтом» → «Процеси» |
| Тестування API | Postman (зовнішній) |
| Аналіз вебхуків | webhook.site (зовнішній) |
| Виділений API-лог у UI | **Немає** ⚠️ |

## Типові помилки розробника

### HTTP 429 / Код 22 — Rate Limit
- **Причина:** >1 запит/сек або вичерпано добовий ліміт
- **Рішення:** реалізувати throttling; контролювати `X-RateLimit-Remaining`

### HTTP 202 — Паралельне оновлення
- **Причина:** задача оновлюється паралельно іншим сценарієм або API-запитом
- **Рішення:** нічого робити не потрібно — запит стає в чергу автоматично ✅

### HTTP 403 / Код 21 — Free тариф
- **Причина:** API-запит з акаунта на безкоштовному тарифі
- **Рішення:** перейти на платний тариф (Professional+)

## Підводні камені

### Права токена
- API-токен успадковує права того, хто його створив
- Якщо співробітнику заборонено бачити контакти в UI → API поверне помилку доступу
- **Рішення:** використовувати обліковий запис робота з максимальними правами

### Видалення кастомних полів
- Видалення поля → **безповоротне видалення всіх даних** у цьому полі по всіх задачах ⚠️
- **Рішення:** перед видаленням — повний бекап через «Керування акаунтом» → «Експорт даних»

### Звернення до полів в API
- Не можна звертатися за відображуваною назвою поля
- Тільки числовий ID або системний slug ✅
