обработки заявок мебельного производства Для этого перейдём в управление аккаунтом объекты и создадим новый Назовём его заявка В поле названия создаваемой задачи можно добавить текст символы или переменные Чтобы задачи создавались уже с нужными данными В объект можно добавлять существующие поля или создавать новые Например настроим поле для указания выбранного товара Оно будет брать данные из подготовленного заранее справочника с перечнем товаров Также добавляем числовое поле цена с доставкой и поле типа список с причинами отказа на случай отмены заявки Теперь удаляем неподходящие поля и располагаем оставшиеся в удобном порядке Переходим к набору статусов Он определяет этапы обработки заявок Удаляем неподходящие статусы и добавляем новые Указываем название и цвет Всё остальное можно оставить по умолчанию Настраиваем переход из этого статуса в другие и нажимаем добавить Также настраиваем остальные статусы чтобы набор полностью соответствовал этапам работы Теперь создадим на их основе канбан После дополнительной настройки он станет воронкой по обработке заявок Сценарии объектов автоматизируют работу Например могут подключать к заявке сборщиков и доставщиков на нужных этапах Создаём новый сценарий В настройках указываем название События для срабатывания Изменён статус задачи Дополнительные условия: статус равно сборка Какое действие нужно выполнить добавить к исполнителям группу Сборщики мебели Всё остальное можно оставить по умолчанию Теперь скопируем этот сценарий и заменим в нём данные чтобы настроить автоподключение для доставщиков Сохраняем настройки и готово
