# Planfix — Кейси по галузях та Сценарій Демо

## Детальні кейси від CRM Solutions

### Кейс 1: Наскрізний бізнес-процес від ліда до виробництва

**Галузь:** виробнича компанія, B2B  
**Об'єкти:** «Лід» (воронка продажів) + «Виробниче замовлення»  
**Інтеграції:** Tilda + Facebook → вхідні ліди → єдина точка входу  
**Автоматика:** при переведенні ліда в «Договір» → автоматично задача «Виробниче замовлення»  
**Фінанси:** аналітика «Специфікація» + обчислюване поле «Загальна сума» + кнопка «Виставити рахунок»

---

### Кейс 2: Helpdesk / Служба підтримки з SLA

**Галузь:** будь-яка з потоком звернень  
**Об'єкт:** «Звернення»  
**Статуси:** Нова → Прийнята → В роботі → Очікуємо клієнта → Вирішена → Закрита  
**SLA:** подія «перевищено час очікування» → сповіщення керівнику  
**Handover:** стоп-слово → «Передаю чат менеджеру» → зміна статусу → сповіщення відділу  
**Customer Feedback:** після закриття → NPS через Facebook Messenger або email

---

### Кейс 3: Агентство нерухомості

**Об'єкти:** «Об'єкт нерухомості» + «Угода» + «Клієнт»  
**Розклад по днях** — сітка об'єктів × дні → зайнятість показів  
**Інтерв'юер** — збирає параметри клієнта (бюджет, район, тип)  
**Автопідбір** — сценарій → API → пошук підхожих об'єктів → коментар зі списком

---

### Кейс 4: Б'юті-сфера (лазерна епіляція) — бот-консультант

**Сценарій:**
1. Клієнт пише боту → задача в ПФ
2. «Послати запит до Planfix AI» з базою знань у промпті → відповідь клієнту
3. AI паралельно витягує суть → оновлює поле «Опис»
4. Результат в «Опис»: `цікавлять підпахви → хоче комплекс → питає про діодний лазер`
5. Стоп-фактор → «Передаю чат менеджеру» → статус → сповіщення

---

### Кейс 5: Вхідні рахунки (бухгалтерія)

**Потік:** email із PDF → задача → AI аналізує PDF → заповнює реквізити (постачальник, сума, дата) → бухгалтер підтверджує

---

### Кейс 6: Облік часу і виплата зарплати

**Аналітики:** «Робочий час» (дата, від, до, вид) + «Витрати» (дата, стаття, сума)  
**Поля співробітника:** % від угоди, реквізити банку, посилання на дзвінок  
**Кнопка «Надіслати посилання на дзвінок»** → підтягує з картки → надсилає клієнту

---

### Кейс 7: Оренда техніки (посуточна)

**Об'єкт «Одиниця техніки»** + **«Бронювання»**  
**Розклад по днях** — рядки = техніка, колонки = дні  
**Перевірка доступності** через підсумки аналітики бронювань

---

### Кейс 8: Розподіл заявок (round-robin)

**Довідник «Черга менеджерів»:** 3 записи + поле «Поточний» (чекбокс)  
**При новій заявці:** API знаходить «Поточний = true» → призначає виконавця → переключає прапор на наступного

---

## Реальні кейси впровадження

### Логістика та кур'єрські служби
- Автоматизація обробки та розподілу замовлень
- Сценарії розподілу замовлень за зонами кур'єрів
- **Результат:** скорочення часу обробки + автоматичне оновлення статусів при доставці

### Digital-агентства та IT-розробка
- Конфігурація «Робота з задачами» + персоналізовані планувальники
- **Результат:** контроль дедлайнів, фіксація часу через таймери, розрахунок собівартості

### B2B продажі та виставлення рахунків
- Автоматизація первинних документів через довідники реквізитів
- **Результат:** генерація рахунків PDF за один клік + автоматична відправка клієнту + фіксація транзакцій

## Сценарій демо для клієнта (5 кроків)

### Для B2B-продажів або сервісних компаній:

**Крок 1 — Планувальник**  
Показати інтерактивний Planner: як співробітники бачать поточні задачі без хаосу

**Крок 2 — Тестова задача-угода**  
Створити задачу та показати автоматичне заповнення реквізитів контрагента з довідників

**Крок 3 — AI в дії**  
Написати тестовий коментар з командою `/chatgpt` або запустити Planfix AI → автоматична генерація відповіді клієнту

**Крок 4 — Автоматизація документів**  
Перевести задачу у фінальний статус → автоматична генерація друкованої форми рахунку

**Крок 5 — Аналітика керівника**  
Показати фінансовий звіт або дашборд: наскрізний контроль бюджетів
