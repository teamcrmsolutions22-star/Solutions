# Planfix — Quick Reference для інтегратора

## Параметри конфігурації

| Параметр | Значення | Маркер |
|----------|---------|--------|
| API тип | REST v2.0 (OAS 3.0) + XML-RPC (застарілий) | ✅ |
| Base URL | `https://<account>.planfix.com/rest/` | ✅ |
| Auth header | `Authorization: Bearer <TOKEN>` | ✅ |
| Rate limit | 1 запит/сек; 20k/50k/100k на добу | ✅ |
| Pagination | `limit` + `offset` (max 100 записів) | ✅ |
| Custom fields | За числовим ID або системним slug | ✅ |
| Files endpoint | `POST /file` (multipart/form-data) | ⚠️ |
| Max file size | Не задокументовано; обмежено дисковим простором | ⚠️ |
| Webhook security | Shared Secret токен | ✅ |
| Webhook retry | Не задокументовано; рекомендується ідемпотентність | ⚠️ |
| Bulk операції | Не підтримуються (max 100 об'єктів на читання) | ⚠️ |
| AI функції | Planfix AI (актуальна) + OpenAI ChatGPT | ✅ |
| Free тариф + API | API повністю заблоковано | ✅ |
| XML-RPC deprecated? | Офіційно не оголошено, але REST пріоритетний | ⚠️ |

## Невизначеності (потребують практичної перевірки)

| Питання | Як перевірити |
|---------|--------------|
| Формат кастомних полів в API | GET /task/{id} → аналіз масиву `customFields` |
| Добовий Rate Limit | Заголовок `X-RateLimit-Remaining` у кожному запиті |
| Webhook HMAC-підпис (SHA256) | Відправити тестовий вебхук на webhook.site → аналіз заголовків |
| AI-функціонал | Перевірити дію «Надіслати запит до Planfix AI» у редакторі сценаріїв |
| Ціни по доменах | Порівняти planfix.com/prices/ vs planfix.ru/prices/ |

## Відмінності між доменами

| Аспект | planfix.com | planfix.ru |
|--------|-------------|------------|
| Функціонал ядра | Ідентичний | Ідентичний |
| Валюта | USD/EUR | Локальна |
| Локальні інтеграції | Міжнародні сервіси | Адаптовані під ринок РФ ⚠️ |
