## Ключові доповнення до бази знань

### Архітектура Objects (нова модель)

Офіційна центральна одиниця платформи — **Object** («цифровий двійник» фізичного об'єкта або процесу):
- Приклади: контракт, зустріч, новина, стаття бази знань, угода
- У кожного Object: власна форма, набір статусів, об'єктні скрипти, кнопки
- **Стара модель** «процеси + шаблони задач» → збережена лише для зворотної сумісності ⚠️

> Практичний висновок: частина документації (справки) відноситься до СТАРОЇ архітектури. При впровадженні читати матеріали про **Objects**, ігнорувати застарілі статті.

## Автоматизація (важливі технічні деталі)

### Скрипти — асинхронна черга ⚠️
- Скрипти виконуються **асинхронно в окремій черзі**, не синхронно
- Якщо сценарій змінив об'єкт — він НЕ запускається повторно через власні зміни (захист від циклів)
- Система враховує порядок скриптів: якщо попередній сценарій вивів об'єкт з умов наступного — наступний пропускається

> Planfix краще проектувати як **чергу подій**, а не синхронний workflow-engine

### Кнопки
- Доступні лише на вищих тарифах (Plan B / Plan X)
- Хвилинний антиспам-обмежувач на одну кнопку
- Автоматичний сценарій може натискати кнопку, навіть якщо вона прихована від користувача

### Incoming Webhooks
- Спеціальні посилання для зовнішніх систем
- Дії: створення задачі, зміна задачі, додавання контакту
- У відповідь повертають JSON з номером створеної задачі ✅

### Email-обробка
- Правила обробки вхідної пошти
- Журнал вхідних листів зберігається **15 днів**
- Трекінг зовнішніх email: статуси відправки та прочитання (через піксель)

## API — повна картина

### REST API (новий)
- JSON + Bearer Auth
- `silent=true` — без сповіщень
- При паралельному оновленні задачі → `HTTP 202` (запит в черзі, нормальна поведінка)
- Вендор офіційно вказує: REST API **ще не повний** і поступово розширюється ⚠️

### XML API v1 (старий, але насичений)
- HTTP + XML, Basic Auth через токен
- Доступний лише на платних акаунтах
- Охоплює: проекти, задачі, статуси, дії, аналітики, співробітники, контакти, файли, довідники
- По задачах: додавання, зміна, оновлення кастомних полів, читання карток/списків, зміна строку, зміна статусу, отримання доступних статусів, управління виконавцями

### Спеціалізовані API
| API | Функції |
|-----|---------|
| **Telephony API** | Підключення хмарних АТС, ім'я абонента, маршрутизація, картка клієнта, запис дзвінків, click-to-call |
| **Chat API** | Двостороння інтеграція чатів |
| **SMS API** | Підключення власних SMS-провайдерів |
| **Email Marketing API** | createList, deleteList, importContacts, updateContact |
