# Planfix — Планувальники та Фільтри (детально)

> Джерело: YouTube-канал Planfix (транскрипти ~50 відео)

---

## Типи планувальників (представлень)

| Тип | Призначення |
|-----|------------|
| **Хроніка** | Список сповіщень про події в акаунті |
| **Ежедневник** | Стандартний планувальник зі зведеними даними |
| **Розклад** | Ежедневник «на боку» — по виконавцях/полях |
| **Календар по годинній сітці** | Почасовий розклад з аналітиками |
| **Простий список** | Швидке накидування задач (тільки постановнику) |
| **Таблиця** | Табличний вигляд (ліміт 150 задач одночасно) |
| **Деревовидний** | Ієрархія задач (над/підзадачі) |
| **Діаграма Ганта** | Часові залежності та зв'язки |
| **Місцезнаходження** | Географічний вигляд (потрібна інтеграція з картами) |
| **Список за аналітиками** | Групування за обраними аналітиками |

---

## Хроніка (Хроника)

- Всі сповіщення про події, що вимагають уваги
- Нові події зверху (за замовч.)
- Лічильник нових подій у вкладці браузера
- Зелена смуга = перехід за посиланням з хроніки
- Оранжева смуга = непрочитані
- Галочка = позначити як прочитане
- Підтримує нагадування для відкладання на пізніше
- Має бути порожньою — тоді всі задачі відпрацьовані

---

## Ежедневник

- Показує всі задачі зі зведеними даними
- Відображає аналітики, зокрема «планований час роботи»
- Перетягування задачі → додається аналітика «планований час»
- При перетягуванні між стовпцями → змінюється значення поля (наприклад, терміновість)
- Параметри відбору визначають, які задачі відображаються

---

## Розклад (Расписание)

- Ежедневник «на боку» — горизонтальне представлення
- «Дані для відображення» — вибір полів (виконавці, ресурси та ін.)
- Показує кілька людей паралельно з їх робочим часом
- Нерабочий час відображається блоком
- Підтримує «коротку картку» при кліку

---

## Почасовий календар (Календарь по часовой сетке)

- Сітка налаштовується (30 хв, 4 год, 20 хв і т.д.)
- Показує робочий і неробочий час виконавців
- Аналітики відображаються знизу, задачі зверху
- Для відображення потрібна аналітика в «Даних для відображення»

---

## Простий список

> ⚠️ Задачі в простому списку доступні **ТІЛЬКИ постановнику**

- Схожий на Trello — швидке накидування
- Інші учасники задачі можуть бачити тільки те, до чого мають доступ як учасники
- Без параметрів відбору

---

## Таблиця

> ⚠️ Технічне обмеження: максимум **150 задач одночасно**

- Якщо задач > 150 → частина не відображається
- Рішення: додати більш суворі фільтри

---

## Діаграма Ганта

### Ключові обмеження
- Тільки 4 стовпці: назва, початок, кінець, тривалість
- Кастомні стовпці **не додаються** ⚠️

### Типи зв'язків між задачами
| Тип | Позначення |
|-----|-----------|
| Кінець→Початок | Finish-to-Start (найпоширеніший) |
| Початок→Початок | Start-to-Start |
| Кінець→Кінець | Finish-to-Finish |
| Початок→Кінець | Start-to-Finish (найрідший) |

- **Запізнення (лаг):** позитивне (затримка) або від'ємне (старт до завершення попередника)
- **Ієрархічні зв'язки** (над/підзадачі): обмежені двома типами для уникнення циклів

### Колір задачі
- Синій = новий / в роботі
- Червоний = прострочений
- Оранжевий = дата початку настала, але виконавець ще не прийняв

### Віхи
- Задача з 0-денною тривалістю = ромб на діаграмі

### Базовий план
- Зберегти «знімок» діаграми на будь-яку дату
- Можна вмикати посередині проекту
- Порівняти поточні дати з базовими

### Масштаби
- Години, дні, тижні, місяці

### Мульти-проектний Ганта
- Кастомний фільтр → кілька проектів → групування за проектом

### Додаткові налаштування
- Дозволити підзадачі перевищувати дату батьківської задачі (опційно)
- Дозволити наступнику зміщуватись слідом за попередником

### Друк і експорт
- Друк: вибір стовпців, кольоровий, орієнтація
- Експорт у Excel через «Звіти» з шаблоном діаграми Ганта (не з самої діаграми)

| # | Причина | Рішення |

| 2 | Немає доступу до задачі | Надати доступ |

| 4 | Помилка в логіці фільтру | Виправити І/АБО/дужки (позначено червоним) |

| 7 | Задача по інтеграції з неправильним об'єктом | Змінити шаблон у налаштуваннях інтеграції |

| 12 | Задача в Простому списку — видна лише постановнику | Перенести в звичайний список з параметрами відбору |

## Налаштування фільтрів/планувальників

### Параметри відбору
- Умови з логікою `І` / `АБО` та дужками
- **Режим тестування**: вставте URL задачі → побачте, яка умова не проходить
- Задачі, створені у фільтрі, успадковують його параметри (шаблон, статус)
- **Останній статус у списку фільтру = статус за замовч. для нових задач** ⚠️

### Групування
- За будь-яким полем; вкладене угрупування можливе
- Поле має бути в об'єкті і доступне користувачу

### Підсумки в списках
- Автоматичний підрахунок кількості задач
- Сума за полями (наприклад, прибуток/збиток)
- Функція «обмеження» — забороняє перетягування більше певної кількості

### Загальні дії над задачами
- Виділити задачі → кнопка «Змінити» → додати аудитора, змінити статус, додати коментар, об'єднати, імпорт/експорт

### Доступ до фільтрів
- Надати доступ конкретним співробітникам/групам — можуть використовувати, але не редагувати
- `Три крапки → Налаштування → Змінити автора фільтру` — змінити власника
