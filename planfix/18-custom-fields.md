# Planfix — Кастомні поля (всі типи, деталі, підводні камені)

> Джерело: YouTube-канал Planfix (транскрипти ~50 відео)

---

## Де доступні кастомні поля

| Об'єкт | Кількість доступних типів |
|--------|--------------------------|
| Задачі | 26 (всі) |
| Контакти | 25 (немає: Сума підзадач) |
| Проекти | 23 (немає: Обчислювані, Підсумки аналітик, Сума підзадач) |
| Аналітики | 22 (немає: Файли, Підсумки аналітик, Сума підзадач, Результат навчання) |
| Довідники | 22 (немає: Обчислювані, Підсумки аналітик, Сума підзадач, Результат навчання) |
| Співробітники | 21 (немає: Обчислювані, Підсумки аналітик, Сума підзадач, Результат навчання) |
| Загальні поля | 19 (немає: Список, Набір значень, Підсумки аналітик, Сума підзадач, Місцезнаходження) |

**Ліміт:** до **150 полів в одному наборі** ⚠️

## Всі 26 типів полів задач

### 1. Рядок (Строка)
- Максимум **250 символів** (як назва задачі)
- Може відображатись як гіперпосилання (вибір протоколу: http, email, tel, file)

### 2. Число (Число)
- Підтримує: знаки після коми, розбивку по розрядах
- **Авто-нумерація** — загальна для всього сайту, НЕ для окремого шаблону ⚠️

### 3. Текст (Текст)
- До **16 МБ**
- Опційний розширений редактор (rich editor)

### 4. Дата (Дата)
- Календарний вибір
- Опція: автозаповнення поточною датою

### 5. Час (Время)
- Крок 5 хвилин
- Опція: автозаповнення поточним часом

### 6. Дата і час (Дата и время)
- Комбіноване поле

### 7. Часовий проміжок (Период времени)
- Діапазон в межах 24 годин

### 8. Прапорець (Чекбокс)
- Булеве поле (так/ні)

### 9. Список (Список)
- Одне значення
- Підтримує кольорові опції для кожного елементу
- ⚠️ Кеш при видаленні значень — проблеми в автоматизаціях
- Рекомендація: використовувати «Запис довідника» замість Списку (можна архівувати, редагувати з поля, без кешу)

### 10. Задача (Задача)
- Посилання на інші задачі з налаштованим фільтром

### 11. Проект (Проект)
- Посилання на проекти

### 12. Набір задач (Набор задач)
- Множинний вибір задач

### 13. Контакт (Контакт)
- Посилання на контакти з фільтром
- Можна створити нового контакту прямо з поля

### 14. Контрагент (Контрагент)
- Посилання на контакти/компанії
- Фільтрування за умовою

### 15. Співробітник (Сотрудник)
- Посилання на співробітників

### 16. Група співробітників або контактів (Группа сотрудников или контактов)
- Змішаний вибір

### 17. Список користувачів (Список пользователей)
- Множинний вибір співробітників/контактів/груп
- **Ліміт: 250 елементів**

### 18. Набір значень (Набор значений)
- Множинні текстові значення
- Два режими: вільний ввід або фіксований список

### 19. Файли (Файлы)
- Необмежена кількість файлів
- Немає обмежень на розмір файлу в налаштуваннях поля

### 20. Підсумки аналітик (Итоги аналитик)
- Підраховує суму значень з аналітик за полем (прапорець, часовий проміжок, обчислюване поле)
- Не редагується напряму — залежить від рядків аналітик

### 21. Обчислюване поле (Вычисляемое поле)
- Формули з використанням змінних і функцій
- Типи результату: рядок, число, текст
- **НЕ рахується в статусі «Чернетка»** ⚠️
- Ліміти за тарифом:
  - Professional: 5 шт.
  - Business: 10 шт.
  - Enterprise: 20 шт.

### 22. Місцезнаходження (Местоположение)
- Потрібна інтеграція з картами

### 23. Сума підзадач (Сумма подзадач)
- Сума з підзадач (число, обчислюване поле, підсумки аналітик)
- Підтримує обмеження глибини

### 24. Запис довідника (Запись справочника)
- Один пов'язаний запис
- Можна налаштувати відображення додаткових полів з довідника

### 25. Набір значень довідника (Набор значений справочника)
- Множинні пов'язані записи

### 26. Результат навчання (Результат обучения)
- Статистичне поле на основі полів типу «Список»
- Потрібно 1500+ заповнених задач для роботи

## Поля в аналітиках — специфіка

**Доступні всі типи крім:** Файли, Підсумки аналітик, Сума підзадач, Результат навчання

### Ключові налаштування полів аналітик
- `Обов'язкове для заповнення` — заповнити перед збереженням рядка
- `Переносити заголовок` — для довгих назв полів
- `Виводити підсумки аналітик` — авторозрахунок суми внизу
- `Авто-нумерація` — для поля «Години»
- `Значення за замовчуванням` — попереднє заповнення
- `Розбивати на групи розрядів` — для чисел

### Поле «Співробітник» в аналітиках
- Значення за замовчуванням: «поточний користувач»
- Варіанти відображення: ПІБ, окремо ім'я, прізвище, стать, телефон, коефіцієнт

### Обчислювані поля в аналітиках
- Посилаються на інші поля в тій самій аналітиці
- Можуть посилатись на поля запису довідника
- Використовують формули і функції
- Автоматичний перерахунок при зміні значень рядка аналітики

---

## Обчислювані поля — КРИТИЧНІ обмеження ⚠️

- Перераховується **ТІЛЬКИ** при зміні полів **ПОТОЧНОЇ** задачі
- ❌ НЕ перераховується при зміні полів в іншій задачі (контрагент, надзадача)
- ❌ НЕ перераховується якщо у формулі є `СЬОГОДНІ()`, `ЗАРАЗ()`, `ТДАТА()`
- Рішення: сценарій → «Перерахувати обчислюване поле»
- Може спрацювати через 2 хвилини — нормально
- **Масовий перерахунок:** виділити задачі → «Змінити» → «Перерахувати поле аналітик»

### Формула: поле контакту в задачі
```
{{Задача.Контрагент.Телефон}}
```
Шлях: Задача → Контрагент (системний) → Телефон → Всі телефони

## Важливі підводні камені

| Ситуація | Наслідок |
|---------|---------|
| Авто-нумерація числового поля | Наскрізна для всього акаунту (не per-шаблон) |
| Обчислюване поле в «Чернетці» | НЕ рахується до зміни статусу |
| Рядок — ліміт 250 символів | Як назва задачі — більше не вміститься |
| Список користувачів | Ліміт 250 елементів |
| Файли в аналітиках | **Не підтримуються** (в планах) |
| Поле «Запис довідника» | При записі через API — передавати `{"id": <target_id>}` |
| Зміна об'єкта в задачі | Статус скидається на ПЕРШИЙ статус нового об'єкта |

## Набори полів

- До **150 полів в одному наборі**
- В шаблоні: основний набір + один додатковий одночасно
- Групування полів у набори — лише на Plan B / Plan X
- Звернення через API: за числовим ID або slug, **не за відображуваною назвою** ⚠️

```
GET /customfield/task       # поля задач
GET /customfield/contact    # поля контактів
GET /customfield/project    # поля проектів
```
