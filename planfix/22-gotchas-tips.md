## Типові помилки при постановці задач

### Помилка 1: Задача без строку
- **Проблема:** завжди знайдеться щось терміновіше — задача загубиться
- **Правило:** якщо строк невідомий — поставити дату створення + 2 год як мінімум

### Помилка 2: Погане описання задачі
**Шаблон правильного опису (5 пунктів):**
1. Контекст — чому і навіщо потрібна задача
2. Що зробити виконавцю (детально)
3. Як розпланувати роботу
4. Чи потрібно закупити матеріали або залучити людей
5. Як контролювати і приймати роботу

**Ознаки поганого опису:**
- Немає дієслова дії («зробити», «виконати», «налаштувати», «дослідити»)
- Відсутній контекст — навіщо задача
- Для виконання потрібні додаткові консультації («спитай Петра»)

### Помилка 3: Задача без виконавця
- Задача існує, але ніхто про це не знає
- Наслідок: статус «в роботі», але ніхто не робить

### Помилка 4: Неправильна ієрархія підзадач
- Поганий приклад: «Ремонт» → 1) Фарбувати, 2) Демонтаж, 3) Проводка
- Правильно: чітка послідовність 1→2→3→4→5

### Помилка 5: Задачі без проекту
- При тисячах окремих задач браузер повільно вантажить
- Рішення: структурувати в проекти + фільтри

### Помилка 6: Видалення виконаних задач
> ⚠️ **Критична помилка** — ніколи не видаляйте виконані задачі

- При спорі з клієнтом — немає доказу виконання
- Зникають: коментарі, логи, прикріплені файли, артефакти
- Аналогічно: не видаляти контакти/угоди (втрачається история конверсії)
- **Правило:** виконані задачі переводити у відповідний статус (закрита, виконана), але не видаляти

### Помилка 7: Зберігання файлів у зовнішніх джерелах
- Скріншоти та файли з зовнішніх посилань → при зміні джерела посилання зламається
- Немає артефакту-доказу виконання (для судових справ)
- **Правило:** зберігати файли на внутрішньому сервері Planfix

## Про-поради

### Тайм-трекінг без розширень
**Реалізація на стандартних кнопках та сценаріях:**
1. Додати поле «Зараз роблю» (список видів робіт)
2. При виборі → автоматично фіксується дата/час початку
3. З'являється кнопка «Стоп»
4. При зупинці → додається аналітика з витраченим часом

- Працює в мобільному додатку ✅
- ⚠️ Коректно тільки при одному виконавці на задачі
- Бонус: керівник бачить, чим зайняті співробітники «прямо зараз»

### Пошук по словоформах
- Planfix підтримує пошук по словоформах (як Google/Yandex)
- «договора» знайде «договір», «договорам», «договорах»
- Два несвязаних слова в запиті → знайде об'єкт, що містить обидва слова

### Навігація клавіатурою
- Почніть вводити в пошуку → ↑↓ для навігації по результатах
- Enter → відкрити знайдений об'єкт
- Без миші, швидше

### Режим тестування фільтру
- При редагуванні списку → включити «режим тестування»
- Вставити URL задачі → система покаже, яка умова не проходить
- Незамінний інструмент для діагностики зникнення задач

### Кошик — відновлення та автоочищення
- В кожному розділі своя кошик (задачі, документи, контакти)
- Пошук по назві та фільтрація по проекту
- Звичайний співробітник відновлює тільки свої об'єкти
- Повне видалення — тільки адмін
- ⚠️ Автоочищення кожні **90 днів** → не розраховуйте на довше

### Сума підзадач для контролю проектів
- Поле «Сума підзадач» на головній задачі автоматично сумує значення дочірніх
- Контроль: весь запланований час/бюджет розподілений без решти
- Без розкриття всього дерева підзадач

### Приховані коментарі
- Опція «Приховати від решти» при відправці коментаря
- Видно тільки аудиторам/постановнику (налаштовується)

### Зверненнь до підтримки
- Окрема задача для кожного питання ⚠️ (не змішувати питання)
- Іконка «?» у правому верхньому куті → ліва панель → іконка звернення
- Всі запити автоматично в проекті «Planfix навчання і підтримка»

## Навмисна поведінка (не баги)

| Поведінка | Причина |
|-----------|---------|
| Таблиця мовчки приховує >150 задач | Технічне обмеження БД, не помилка |
| Простий список видно тільки постановнику | Архітектурне рішення (особисті нотатки) |
| Тест фільтру показує точну причину | Спеціальний інструмент діагностики |
| HTTP 202 при оновленні задачі | Паралельне оновлення — нормальна поведінка Planfix |
| Сценарій не запускається власними змінами | Захист від нескінченних циклів |
