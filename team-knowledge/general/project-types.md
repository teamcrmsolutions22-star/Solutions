# Виды проектов

Источник: Notion «Процесс внедрения CRM — коротко об этапах».

**Виды проектов → Активный / Пассивный.**

## Активный проект
Ведём с нуля до завершения.
- Для нас — дороже, проще и быстрее по времени.
- Для клиента — оперативнее решаются вопросы, выше скорость реагирования.
- **Этапы:** Ввод → Инициализация → Анализ → Настройка → Обучение → Техподдержка.

Делится по масштабу:
- **Крупный проект:** Планфикс / Битрикс / Ванбокс / **Zoho**.
- **Малый проект:** amoCRM / WeClients / Мой склад (в подписи также: KeyCRM / **KeepinCRM** / Asteril / **NetHunt**).

## Пассивный проект
Ранее настраивали мы / другой подрядчик / сам клиент — нам отдали на поддержание и доработки.
- Для нас — дороже по себестоимости, сложнее понять, кто что настроил.
- Для клиента — дольше решаются вопросы ТП.
- **Этапы:** Техподдержка / Доработки, Обучение.

> На странице есть 3 PDF («Для клиента», «Проектная работа», «Проект-шаблонка») — недоступны для скачивания, см. `../_pending-review.md`.
