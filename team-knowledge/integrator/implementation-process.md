# Процесс внедрения CRM (детальная карта)

Источник: «Производство CRM — детально» + под-страницы. Воронка стадий (BPMN-схема в PDF «Автоматизация Амо»): **Ввод → Инициализация → Анализ → Настройка → Защита/Тестирование → Обучение → Техподдержка → Обратная связь → Окончание**.

Роли: **Интегратор** (Сергей Тымчук — встречи/аудит/ТЗ), **PM/Клиент-менеджер** (контроль, сроки, счета), **Исполнитель** (настройка), **Помощник** (тестирование/обучение).

## 1. Ввод клиента в проект (мост Продажи → Производство) — PM
Проверка задачи «Предпродажная работа», объединение дублей контактов/задач, актуализация типа клиента (лид/аудит/клиент). Создание проекта в Планфикс (PM = клиент-менеджер и аудитор проекта), папок на Google Drive (Настройка, Оплата) с доступами, справочников логинов/паролей, карточек контактов/компании, рабочего чата Telegram (бот Планфикс). Кнопку «Инициализация» нажимает PM. Определить приоритет, оценить нагрузку, спланировать риски, согласовать с руководителем производства.

## 2. Инициализация клиента (Т ≈ 60 мин)
Клиенту на email через Планфикс (ссылка + PDF, неизменяемые): Правила игры, документы по CRM/телефонии, тарифы (калькулятор ПФ), форма «по желанию клиента», документ «До интервью». PM проговаривает содержание на встрече (фиксировать аудио/видео). Создание документа «Запрос данных для Амосрм» (из почты project@crmsolutions), отдача клиенту с видео и сроками, проверка заполненности.

## 3. Анализ (на объекте 2–5 дней / удалённо 5 дней; клиент оплачивает дни на объекте) — Интегратор
Цель — понять бизнес-процесс (коммуникации, отделы, этапы ЦКП). Этапы: глубинное интервью → выводы → ТЗ.
**Итог:** требования к системе (логические+функциональные), ТЗ (блок-схемы для разработчиков + этапы для пользователей), План действий (развёрнутое КП), рекомендации к БП.
Процесс: до (подтверждение встречи, аналитика расходов) → в процессе (регламент, визуализация БП, аудиозапись, перерывы раз в час) → после (видео-резюме, транскрибация «Хронология клиента», итог в чат, реквизиты 50% предоплаты). Защита анализа по видеосвязи + запись экрана.
Скрипт проведения — см. [`analysis-guide.md`](analysis-guide.md).

## 4. Настройка (период 2 недели; ≈8–15 ч в зависимости от сложности)
Интегратор ставит задачи и координирует; работа удалённая, время фиксируется в аналитике ПФ.
- **PM до:** проверка доступов/почты, шаблон задачи «Настройка Амосрм», подзадачи на программные доработки, создание аккаунта, внесение данных в справочник.
- **Исполнитель:** чек-лист, аналитика (вид+описание работы), складывает данные в справочник, при паузах пишет в комментарии.
- **PM после:** реагирует на уведомления и авто-уведомления (50% и конец срока), промежуточный отчёт клиенту (скрин в Telegram), проверка орг-части перед сдачей.
**Итог:** настроенная CRM (контакты, сделки, справочники, шаблоны задач, автоматизация, подключённые сервисы — почта/телефония/соцсети/сайты, отчёты). Еженедельный отчёт PM (видео+текст).

## 5. Защита настройки / Тестирование (Т = 1–2 ч)
Тестирует Исполнитель (кто настраивал) + PM (Ксюша), видеозапись без звука с таймкодами для нарезок клиенту.
**Чек-лист проверки:** 1) Контакт и Сделка (поля vs ТЗ); 2) Заявки по источникам (сайт/FB/Instagram/реклама/Telegram/Viber + автотеги); 3) Разбор письма; 4) Отчёты (теги/фильтры/рабочий стол — применить ко всем); 5) Воронка и этапы (по документу БП); 6) Автоматизация (розовые = ручные, голубые = авто; тестировать на наложения); 7) Виджеты (SMS-центр: шаблоны, альфа-имя, лимиты), Почта и телефония (вх/исх, записи звонков, интеграция); 8) Доступы в справочниках ПФ.
Клиенту: справочник логинов/паролей, хронология, видеоотчёт системы, видеоотчёт БП (со звуком).

## 6. Обучение (1 блок = 2 дня; группы до 5 чел.)
3 этапа: дистанционное по инструкциям → самостоятельное под клиента → видео/очно 2 ч с Исполнителем. 1-й день (4 ч) — Интегратор+Помощник (обучить, донастроить); 2-й день (8 ч) — Помощник (наблюдение, контроль Д/З). За 3 дня — базовый материал с тестами (авто), за 1 день — видео с ответами (авто), бонусы/медальки. **Экзамен:** анкетирование (гугл-форма, 2 ч) + интервью (2 ч), проверка в течение 1 дня. **Итог:** обученные сотрудники, протестированная работа, инструкция.

## 7. Техподдержка (ТП)
**KPI:** реакция 15–30 мин; решение 1–7 дней; работа 10:00–18:00 (в чате клиента иногда 11:00–21:00); запрос через Telegram; срочные Пн–Сб. Сроки: ТП amoCRM — 3 дня, ТП Планфикс — 7 дней, горящие — в тот же день.
Виды: **Пассивная** (входящий запрос — решаем; вина в настройке → за счёт компании, иначе счёт) и **Активная** (сами инициируем: еженедельные оповещения, созвоны с сотрудниками, проверка автоматизации раз в месяц).
Алгоритм: КМ отписывается клиенту (15–30 мин) и фиксирует в Excel ТП → Исполнитель решает в сервисах (amo/телефония/SMS) и отписывается → фиксирует выполнение. КМ контролирует чат/Excel/задачу ежедневно; раз в неделю — отчёт+счёт заказчику и созвон с сотрудниками; контроль из планировщика раз в 3 дня. Перед допуском Исполнитель защищает знание автоматизаций перед КМ.
Детальный алгоритм решения — см. `../playbooks-templates/support-algorithm.md`.

## 8. Обратная связь по проекту
Берёт тот, кто НЕ участвовал в производстве (Помощник/КМ, не PM и не Исполнитель). Активный проект — в конце + на промежуточных стадиях (после Анализа/Настройки/Обучения); пассивный — раз в месяц. Цель — как «психолог»: выяснить ощущения, показать превышение ожиданий (ценность), пресечь недовольство на стадии созревания. Скрипты — см. `../playbooks-templates/feedback-scripts.md`.

## 9. Окончание проекта
Документирование (даты, статус «Завершённый», все задачи в нужных статусах). Клиенту: доступы к аккаунту, хронология (отчёт+ссылка), видео работы системы, запуск задачи ТП. После: обратная связь (аудио+текст, КМ), видео- и письменный отзыв (КМ), отчёт с метриками результата (PM).

## Контроль и ретроспектива
- **Контроль стадий по проектам:** поле «кто что должен сделать» → чек-лист подзадачи с ответственным; отслеживание сроков запуска, ответил ли клиент, стадии «мои проекты».
- **Ретроспектива клиентов:** база «Проблемы клиентов и решения» + риски (отдельно по Планфикс и amoCRM).
- **Документы клиента:** Правила игры, документы по CRM/телефонии, запрос метрик (точка А) — высылаются на email до старта; называть по стандарту для поиска.
