# Должностная инструкция Исполнителя

Источник: Notion «Должностная инструкция Исполнителя» (Люди → Исполнитель).

## ЦКП (ценный конечный продукт)
Качественно и в срок выполненная задача. Соблюдение процесса выполнения задачи внутри команды.

## Обязанности
1. Выполнять работу по настройке систем. Цель — давать результат клиенту.
2. Ежедневно синхронизироваться с Руководителем отдела и Клиент-менеджером (вводные: информация, доступы, обратная связь). Цель — правильные приоритеты, быть в фокусе ситуации.
3. Присутствовать на еженедельном планировании (приём новых и анализ предыдущих задач).
4. Самостоятельно обучаться системам и правилам работы. Цель — экспертность.
5. Проходить обучение по системам с Интегратором и выполнять д/з.
6. Вести загрузку рабочего времени в Гугл-таблице (график, план задач, стадии, факт/план времени).
7. Сдавать отчёт за день.
8. Соблюдать алгоритм действий проектной работы, включая фактическое время (добавленное и описанное).
9. Соблюдать Правила работы (см. `process-of-work.md`).
10. Описывать инструкции, дополнять мануалы при несоответствии (актуальность: «память обнуляется» через 7 дней).
11. Полученные доступы клиента складывать в Справочник клиента в Планфикс (все доступы в одном месте).
12. Формировать документ просчёта (КП) по шаблону с плановым временем услуг (учёт рисков, конкурентоспособность).
13. Проводить консультации по подбору CRM (платные/бесплатные) — продажа через экспертность.
14. Изучать и искать виджеты и программные доработки для проектов внедрения.
15. Уважать людей в работе (без эмоций, конфликтов, провокаций, оскорблений; не пропадать со встреч без предупреждения).

## Структура описания работы (пример: виджет)
- Что это.
- Для чего (какие задачи решает).
- Нюансы в работе (минусы/ошибки).
- План время.
- Как настраивать (Шаг 1, Шаг 2 … с фото).
- Итог работы (в каком виде должна быть сдана задача).

## Нарушение сроков
- Нельзя нарушать сроки без уважительных причин; решение ТП > 5 дней по вине Исполнителя недопустимо.
- Последствие: задача забирается и отдаётся другому Исполнителю; время на первоначальное выполнение не оплачивается.

## Обязанности Руководителя (по отношению к Исполнителю)
1. Ставить задачу корректно по Правилам (некорректную Исполнитель вправе отклонить).
2. Проверять работу в течение 3 дней от даты выполнения.
3. Вовремя оплачивать ЗП.
4. Давать загрузку: ⅓ обучение, ⅔ выполнение работы (нельзя только обучение без оплаты, кроме стажировки).
5. Проводить 15-минутную синхронизацию перед началом работы.
6. Уважать человека.

> Портрет соискателя, профайлинг и условия оплаты — см. `../onboarding-hr/`.
