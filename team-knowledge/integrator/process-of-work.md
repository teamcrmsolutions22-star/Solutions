# Процесс выполнения работы

Источник: БД «Процесс выполнения работы» + «Правила работы для Исполнителя».

## Как видеть задачи
В разделе «Задачи» (фильтр по своему имени) и в Планировщике (название = твоё имя).

## Как принимать задачу
1. Задача приходит и уведомляет в Телеграм → принять или отклонить (через ПФ или телеграм-бот Планфикса); постановщику придёт уведомление.
2. Ознакомиться с описанием + видео/файлом (аккаунт клиента, данные, срок, аудитор).
3. Созвониться с постановщиком — **Координация** (убедиться, что задача понята правильно; чтобы не переделывать).
4. При выполнении использовать тайм-трекинг (плагин Chrome). В задаче есть «Плановое время» — среднее время в обычном режиме (без скорости Интегратора).
5. В отчётах видно затраченное время за задачу и за период.
6. После выполнения проверить и зафиксировать на видео/скриншотах → уведомление аудитору. Кнопки ПФ: «принять» (в работе), «выполнить» (выполнено), статус «Завершённая» = проверено без корректировок.

**Важно:** задача ставится минимум за 1 день до начала; даётся промежуток на выполнение (напр. задача на 40 мин → 2 дня).

## Как выставлять приоритеты
Срочно / не срочно → крайний срок → горизонт (день / 2 дня / неделя / недели / месяц). Сверху вниз по важности.

## Правила постановки задачи
Ставит Интегратор в начале проекта или Клиент-менеджер.
- **Название:** что сделать.
- **Описание:** Ситуация (почему) → Что делать → Как выполнить (методы/техника) → Где → Результат (ЦКП) + видеообъяснение + ссылки/доступы.
- **Поля:** Исполнитель, Аудитор, Проект, Дата начала. Необязательные: Срочность, Важность/Крайний срок, Координатор.

## Правила выполнения задачи
1. Ошибку в настройке исправляешь в рамках уже затраченного времени (вместе с последствиями) — не новая задача, не доплачивается.
2. Если не доделал задачу к дате окончания — оплата не начисляется (задачу переделывает другой). Не уверен в сроках — не принимай, отклони с аргументацией.
3. В аналитике «Факт время» прописывать вид работы + описание. Данные/ссылки для проверки — в справочник и комментарии (не в личку телеграм).
4. При паузах в проекте — отписываться в комментарии (чтобы не перепроверять) и уведомлять PM-отдел о причинах пауз/затянутых блоков.

## Работа аудитора (контроль)
- Перед началом — связаться с Исполнителями, пройтись по задачам (правильно ли поняли, нужна ли вводная). Правило: лучше сделать один раз правильно, чем переделывать.
- В процессе — реагировать на комментарии, давать данные; контролировать сроки, при невыполнении подменять Исполнителя.
- По окончании — проверить корректность и нажать «Завершить/Проверено».
