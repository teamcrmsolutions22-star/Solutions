# Процессы продаж

Источник: БД «Продажи» + ветка «Процессы → Продажа».

## Задачи отдела (с целями)
1. **Проводить платные/бесплатные консультации** → для производства = новые проекты.
2. **Делать просчёт времени и работ в КП** → новые проекты и полученные деньги.
3. **Выставлять чеки** (отчёт о выполненной работе) → полученные деньги.

## Процесс: Платная консультация / Аудит
Предпродажный процесс с маленьким чеком, через который продаём продукт с бо́льшим чеком.
- **Результат:** анализ аккаунта; что докрутить (продажи, маркетинг, технич. моменты — виджеты, лайфхаки); новинки CRM. Подход — сервисный, с «вау-эффектом».
- **Где видеть заявки:** в Сделках (шаблон «Предпродажная работа» — вид анализа, дата, время); удобно через Фильтр задач / Планировщик в Планфикс. За заполнение полей отвечает Отдел продаж.
- **Ход консультации Исполнителем:** *До* — КМ организует телеграм-группу и автоматизацию; Исполнитель знакомится с задачей, запрашивает доступы, вносит их в карточку контакта + срок лицензии. *В процессе* — огласить регламент, включить видеозапись, «слышать клиента», давать пользу. *В конце* — подробно описать аналитику (конкретно, с таймингом), дата/время начала-окончания, ссылка на видеозапись, чек-лист нерешённых вопросов, статус «Выполнено», резюме + след. шаг.

## Процесс: Составление КП
КП = задокументированный результат встречи/созвона с заказчиком. Содержит: рекомендации, бизнес-процесс клиента, подбор CRM и выгоды, этапы работы + партнёры. **Ответственный — Клиент-менеджер.** Время ≤ 1 час.
- После созвона Интегратор надиктовывает аудио (исходные данные, рекомендации, этапы БП) → КМ ставит задачу Исполнителю (план 1 ч, срок 1 день) → Исполнитель прописывает (тайм-трекинг «Визуализация КП») → проверяет КМ.
- Подготовка: проверить клиента в чате Телеграм «ЛИДЫ» → работать с контактом в Планфикс (переименовать, должность, почта, сфера, CRM, продукт, цель внедрения) → кнопка «Предпродажная работа» → сделка «предварительное согласие» (откуда узнал, тип работы, дата встречи/аудита, кто проводит, какая CRM, видео) → статус «Этап подготовка КП» → Google Drive (папка Лиды → год → шаблон) → калькулятор ценообразования (см. `kp-calculators.md`).

## Процесс: Выставление чеков (автоматизация Планфикс)
Информировать клиента о начислении денег; сократить время на счёт. **Работает только для проектов ПФ.**
- Каждую субботу клиенту в чат уходит уведомление: время и сумма за 7 дней по Коммуникации, ТП и Настройке (с учётом подзадач).
- Реализовано спец-полями + аналитиками + автоматизациями. **Важно:** при создании задач «Коммуникация» и «ТП» выставить дату завершения на субботу (старт автоматизации); в пятницу PM проверяет аналитики.

## Процесс: Поиск виджетов / программных решений (предпродажа)
Цель — найти готовое решение у подрядчиков или оценить стоимость своими силами. Т = 3 раб. дня. Виды доработок: программная (подрядчики) / ручная (команда).
- Поиск: документ «Виджеты» (Google Sheets), FB-группа amoPartners + Telegram «amoCRM helper», Google/YouTube, существующие программисты в системе.
- **Итог:** оформить в Notion (название, демо-период, стоимость, мин. срок оплаты, как работает, видео, контакты) + добавить программиста в базу (группа «представитель системы», шаблон «партнёр»).
