## Організаційна структура (Company & People)

- Створення відділів і підвідділів
- Прив'язка батьківський→дочірній відділ
- Профілі співробітників з кастомними полями
- Провізія: email-запрошення, пряме посилання, масовий імпорт

### Offboarding (звільнення)
Автоматизований workflow:
1. Відкликати доступ до workspace
2. Перепризначити активні завдання та відкриті ліди
3. Заархівувати комунікаційну історію

**Що це:** власні бізнес-об'єкти, якщо стандартних не вистачає.

**Налаштування при створенні:**

- `Support Products` — вмикає роботу з товарами в картці
- `Support Activities` — вмикає активності та прив'язку до інших сутностей

**Ліміти:**
- Standard: до 5 Smart Objects
- Professional: необмежено

**Джерело:** https://ua.uspacy.support/en/articles/8472836

## Автоматизація — деталі

### Process Builder
- Візуальний canvas для мультиетапних процесів
- Підтримка: умовні гілки, очікування, сповіщення
- Тригери: створення ліда, зміна етапу, зміна поля
- Дії: завдання, email, SMS, webhook, оновлення полів
- **Реліз:** ~квітень 2026 (вебінар #14, 14.04.2026)
- **Доступний:** тільки на максимальному тарифному плані
- **Розташування:** Ліве меню → Автоматизація → Процеси

#### Блоки конструктора

| Блок | Призначення |
|------|-------------|
| Тригер | Вибір об'єкта (сутності) + події (створення/зміна/тощо) |
| Фільтр (жовтий) | Умова «пропустити / не пропустити» далі по гілці |
| Розгалуження (Branch) | Кілька гілок з умовами; лівий→правий порядок перевірки; «catch-all» остання гілка |
| Отримати дані (Get Info) | Завантажити дані пов'язаної сутності (наприклад, компанії з ліда) |
| Створити | Створити будь-яку сутність (лід, угоду, завдання, РО, контакт) |
| Змінити поля | Змінити значення полів |
| Змінити етап | Перевести на інший статус/воронку |
| Видалити | Видалити елемент |
| Надіслати сповіщення | Push-нотифікація всередині Uspacy |
| Очікування (Wait) | Пауза: фіксований час або до дати з поля |
| Виконати вебхук | Зовнішній POST-запит |

#### Ключові налаштування
- Чекбокс **«Дозволити іншим процесам запускати цей»** (за замовчуванням OFF — важливо для уникнення циклів)
- Процеси можуть запускати один одного (двопроцесний патерн)
- Блоки перетягуються, з'єднуються стрілками

#### Приклади (з вебінару #14)
- **Розподіл лідів за UTM:** Тригер Лід→Створення → Розгалуження: UTM=facebook → Менеджер-1, UTM=google → Менеджер-2, catch-all → загальний
- **Реактивація залежалих:** Тригер Лід→Тайм-аут 7 днів → Сповіщення менеджеру → Wait 3 дні → Змінити відповідального
- **Онбординг за розміром:** Угода→Виграно → Get Info Компанії → Розгалуження: Enterprise → завдання «Виділений менеджер», SMB → email «Онбординг-матеріали»
- **Погодження відпустки:** РО «Відпустка»→Зміна статусу → Сповіщення HR → Фільтр > 14 днів → wait 48 год + сповіщення директору
- **Двопроцесний патерн:** Процес A (Завдання→Завершено) → запускає Процес B (Угода→зміна етапу)

#### Roadmap Process Builder (анонс квітень 2026)
- Травень 2026: Журнал/Історія виконання процесів
- Найближчі релізи: Блок «Дублювати елемент», Ручний тригер з картки сутності
- Далі: відправка через месенджери

### Conditional Actions (Умовні дії)
- Правила для карток завдань та CRM-полів
- Приклад: поле «Причина відмови» — обов'язкове тільки при переміщенні в «Програно»
- Ліміти: Standard — до 5 правил, Professional — необмежено

### Robots & Triggers
- **Robots** — виконують дії: надсилають email/SMS, призначають завдання
- **Triggers** — реагують на події: заповнення форми, зміна стадії

## Реєстрація та онбординг

### Реєстрація нового простору
1. Перейдіть на uspacy.ua → кнопка «Спробувати безкоштовно»
2. Вигадайте унікальну адресу простору (піддомен)
3. Введіть email → погодьтеся з умовами → «Створити свій Uspacy»
4. Підтвердьте email (перехід по посиланню з листа) — рекомендовано
5. Після підтвердження — вхід у простір

### Авторизація
- Пряме URL: `https://{ваш-піддомен}.uspacy.ua` → введіть email + пароль
- Або: uspacy.ua → «Увійти»
- Опція «Запам'ятати мене» зберігає авторизаційні дані на пристрої
- «Забули пароль» — відновлення через email

### Демо-режим
- Активується: Налаштування → Тариф та оплата → «Активувати демо»
- Тривалість: 14 днів
- Тариф: Professional (всі функції без обмежень)
- Після активації: потрібна переавторизація

## Ролі та адміністрування — деталі

### Власник простору
- Створюється автоматично при реєстрації
- Має позначку в списку співробітників та у профілі
- Повний доступ до всіх налаштувань

### Адміністратор
- Призначається власником або іншим адміністратором
- Через меню взаємодії у списку співробітників або профілі
- Кількість залежить від тарифу (Free: 1, Standard: до 5, Pro: ∞)
- Відмова прав: «Позбавити прав адміністратора» в тому ж меню

### Створення ролі
1. Налаштування → Ролі та права → «Створити»
2. Вказати назву та призначити користувачів
3. Налаштувати права для CRM: дозволити/заборонити для кожної сутності
   - Повний доступ, обмежений (лише свої елементи) або заборона
4. Professional: для лідів і угод — налаштування окремо по воронках
5. Налаштувати права для Smart Objects (окрема вкладка, та сама логіка)
6. Зберегти

## Організаційна структура — деталі

### Побудова структури компанії
- Розділ «Структура компанії» (ліве меню)
- За замовчуванням: один головний відділ зі всіма співробітниками
- Для створення відділу: назва (обов'язково), опис (опціонально), керівник, співробітники
- Місце у структурі: вищестоящий підрозділ = «Головний відділ» → стає відділом
- Для підвідділу: вищестоящий підрозділ = інший відділ/підвідділ

### Запрошення користувачів
1. Компанія та люди → Співробітники → «Запросити»
2. Два типи: по email (ім'я + пошта) або масове (тільки email через кому)
3. Запрошені відображаються у списку; після входу — статус «Активний»
4. Прийняття запрошення: через лист-запрошення або пряме посилання

### Кастомні поля для профілів співробітників
- Додаються в налаштуваннях Company & People
- Доступні типи полів: текст, число, дата, список, тощо
- Видимі у профілі кожного співробітника

## CRM — деталі роботи

### Ліди
- **Створення**: кнопка «Додати» на сторінці лідів, або quick-форма
- **Картка ліда**: назва, контактні дані, джерело, коментар, кастомні поля
- **Конвертація ліда** (кінцева обробка):
  - Неякісний: клієнт не купить → вказується причина
  - Якісний → конвертується в Угоду + Контакт + Компанія (у різних комбінаціях)
  - При конвертації: назва угоди = назва ліда; переносяться контактні дані, коментар, джерело
  - Вибір воронки та етапу при конвертації в угоду
- **Підтримка продуктів у лідах**: активується в Налаштування → CRM → Ліди → «Підтримка продуктів»
  - Після активації: вкладка «Продукти» у картці + поле «Сума» + автопідрахунок
  - На дошці відображається загальна вартість лідів по етапах

### Угоди
- Мультиворонки: Standard до 5, Professional необмежено
- **Створення воронки**: на сторінці угод → перемикач воронок → «Нова воронка»
  - Або: Налаштування → CRM → Угоди → «Створити нову воронку»
  - Налаштування етапів: назва, колір, причини провалу для фінального негативного етапу
- Переміщення угоди між воронками: через картку угоди → поле воронки

### Обов'язковість полів
- Налаштовується для конкретних воронок і їх етапів (CRM та Smart Objects)
- Приклад: поле «Причина відмови» стає обов'язковим лише на етапі «Програно»
- Шлях: Налаштування → CRM → відповідна сутність → налаштування воронки → поля

### Дублікати в CRM
- Автоматичний пошук при створенні нових елементів (за email/телефоном)
- Ручне об'єднання через меню взаємодії в картці

### Фільтри в CRM та Smart Objects
- **Збережені фільтри**: вибрати поля → встановити значення → «Зберегти фільтр» → ввести назву
  - Зберігається у вкладці «Збережені», застосовується як типовий
  - Умови: «Усі умови» (AND) або «Хоч одну умову» (OR)
- **Швидкі фільтри** (під рядком пошуку): максимум 5 полів
  - Змінити набір: Розширені налаштування → «Швидкі фільтри» → «Додати поля для фільтрації»
  - Повернути стандартні: кнопка «Відновити стандартний порядок полів»

### Пов'язані елементи
- З картки CRM можна одразу створити пов'язаний елемент іншої сутності (кнопка або вкладка)
- Зв'язки відображаються у картках обох елементів

### Продукти в CRM
- **Каталог продуктів**: CRM та продажі → Продукти
- **Поля продукту**: назва, категорія, тип (товар/послуга), артикул, ціна, валюта, податок (включений/не включений), одиниця виміру, наявність/залишок, опис, зображення, посилання на відео, коментар, кастомні поля
- Кілька валют: кнопка «Додати» для додаткових цін
- **Типи цін**: базова (за замовчуванням) + кастомні
  - Налаштування → CRM → Продукти → «Типи цін» → «Додати»
  - Один тип = основний (підставляється автоматично)
  - Видалення типу: поле в картках продуктів стає порожнім
- **Категорії продуктів**: мультирівнева структура (категорія → підкатегорія → ...)
  - Створення: сторінка продуктів / картка продукту / Налаштування → CRM → Продукти → Категорії

### Створення елементів CRM вручну

**Ліди**: CRM та продажі → Ліди → «Додати» → заповнити назву (обов'язково), контактні дані, джерело, коментар, кастомні поля  
**Контакти**: CRM та продажі → Контакти → «Додати» → прізвище + ім'я (обов'язково), прив'язати компанію, телефони/email (множинні), тип  
**Компанії**: CRM та продажі → Компанії → «Додати» → назва (обов'язково), альтернативна назва, сайт, телефони/email, прив'язати контакти  
**Угоди**: CRM та продажі → Угоди → «Додати» → назва + воронка (обов'язково), сума + валюта, контакти, компанія  
- Швидка форма угоди на дошці: «+» під назвою етапу → мінімальні поля → Enter або «Створити»
- З картки контакту/компанії — кнопка «Створити угоду»
- З ліда — конвертація (угода + контакт + компанія)

> У будь-якій картці CRM можна одразу створити нові кастомні поля, не виходячи з форми.

### Оплати (Monobank)
- Розділ: CRM та продажі → Оплати (кастбор + список)
- Статуси: Чернетка → Очікує оплати → Оплачено / Скасовано
- **Підключення**: Крамниця → застосунок Monobank → налаштування (токен з кабінету Monobank, сутності, термін посилання)
- Умова: у картці угоди/ліда/РО має бути сума + хоча б 1 продукт → «Прийняти оплату» → Monobank
- Посилання надсилається клієнту вручну (копіювання), платіж оновлює статус автоматично

### Справи (Activities) — деталі
- Типи: 5 варіантів (дзвінок, зустріч, лист, задача, інше)
- Прив'язка до: лід, угода, контакт, компанія, Smart Object, завдання
- **Календар справ**: 3 режими — день, тиждень, місяць
  - Кольорове позначення справ (задається при створенні)
  - Drag & drop зміна часу; у режимі «тиждень» — зміна тривалості
  - Закреслені = виконані або скасовані (залишаються видимими)
- **Розклад зайнятості**: перегляд вільних слотів перед плануванням справ
- **Резервування часу**: Справи → шестерня → «Резервування»
  - Розклади для різних типів послуг/менеджерів
  - Параметри: назва, тривалість, перерва між подіями, часовий пояс
  - Тип резервування: Справа, або Лід + Справа
  - Після резервування клієнтом → автоматично створюється справа (і лід)

## Smart Objects — деталі роботи

### Створення Smart Object
1. Налаштування → Smart Objects → «Створити»
2. Вказати назву
3. Опції:
   - `Support Directions` — вмикає Kanban (напрямки)
   - `Support Products` — вмикає вкладку «Продукти» в картках
   - `Support Activities` — вмикає справи та зв'язки
4. Зберегти

### Права доступу для Smart Objects
- Права налаштовуються в ролях (окрема вкладка)
- Права по напрямках — коли їх більше одного (аналогічно воронкам в CRM)

### Кошик (Recycle Bin) — деталі
- Окремий кошик для: завдань, справ, угод, лідів, контактів, компаній, кожного Smart Object
- Переміщення: меню взаємодії → «Видалити» (або масово)
- Нотатки та коментарі не потрапляють у кошик — видаляються назавжди
- Сортування в кошику: за датою видалення (найсвіжіші зверху)
- Тариф Standard: зберігання до 30 днів; Professional: необмежено

## Маркетингові інструменти

### Форми для збору лідів
- Розділ: Маркетинг → Форми (тільки адмін/власник)
- **Створення**: «З нуля» або з шаблону
  - Сутність: лід або контакт
  - Поля: стандартні + кастомні
  - Налаштування поля: назва, плейсхолдер, підказка, обов'язковість
  - Блок «Інші елементи»: логотип, заголовок, підзаголовок, згода на обробку (кастомний текст + посилання на політику)
  - Налаштування об'єктів: відповідальний, джерело (підставляється автоматично)
  - Кнопка надсилання: текст, іконка, колір (стандартний або hex-код)
- **Дизайн форми** (окрема вкладка):
  - Колір фону сторінки та форми
  - Стиль та заокругленість кнопки, розмір
  - Стиль полів (з обведенням або з підкресленням)
  - Розташування форми на сторінці (зверху / по центру)
  - Branding Uspacy (вмкн/вимкн)
- **Дії після заповнення** (окрема вкладка):
  - Повідомлення: заголовок, допоміжний текст, іконка
  - Перенаправлення на URL після надсилання (з затримкою в секундах)

### Email-розсилки
- Розділ: Маркетинг → Email розсилки → «Створити»
- Параметри: назва, відправник, отримувачі (збережені фільтри контактів = сегменти)
- Відображає кількість унікальних email-адрес
- Пакети листів: перший 1 000 безкоштовно; додаткові: 10 000 або 100 000
- Час надсилання: зберегти як чернетку / запланувати / надіслати одразу
- Відстеження: відкриття листа + переходи за посиланням (активується окремо)
- Листи створюються через конструктор або шаблони

## Пошта

- Розділ: Центр комунікацій → Пошта
- Підключення кількох скриньок (Gmail, Outlook та інші)
- **Функції**: перегляд, відповідь, пересилання, переміщення, видалення, масові дії
- **Підписи**: кожен користувач налаштовує свої підписи окремо
  - Декілька підписів; один — «Основний» (підставляється автоматично)
  - Підпис недоступний для перегляду/редагування іншими користувачами
- **Зв'язок пошти з CRM**: листи прив'язуються до карток угод/лідів/контактів
- **Синхронізація**: Free — раз на 60 хв, Standard — раз на 10 хв, Pro — раз на 5 хв

## Завдання — деталі

### Створення завдання
- **Повна форма**: всі поля — назва, опис, відповідальний, учасники, спостерігачі, дедлайн, пріоритет, кастомні поля, прив'язки до CRM
- **Швидка форма**: мінімальний набір → найшвидший старт
- З повідомлення/коментаря/нотатки: кнопка AI → «Завдання» або «Підзавдання»
- З листа в пошті: кнопка «Створити завдання» у верхній панелі листа

### Підзавдання
- Додаються всередині картки завдання
- Мають власні дедлайни, файли, виконавців
- Вкладеність необмежена (ієрархія)

### Чек-листи
- Додаються в картку завдання
- Кожен пункт можна відмітити як виконаний

### Терміни та дедлайни
- Deadlines Board — автосортування по дедлайнах
- Можна отримати повідомлення за N часу до дедлайну

### Kanban-дошка
- Drag & drop між стадіями
- Кастомні назви та кольори колонок
- «Прийняти результат виконання» — окрема кнопка в завданні (контроль виконання)

### Делегування (доручення виконання)
- Виконавець може перепризначити завдання іншому користувачу

### Шаблони завдань
- «Звичайний шаблон завдань» — для повторюваних завдань зі стандартними налаштуваннями

### Масові дії в завданнях
- Масове переміщення, видалення, зміна відповідального тощо

### Кастомні поля для завдань
- Налаштовуються в розділі Налаштування → Завдання
- Типи полів: текст, число, дата, список тощо

### Прив'язка чатів External Lines до завдань
- З діалогу: «Зв'язки з сутностями» → вкладка «Завдання» → «Додати» або «Нове завдання»

### Відстеження змін у завданні (Історія)
- Вкладка «Історія» у картці завдання
- Перший запис — створення завдання
- Фіксуються всі зміни: поля, статуси, учасники
- Фільтри: «Зміна статусу», «Інші зміни», «Всі зміни»

## Відстеження змін у CRM та Smart Objects (Історія)

- Перемикач «Історія» в кожній картці CRM/Smart Object
- Перший запис — факт створення елемента
- Фіксуються: зміни полів, зв'язків, переходи між стадіями, зміна відповідального
- Фільтри: «Зміна етапу», «Зв'язки з об'єктами», всі зміни
- Нові відповідальні бачать всю попередню історію

## Зовнішні канали (External Lines) — деталі роботи

### Центр комунікацій (журнал звернень)
- Показує всі діалоги з месенджерів та соцмереж + віджет чату
- Автоматично піднімає найактуальніші розмови
- Відображає: назву чату (нік клієнта з Telegram/Instagram/Viber тощо), учасників
- Статус чату: активний / неактивний / не взятий у роботу

### Швидкі відповіді
- Управління: Центр комунікацій → «Швидкі відповіді» (тільки адмін/власник)
- Кожна швидка відповідь: назва + текст (з форматуванням)
- У чаті: значок швидких відповідей → вибрати → підставляється в повідомлення
- Деактивована відповідь зникає зі списку вибору

### Розпізнавання телефону в External Lines
- Система автоматично шукає клієнта в CRM за номером телефону з месенджера

### Прив'язка чатів до завдань
- У чаті: «Зв'язки з сутностями» (правий верхній кут) → вкладка «Завдання»

## Стрічка новин та групи

### Стрічка новин (Newsfeed)
- Корпоративні пости: текст, медіа, посилання
- Коментарі та лайки до постів
- AI-резюме коментарів під постами

### Групи
- Публічні / приватні / секретні
- Kanban-дошка завдань у межах групи
- Прив'язка завдань до групи → завдання потрапляє на перший етап (беклог)
